## 🚀 **LOTIS — Примеры использования**

> Демонстрационные приложения на PHP-фреймворке LOTIS  
> Чистый код. Минималистичная архитектура. Максимальная гибкость.

---

### 📂 Структура репозитория

```bash
examples/
├── 1-hello.php             # Простой вывод
├── 2-formevent.php         # Форма с хуками
├── 3-charttest.php         # Интерактивная диаграмма
├── 4-converter.php         # Конвертер валют
├── 5-gridtest.php          # Адаптивная сетка
├── 6-dashboard.php         # Живая панель

└── 9-payment.php           # Полноценное приложение: документ
```

## 📘 Подробное описание каждого примера

---

### 1. `hello.php` 

> **Простейший вывод "Hello World" через `Div`**

Показывает базовое использование фреймворка:
- Создание контейнера через `LTS::Div()`
- Добавление текста через `capt('<h2>...')`
- Сборка через `Space`

🔹 Используется: `Div`, `capt`, `Space`  
🎯 Цель: начать с самого простого — убедиться, что всё работает.

---

### 2. `formevent.php` 

> **Форма с событиями и хуками: check, before, on**

Демонстрирует механизм обработки действий:
- Кнопка вызывает событие
- `checksave()` проверяет корректность
- `beforesave()` модифицирует данные
- `onsave()` реагирует на результат

🔹 Используется: `Form`, `Button`, `Events`, `method`  
💡 Особенность: полная цепочка жизненного цикла события  
🎯 Цель: показать, как контролировать поток данных

---

### 3. `charttest.php` 

> **Интерактивная диаграмма с переключением типа (line, bar, pie)**

Позволяет пользователю динамически менять тип графика.

🔹 Используется: `SimpleChart`, `Button`, `Div`, `rowbox`  
💡 Особенность: `LTS(chart).chart.setType('bar')` — прямое управление графиком  
🎯 Цель: визуализация данных с возможностью интеракции

---

### 4. `converter.php` 

> **Конвертер валют с внешним API (ЦБ РФ)**

Рассчитывает сумму в разных валютах на основе актуальных курсов.

🔹 Используется: `Input`, `Select`, `JS`, `Element`, `add('recount()')`  
🌐 Источник данных: `cbr-xml-daily.ru`  
💡 Особенность: серверный парсинг XML + клиентская логика  
🎯 Цель: интеграция с внешними данными и мгновенный расчёт

---

### 5. `gridtest.php` 

> **Адаптивная сетка с несколькими режимами отображения**

Показывает мощь `Grid` с поддержкой разных устройств: desktop, mobile, watch.

🔹 Используется: `Grid`, `device(...)`, `setMode(...)`, `priority()`  
💡 Особенность: `deviceQuery` определяет размер экрана  
🎯 Цель: создание адаптивного UI без CSS-медиазапросов

---

### 6. `dashboard.php` 

> **Живая панель управления с автообновлением**

Отображает ключевые метрики: выручка, задачи, температура, курсы валют.

🔹 Используется: `Vars`, `Events`, `Span`, `Grid`, `setInterval`  
⏱️ Автообновление: `LTS(events).refreshData()` каждые 30 секунд  
🎯 Цель: имитация реального времени в административной панели

---

### 7. `lookuptest.php` 

> **Поле выбора из справочника (например, «Услуги»)**

Пользователь выбирает элемент из выпадающего списка с поиском.

При первом запуске - указать актуальные параметры подключения к MySql, раскомментировать код создания базы данных
При последующих запусках после того, как база данных будет создана, этот код нужно снова закомментировать.

🔹 Используется: `DataTable`, `LookupField`, `search`, `out`  
💡 Особенность: `loadrowclick` загружает выбранный объект  
🎯 Цель: работа со связанными данными (many-to-one)

---

### 8. `tabletest.php` 

> **Таблица с формой редактирования строки**

Клик по строке — открывается форма. Есть добавление и удаление.

🔹 Используется: `DataTable`, `Form`, `rowclick`, `cells`, `signal`  
🛠 Хуки: `checkrowsave`, `before`, `on`  
🎯 Цель: стандартный CRUD-интерфейс в минималистичном виде

---

### 9. `payment.php` 

> **Полноценное приложение: документ «Оплата»**

Мощный `DataView` с табличной частью, сохранением, привязкой к БД. 

При первом запуске - указать актуальные параметры подключения к MySql, раскомментировать код создания базы данных
При последующих запусках после того, как база данных будет создана, этот код нужно снова закомментировать.

🔹 Используется: `DataView`, `subtable`, `bindtodb`, `Stock`, `collector`  
🔐 Проверки: `checkSave`, `beforeSave`, `onSave`  
📊 Связь: `kassatable` → `users`, `money`  
🎨 Макет: `setmodes`, `areas`, `columns`  
🎯 Цель: показать всю силу фреймворка в одном приложении
