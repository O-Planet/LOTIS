# Класс Lang

## Назначение
Класс локализации и поддержки мультиязычности фреймворка LOTIS, наследующий `Construct`. Предназначен для управления переводами интерфейса приложения, загрузки словарей из файлов и предоставления переведённых строк как на сервере, так и на клиенте. Автоматически определяет файл перевода на основе имени текущего скрипта и идентификатора языка. Генерирует клиентский JavaScript-код для доступа к переводам в браузере через `LTS.say()`. Поддерживает многострочные значения в файлах переводов и позволяет сохранять изменённые словари обратно в файлы.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$elements** | `array` | `private` | Массив метаданных для генерации клиентского JavaScript-кода. Заполняется методом `shine()` и используется при рендеринге пространства. |
| **$words** | `array` | `public` | Ассоциативный массив переводов. Ключи — идентификаторы строк, значения — переведённый текст. Поддерживает многострочные значения. |
| **$loaded** | `bool` | `private` | Флаг успешной загрузки файла перевода. Устанавливается в `true` если файл найден и прочитан, иначе `false`. |
| **$classname** | `string` | `public` | Наследуется от `Element`. CSS-классы элемента. Для Lang не используется (пустая строка). |
| **$type** | `string` | `public` | Наследуется от `Construct`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Идентификатор языка (например, `'ru'`, `'en'`, `'de'`). Используется для формирования имени файла перевода. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Construct`, инициализирует `$elements` пустым массивом, `$classname` пустой строкой, `$type` в `'html'`. Если `$id` не пуст — формирует путь к файлу перевода на основе имени текущего скрипта (`{filename}.{id}`) и вызывает `loadLangFile()`. Результат загрузки сохраняется в `$loaded`. |
| **say** | `string $word`, `string|bool $key = false` | `string` | Возвращает переведённую строку. Если `$key === false` — использует `$word` как ключ поиска. Если файл загружен и ключ найден в `$words` — возвращает перевод. Иначе — возвращает исходное слово. Если ключ не найден — добавляет его в `$words` со значением `$word` для последующего сохранения. |
| **loadLangFile** | `string $filePath` | `bool` | Приватный метод загрузки файла перевода. Читает файл построчно, игнорируя пустые строки. Строки без начального пробела считаются новыми ключами (формат `key:value`). Строки с начальным пробелом — продолжением предыдущего значения (многострочный текст). Заполняет массив `$words`. Возвращает `true` при успехе, `false` если файл не найден. |
| **save** | `string $lng` | `$this` | Сохраняет текущий словарь переводов в файл. Формирует имя файла на основе имени текущего скрипта и `$lng` (`{filename}.{lng}`). Для каждого ключа записывает первую строку значения после `:`, последующие строки — с отступом (4 пробела). Использует `file_put_contents()`. Возвращает `$this` для поддержки fluent interface. |
| **shine** | `Element $parent = null` | — | Генерация метаданных для клиентского JavaScript. Если файл загружен (`$loaded == true`) — кодирует `$words` в JSON через `json_encode()`. Заполняет `$elements['script']` массивом с типом `'JS'`, областью `'script'`, ID `0` и телом скрипта `LTS.lang('{id}', {json})`. Не создаёт DOM-элемент напрямую. |
| **addinspace** | — | — | Регистрация языковых данных в пространстве имён. Если файл загружен — добавляет `$elements['script']` в `$space->storage['elements']` через метод `set()` с ключом `$this->id`. Обеспечивает передачу словаря клиенту при рендеринге страницы. |

## Формат файла перевода

Файлы переводов используют текстовый формат с ключами и значениями:

```
welcome:Добро пожаловать
description:Это сайт о программировании
footer:© Все права защищены
multiline:Первая строка
    Вторая строка
    Третья строка
```

*   **Ключ**: Строка до первого двоеточия `:` (без начальных пробелов).
*   **Значение**: Строка после двоеточия.
*   **Многострочные значения**: Продолжения начинаются с пробела(ов) и добавляются к предыдущему значению через символ новой строки.
*   **Пустые строки**: Игнорируются при загрузке.
*   **Имя файла**: `{имя_скрипта}.{язык}` (например, `index.ru`, `index.en`).

## Глобальные функции LTS для работы с Lang

| Функция | Параметры | Возвращает | Описание |
|---------|-----------|------------|----------|
| **LTS.lang(lng, words)** | `string lng`, `object words` | — | Регистрирует словарь переводов для указанного языка на клиенте. Вызывается автоматически при рендеринге объекта Lang. |
| **LTS.say(word, key, lng)** | `string word`, `string key`, `string lng` | `string` | Возвращает переведённую строку на клиенте. Если `$key` не указан — использует `$word` как ключ. Если `$lng` не указан — использует текущий язык. Если перевод не найден — возвращает исходное слово. |

## Примечания
*   Класс расширяет `Construct`, поэтому поддерживает все хуки жизненного цикла: `$check`, `$before`, `$on`, `$checkchilds`, `$beforechilds`, `$onchilds`.
*   Класс не расширяет `Element` или `Quark` — не имеет визуального представления и не создаёт DOM-элементы.
*   Файл перевода ищется в той же директории, что и текущий скрипт — имя файла формируется как `{filename}.{id}` где `{filename}` — имя текущего PHP-файла без расширения.
*   Метод `say()` автоматически добавляет новые ключи в `$words` если они не найдены — это позволяет легко расширять словари в процессе разработки.
*   Метод `save()` полезен для экспорта накопленных переводов в файл — можно использовать для создания начальных словарей или обновления существующих.
*   Многострочные значения поддерживаются через форматирование с отступами — продолжения строк начинаются с пробела.
*   Клиентский словарь регистрируется через `LTS.lang()` — после этого доступен через `LTS.say()` в любом JavaScript-коде.
*   Методы `shine()` и `addinspace()` не создают визуальных элементов — Lang является функциональным объектом без DOM-представления.
*   Класс не реализует методы `compile()` и `childs()` — они наследуются от `Construct` без изменений (пустые реализации).
*   Флаг `$loaded` определяет, был ли найден файл перевода — если `false`, метод `say()` всегда возвращает исходное слово.
*   Для каждого языка создаётся отдельный объект Lang — можно использовать несколько языков одновременно в одном приложении.
*   JSON-кодирование словаря использует `JSON_UNESCAPED_UNICODE` — кириллица и другие Unicode-символы сохраняются без экранирования.
*   При сохранении через `save()` многострочные значения разбиваются на строки — первая строка идёт после `:`, остальные — с отступом 4 пробела.
*   Класс не поддерживает автоматическое определение языка браузера — язык задаётся явно через `$id` в конструкторе.
*   Для переключения языка нужно создать новый объект Lang с другим `$id` и перерендерить интерфейс.
*   Метод `say()` на сервере и `LTS.say()` на клиенте имеют одинаковую логику работы — возвращают перевод или исходное слово.
*   Словари не кэшируются между запросами на сервере — файл загружается при каждом создании объекта Lang.
*   На клиенте словари сохраняются в `LTS.languages` — доступны до конца сессии браузера.
*   Для больших проектов рекомендуется выносить файлы переводов в отдельную директорию и модифицировать логику загрузки в `loadLangFile()`.
*   Класс совместим с системой событий Events — можно использовать переводы в обработчиках событий на сервере и клиенте.
*   При отсутствии файла перевода объект Lang всё равно создаётся — `$loaded` будет `false`, но метод `say()` останется работоспособным.
*   Для добавления переводов "на лету" можно напрямую модифицировать `$words` до вызова `shine()`.
*   Метод `save()` перезаписывает существующий файл — для слияния с существующими переводами нужна дополнительная логика.
*   Ключи переводов регистрозависимы — `'Welcome'` и `'welcome'` считаются разными ключами.
*   Значения переводов могут содержать HTML-разметку — она не экранируется при выводе на клиенте.
*   Для безопасности рекомендуется проверять и очищать переводы из ненадёжных источников перед использованием.
*   Класс не поддерживает плюрализацию (множественное число) — для разных форм чисел нужны отдельные ключи.
*   Класс не поддерживает интерполяцию переменных в переводах — для подстановки значений используйте конкатенацию в коде.
*   Объекты Lang автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Construct`.
*   При рендеринге словарь передаётся клиенту через скрипт инициализации `LTS.lang("{id}", {body})` где `{body}` — JSON-представление `$words`.
