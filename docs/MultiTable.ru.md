# Класс MultiTable

## Назначение
Класс управления документами со структурой «шапка-детали» фреймворка LOTIS. Предназначен для работы с документами, состоящими из одной заголовочной таблицы (header) и нескольких подчинённых таблиц (details). Предоставляет единый интерфейс для чтения, сохранения и удаления полных документов включая все табличные части. Автоматически управляет связями между шапкой и деталями через поля связи (например, `parent_{table}`), поддерживает стратегии сохранения (replace/merge) и кастомизацию условий отбора для каждой табличной части.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$header** | `MySqlTable/null` | `private` | Таблица базы данных для заголовка документа. Устанавливается через конструктор или метод `header()`. Используется для операций с шапкой документа. |
| **$details** | `array` | `private` | Массив конфигураций подчинённых таблиц. Каждый элемент содержит: `table` (объект MySqlTable), `field` (поле связи), `alias` (псевдоним), `conditions` (условия отбора). Заполняется через метод `add()`. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `MySqlTable $header = null` | — | Конструктор класса. Вызывает `header($header)` для установки таблицы заголовка. Инициализирует `$details` пустым массивом. |
| **header** | `MySqlTable $header` | — | Устанавливает таблицу заголовка документа. Сохраняет в `$this->header`. Используется для всех операций с шапкой документа. |
| **add** | `MySqlTable $detailTable`, `string $name = null` | `$this` | Добавляет подчинённую таблицу к документу. Если `$name === null` — использует имя таблицы как псевдоним. Определяет поле связи как `"parent_{$header->name}"`. Сохраняет конфигурацию в `$details[$name]`. Поддерживает fluent interface. |
| **bindField** | `string|MySqlTable $name`, `string $field` | `$this` | Устанавливает поле связи для подчинённой таблицы. Если `$field === null` — связь будет только по условиям (conditions). Проверяет наличие детали через `resolveDetailName()`. Обновляет `$details[$detailName]['field']`. Поддерживает fluent interface. |
| **bindCondition** | `string|MySqlTable $name`, `mixed $condition` | `$this` | Добавляет условие отбора к подчинённой таблице. Принимает строку или массив условий. Добавляет в `$details[$detailName]['conditions'][]`. Поддерживает fluent interface. |
| **resolveDetailName** | `string|MySqlTable $name` | `string` | Приватный метод преобразования имени детали. Если передан объект MySqlTable — ищет по ссылке в `$details`, иначе возвращает имя таблицы. Если не найдено — использует `$name->name`. Возвращает строковое имя детали. |
| **getHeader** | — | `MySqlTable` | Возвращает объект таблицы заголовка. Прямой геттер для `$this->header`. |
| **getDetails** | — | `array` | Возвращает массив всех подчинённых таблиц. Прямой геттер для `$this->details`. |
| **getDetail** | `string $name` | `array/null` | Возвращает конфигурацию конкретной подчинённой таблицы по имени. Если не найдено — возвращает `null`. |
| **get** | `int $headerId` | `array/false` | Получает документ по ID шапки. Вызывает `all($headerId)`. Возвращает массив `['header' => obj, 'details' => [name => [...]]]` или `false` если не найдено. |
| **all** | `int $headerId`, `array $customConditions = []` | `array/false` | Получает документ с возможностью кастомизации условий. Читает шапку через `$this->header->get()`. Для каждой детали создаёт QueryBuilder, применяет базовые условия, поле связи и кастомные условия. Возвращает массив с шапкой и деталями или `false` если шапка не найдена. |
| **del** | `int $headerId` | `bool` | Удаляет документ и все подчинённые таблицы. Для каждой детали вызывает `delall()` с условиями связи. Затем удаляет шапку через `$this->header->del()`. Возвращает `true` при успехе, `false` при ошибке. |
| **applyDetailConditions** | `QueryBuilder $query`, `array $detail`, `int $headerId` | — | Приватный метод применения условий к QueryBuilder детали. Применяет базовые условия из `$detail['conditions']` и поле связи `$detail['field'] = $headerId`. Используется в `del()` и `save()`. |
| **save** | `array $data`, `string $saveMode = 'replace'` | `array/false` | Сохраняет документ и возвращает обогащённую структуру с ID. `$data` содержит `['header' => [...], 'details' => [name => [...]]]`. `$saveMode`: `'replace'` — удаляет отсутствующие строки, `'merge'` — только обновляет/добавляет. Сохраняет шапку через `insert()/set()`, затем детали через `set()` с привязкой к ID шапки. Возвращает `['header' => [...], 'details' => [...]]` или `false` при ошибке. |
| **hasDetail** | `string $name` | `bool` | Проверяет наличие подчинённой таблицы по имени. Использует `resolveDetailName()` и проверяет `isset($this->details[$detailName])`. |
| **removeDetail** | `string $name` | `$this` | Удаляет подчинённую таблицу из документа. Использует `resolveDetailName()` и `unset($this->details[$detailName])`. Поддерживает fluent interface. |
| **query** | `string|MySqlTable $name` | `QueryBuilder` | Возвращает QueryBuilder для подчинённой таблицы с уже применёнными условиями связи. Не добавляет поле связи автоматически — позволяет пользователю настроить запрос. Вызывает исключение если деталь не найдена. |

## Примечания
*   Класс не наследуется от `Construct` — это функциональный класс без жизненного цикла UI-объектов.
*   **Структура документа:** Документ состоит из одной шапки (header) и множества деталей (details) — каждая деталь представляет собой отдельную таблицу БД.
*   **Поле связи:** По умолчанию связь между шапкой и деталью определяется как `"parent_{header->name}"` — например, `parent_orders` для таблицы заказов.
*   **Режимы сохранения:** `'replace'` — удаляет строки детали которые отсутствуют во входных данных, `'merge'` — только обновляет существующие и добавляет новые.
*   **Кастомные условия:** Через `bindCondition()` можно добавить постоянные условия отбора для каждой детали — применяются ко всем операциям чтения/записи.
*   **Чтение документа:** Метод `all()` возвращает массив с ключами `'header'` (объект шапки) и `'details'` (массив массивов строк по имени детали).
*   **Удаление документа:** Метод `del()` сначала удаляет все строки деталей по условию связи, затем удаляет шапку — обеспечивает каскадное удаление.
*   **Сохранение шапки:** Если в `$data['header']` есть `id` — вызывается `set()`, иначе — `insert()` с возвратом нового ID.
*   **Сохранение деталей:** Для каждой строки детали устанавливается поле связи равным ID шапки — обеспечивает корректную привязку.
*   **Удаление строк детали:** В режиме `'replace'` вычисляются ID строк которые есть в БД но отсутствуют во входных данных — удаляются через `delall()`.
*   **QueryBuilder для детали:** Метод `query()` возвращает QueryBuilder с применёнными базовыми условиями — поле связи не добавляется автоматически для гибкости.
*   **Идентификация детали:** Через `resolveDetailName()` — поддерживает передачу имени строкой или объекта MySqlTable.
*   **Массив данных:** Входные данные для `save()` должны содержать `['header' => [...], 'details' => [name => [...]]]` — структура соответствует результату `all()`.
*   **Возврат из save():** Возвращает обогащённую структуру с актуальными ID всех сохранённых строк — удобно для последующей работы.
*   **Обработка ошибок:** Возвращает `false` при ошибке подключения к БД, неудачном сохранении шапки или отсутствии таблицы заголовка.
*   **Удаление по ID:** Метод `del()` принимает только числовой ID шапки — для удаления по другим условиям используйте QueryBuilder напрямую.
*   **Чтение по ID:** Метод `get()` является обёрткой над `all()` без кастомных условий — для простого получения документа.
*   **Проверка наличия:** Метод `hasDetail()` возвращает `true` если деталь была добавлена через `add()` — не проверяет реальное наличие в БД.
*   **Удаление детали:** Метод `removeDetail()` удаляет деталь из конфигурации — не удаляет данные из БД.
*   **Привязка поля:** Метод `bindField()` позволяет переопределить поле связи по умолчанию — полезно для нестандартных схем БД.
*   **Условия в save():** Кастомные условия из `bindCondition()` применяются при чтении старых строк для определения удаляемых записей.
*   **ID строк детали:** В `$data['details'][name][]` каждая строка может содержать `id` — если есть — обновляется, иначе — вставляется новая.
*   **Специальные ключи:** Ключ `ltsDELETED` в строке детали помечает строку как удалённую — пропускается при сохранении.
*   **Возврат QueryBuilder:** Метод `query()` позволяет строить сложные запросы к детали с JOIN, GROUP BY и др. — без привязки к шапке.
*   **Геттеры:** Методы `getHeader()`, `getDetails()`, `getDetail()` предоставляют прямой доступ к внутренней структуре — для отладки и расширения.
*   **Конструктор:** Принимает опциональную таблицу заголовка — можно установить позже через `header()`.
*   **Fluent interface:** Методы `add()`, `bindField()`, `bindCondition()`, `removeDetail()` возвращают `$this` для цепочечных вызовов.
*   **Приватные методы:** `resolveDetailName()` и `applyDetailConditions()` используются внутри класса — не предназначены для прямого вызова.
*   **Типы параметров:** Методы принимают `string|MySqlTable` для имени детали — автоматически преобразуют к строковому имени.
*   **Массив conditions:** В `$details[$name]['conditions']` хранятся условия — применяются через `QueryBuilder::where()` или `addCondition()`.
*   **Связь с ORM:** Использует классы `MySqlTable`, `QueryBuilder` — все операции выполняются через ORM фреймворка.
*   **Транзакции:** Для атомарности рекомендуется оборачивать `save()` и `del()` в `$header->owner->transaction()`.
*   **Валидация данных:** Класс не валидирует входные данные — ответственность на разработчике перед вызовом `save()`.
*   **Кэширование:** Не кэширует данные — каждый вызов `all()` выполняет запрос к БД.
*   **Производительность:** Для больших документов рекомендуется использовать `FIELDS` в условиях для ограничения выбираемых полей.
*   **Наследование:** Класс не поддерживает наследование — для расширения используйте композицию или обёртки.
*   **Статические методы:** Не содержит статических методов — требует создания экземпляра для работы.
*   **Зависимости:** Требует подключенной БД через `MySql` — все таблицы должны иметь `$owner`.
*   **Именование:** Рекомендуется использовать стандартное именование полей связи `parent_{table}` для совместимости.
*   **Множественные детали:** Поддерживает неограниченное количество подчинённых таблиц — каждая добавляется через `add()`.
*   **Псевдонимы:** Псевдоним детали используется как ключ в `$details` и в возвращаемых данных — должен быть уникальным.
*   **Поле связи null:** Если `$detail['field'] === null` — связь определяется только через `conditions` — для сложных схем.
*   **Возврат из all():** Если шапка не найдена — возвращает `false`, если детали пусты — возвращает пустые массивы в `'details'`.
*   **Ошибки QueryBuilder:** Метод `query()` вызывает `InvalidArgumentException` если деталь не найдена — требуется проверка через `hasDetail()`.
*   **Сохранение без details:** Можно сохранить только шапку без деталей — `$data['details']` может быть пустым или отсутствовать.
*   **Удаление без details:** Метод `del()` удаляет детали даже если они не переданы во входных данных — по условию связи.
*   **Режим merge:** В режиме `'merge'` строки детали не удаляются — только обновляются существующие и добавляются новые.
*   **Режим replace:** В режиме `'replace'` строки которых нет во входных данных удаляются — для полной синхронизации.
*   **ID из insert():** После вставки шапки новый ID используется для привязки всех строк деталей — обеспечивает целостность.
*   **freevalues():** После сохранения вызывается `$table->freevalues()` — сбрасывает флаги `$isset` для полей.
*   **Массив oldrows:** В `save()` загружаются старые строки детали для определения удаляемых — через `$this->dbtable->all($conditions)`.
*   **Массив delids:** Вычисляются через `array_diff($oldids, $myids)` — ID которые есть в БД но отсутствуют во входных данных.
*   **Цикл сохранения:** Для каждой строки детали вызывается `set($rowid)` — если `rowid` пуст — `insert()`.
*   **Возврат true/false:** Метод `save()` возвращает массив данных при успехе, `false` при ошибке — требуется проверка.
*   **Возврат из del():** Возвращает `true` если все операции успешны, `false` при ошибке удаления шапки или деталей.
*   **Геттер getDetail():** Возвращает массив конфигурации или `null` — для проверки наличия и получения настроек детали.
*   **Массив details:** Структура: `[$name => ['table' => ..., 'field' => ..., 'alias' => ..., 'conditions' => [...]]]`.
*   **Имя по умолчанию:** Если `$name === null` в `add()` — используется `$detailTable->name` как псевдоним.
*   **Проверка owner:** Методы проверяют `$this->header->owner !== null` — требуется подключенная БД для работы.
*   **Подключение БД:** Если `$this->owner->link === null` — вызывается `connect()` — автоматическое подключение при первом запросе.
*   **Обработка null:** Если `$data['header']` не задан но есть `$data['headerid']` — используется существующий ID для обновления.
*   **Специальные поля:** Поле `id` в шапке обрабатывается отдельно — не передаётся в `value()` для вставки.
*   **Массив rows:** В `save()` `$rows = $detailsData[$name]` — массив строк конкретной детали из входных данных.
*   **Поле ltsDataId:** Специальное поле для связи с клиентской частью — не сохраняется в БД, используется для идентификации.
*   **Возврат savedRows:** В `save()` для каждой детали возвращается массив сохранённых строк с актуальными ID.
*   **Проверка is_numeric:** В `get()` и `exists()` проверяется `is_numeric($id)` — для безопасности от SQL-инъекций.
*   **LIMIT 1:** В `get()` и `exists()` используется `LIMIT 1` — для оптимизации запросов к БД.
*   **prepare/execute:** Все запросы используют prepared statements — защита от SQL-инъекций через `bind_param()`.
*   **fetch_object:** Результаты читаются через `fetch_object()` — возвращаются как stdClass с доступом через `->property`.
*   **stmt->close():** После выполнения запроса вызывается `$stmt->close()` — для освобождения ресурсов БД.
