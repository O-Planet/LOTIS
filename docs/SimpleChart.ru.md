# Класс SimpleChart

## Назначение
Класс визуализации данных фреймворка LOTIS, наследующий `Element`. Предназначен для создания интерактивных графиков и диаграмм на основе HTML5 Canvas. Поддерживает множественные типы визуализации: линейные графики (line), столбчатые диаграммы (bar), круговые диаграммы (pie, doughnut) и составные круговые диаграммы (stacked-pie). Класс автоматически генерирует клиентский JavaScript-код для рендеринга графиков, предоставляет методы для динамического обновления данных, изменения типа диаграммы и управления наборами данных. Интегрируется с системой событий LOTIS для реакции на изменения данных.

## Свойства (серверная часть, PHP)

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$config** | `array` | `private` | Конфигурация графика. Содержит ключи: `type` (тип диаграммы), `data` (данные: labels, datasets), `options` (дополнительные настройки). Инициализируется в конструкторе с типом `'line'` и пустыми данными. |
| **$tagname** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'canvas'` в конструкторе. Определяет HTML-тег элемента при рендеринге. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор графика, используется для связи с клиентским объектом в `ltsSimpleChart`. |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Свойства (клиентская часть, JS)

| Свойство | Тип | Описание |
|----------|-----|----------|
| **charts** | `object` | Хранилище активных графиков по ID. Каждый элемент содержит: `config` (конфигурация), `ctx` (canvas context), `canvas` (DOM-элемент). |

## Методы (серверная часть, PHP)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`, устанавливает `tagname` в `'canvas'`, инициализирует `$config` с типом `'line'`, пустыми labels и datasets. |
| **type** | `string $type` | `$this` | Устанавливает тип диаграммы. Допустимые значения: `'line'`, `'bar'`, `'pie'`, `'doughnut'`, `'stacked-pie'`. Сохраняет в `$config['type']`. Поддерживает fluent interface. |
| **label** | `string $label` | `$this` | Добавляет одну метку в массив labels. Добавляет значение в `$config['data']['labels']`. Поддерживает fluent interface. |
| **labels** | `array $array` | `$this` | Устанавливает массив меток. Заменяет `$config['data']['labels']` на переданный массив. Использует `array_values()` для нормализации ключей. Поддерживает fluent interface. |
| **autolabels** | `int $min`, `int $max`, `int $step = 1` | `$this` | Генерирует числовые метки от min до max с заданным шагом. Создаёт массив строк через цикл for. Вызывает `labels()` для установки. Поддерживает fluent interface. |
| **dataset** | `mixed $data`, `array $options = []` | `$this` | Добавляет набор данных. Поддерживает: (1) Числовой массив — `[10, 20, 30]`. (2) Ассоциативный массив — `['Пн' => 10, 'Вт' => 20]` (автоматически дополняет нулями по labels). Создаёт объект dataset с `data` и `borderWidth`. Цвета генерируются на клиенте если не заданы. Сохраняет в `$config['data']['datasets']`. Поддерживает fluent interface. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Кодирует `$config` в JSON. Создаёт JS-объект с методами: `render()`, `update(data)`, `setType(type)`, `addDataset(dataset)`, `destroy()`. Добавляет автоматический вызов `render()` после создания. Вызывает `parent::compile()` в конце. |

## Методы (клиентская часть, SimpleChart.js)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **ltsSimpleChart.create** | `string id`, `object config` | — | Создаёт новый график или обновляет существующий. Находит canvas по ID, получает 2D context. Сохраняет config, ctx, canvas в `charts[id]`. Вызывает `render(id)` для отрисовки. |
| **ltsSimpleChart.render** | `string id` | — | Отрисовывает график. Очищает canvas через `clearRect()`. Вычисляет размеры области построения с учётом padding. В зависимости от типа вызывает: `drawLine()`, `drawBars()`, `drawPie()`, `drawStackedPie()`. В конце вызывает `drawLegend()` для всех типов. |
| **ltsSimpleChart.update** | `string id`, `object newData` | — | Обновляет данные графика. Использует `Object.assign()` для слияния новых данных с существующими. Вызывает `render(id)` для перерисовки. |
| **ltsSimpleChart.setType** | `string id`, `string type` | — | Изменяет тип графика. Обновляет `config.type` в хранилище. Вызывает `render(id)` для перерисовки с новым типом. |
| **ltsSimpleChart.addDataset** | `string id`, `object dataset` | — | Добавляет новый набор данных. Добавляет dataset в `config.data.datasets` через `push()`. Вызывает `render(id)` для перерисовки. |
| **ltsSimpleChart.destroy** | `string id` | — | Удаляет график. Находит canvas, удаляет из DOM через `parentNode.removeChild()`. Удаляет запись из `charts[id]`. |
| **ltsSimpleChart.drawLine** | `ctx`, `data`, `xScale`, `yScale`, `padding`, `chartHeight` | — | Рисует линейный график. Проходит по данным, строит линии через `moveTo()`/`lineTo()`. Добавляет точки на каждом значении через `arc()`. |
| **ltsSimpleChart.drawBars** | `ctx`, `data`, `xScale`, `yScale`, `padding`, `chartWidth`, `chartHeight`, `index`, `total` | — | Рисует столбчатую диаграмму. Вычисляет ширину группы и одного столбика. Рисует прямоугольники через `fillRect()` и `strokeRect()`. |
| **ltsSimpleChart.drawPie** | `ctx`, `datasets`, `centerX`, `centerY`, `radius`, `totalRadius` | — | Рисует круговую или кольцевую диаграмму. Вычисляет общую сумму, толщину колец пропорционально данным. Рисует сектора через `arc()`. После отрисовки рисует белые границы между секторами. |
| **ltsSimpleChart.drawStackedPie** | `ctx`, `datasets`, `centerX`, `centerY`, `radius` | — | Рисует составную круговую диаграмму. Один сектор — одна метка, внутри сектора — части по каждому dataset. Вычисляет общую сумму по каждой метке. Рисует части сектора через `arc()` с белыми границами. |
| **ltsSimpleChart.drawLegend** | `ctx`, `datasets`, `padding`, `width` | — | Рисует легенду для всех типов диаграмм. Рисует цветные квадраты 15x15px с подписями. Автоматически переносит на новую строку если не хватает места. |
| **ltsSimpleChart.getColor** | `int index` | `string` | Генерирует уникальный цвет на основе индекса. Использует "золотой угол" (137.508°) для максимального разнообразия. Варьирует насыщенность (70-85%) и светлоту (55-75%). Возвращает строку RGBA через `hslToRgba()`. |
| **ltsSimpleChart.hslToRgba** | `int h`, `int s`, `int l`, `float a = 0.7` | `string` | Конвертирует HSL в строку RGBA. Использует стандартный алгоритм конвертации через промежуточные p, q значения. Возвращает строку вида `rgba(r, g, b, a)`. |
| **ltsSimpleChart.hue2rgb** | `float p`, `float q`, `float t` | `float` | Вспомогательная функция для преобразования оттенка в RGB. Обрабатывает диапазоны t < 0, t > 1, t < 1/6, t < 1/2, t < 2/3. Возвращает значение канала 0-1. |

## Методы (клиентская часть, генерируются при compile)

Эти методы становятся доступны на клиенте через `LTS.get('{id}').chart` после компиляции.

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(chart).chart.render()** | — | — | Отрисовывает график с текущими данными. Вызывает `ltsSimpleChart.create()` с ID и конфигурацией. |
| **LTS(chart).chart.update(data)** | `object data` | — | Обновляет данные графика. Вызывает `ltsSimpleChart.update()` с новыми данными. |
| **LTS(chart).chart.setType(type)** | `string type` | — | Изменяет тип диаграммы. Вызывает `ltsSimpleChart.setType()` с новым типом. |
| **LTS(chart).chart.addDataset(dataset)** | `object dataset` | — | Добавляет новый набор данных. Вызывает `ltsSimpleChart.addDataset()`. |
| **LTS(chart).chart.destroy()** | — | — | Удаляет график. Вызывает `ltsSimpleChart.destroy()`, удаляет canvas из DOM. |

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   **Зависимость от SimpleChart.js:** Для корректной работы клиентских методов необходимо подключение файла `SimpleChart.js`, который содержит реализацию объекта `ltsSimpleChart`. PHP-класс `SimpleChart` только генерирует конфигурацию и вызовы методов.
*   Все методы настройки графика возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Типы диаграмм: `'line'` (линейный), `'bar'` (столбчатый), `'pie'` (круговой), `'doughnut'` (кольцевой), `'stacked-pie'` (составной круговой).
*   Метод `dataset()` автоматически сопоставляет ассоциативные массивы с labels — если ключ не найден в labels, значение дополняется нулём.
*   Цвета наборов данных генерируются автоматически через `getColor()` если не заданы в `$options` — используется "золотой угол" для максимального разнообразия.
*   Метод `autolabels()` полезен для числовых осей — генерирует последовательность с заданным шагом (например, от 0 до 100 с шагом 10).
*   Метод `compile()` генерирует 5 клиентских методов через `compilemethod()` — все они доступны через `LTS.get('{id}').chart`.
*   При компиляции автоматически добавляется вызов `render()` — график отрисовывается сразу после загрузки DOM.
*   Метод `update()` использует `Object.assign()` для слияния данных — не заменяет полностью конфигурацию, а обновляет указанные поля.
*   Метод `destroy()` удаляет canvas из DOM — после вызова график нельзя восстановить без пересоздания объекта.
*   Легенда рисуется для всех типов диаграмм — автоматически позиционируется в верхней части canvas с переносом строк.
*   Для линейных и столбчатых диаграмм автоматически вычисляются масштабы осей X и Y — макс. значение округляется вверх до 10.
*   Для круговых диаграмм толщина колец пропорциональна сумме данных в каждом dataset — визуализирует вклад каждого набора.
*   Для составных круговых диаграмм один сектор соответствует одной метке — внутри сектора части пропорциональны значениям datasets.
*   Метод `getColor()` генерирует цвета через HSL — обеспечивает равномерное распределение по цветовому кругу.
*   Конвертация HSL в RGBA через `hslToRgba()` — поддерживает стандартный алгоритм с промежуточными p, q значениями.
*   Canvas очищается перед каждой перерисовкой через `clearRect()` — предотвращает наложение графиков.
*   Padding по умолчанию 50px — обеспечивает место для осей, меток и легенды.
*   Для столбчатых диаграмм ширина группы вычисляется как 80% от доступной ширины — 20% остаётся для отступов.
*   Для линейных графиков точки рисуются радиусом 4px — заполняются цветом линии.
*   Границы между секторами круговых диаграмм рисуются белыми линиями толщиной 2px — для чёткости.
*   Границы между секторами составных диаграмм рисуются чёрными линиями толщиной 1px — для разделения частей.
*   Легенда переносится на новую строку если `x > width - 100` — сдвигается на 30px вниз через `translate()`.
*   Метод `render()` проверяет наличие графика в `charts[id]` — предотвращает ошибки при несуществующем ID.
*   При создании графика canvas должен существовать в DOM — иначе выводится ошибка в консоль.
*   Контекст 2D получается через `getContext('2d')` — требуется поддержка HTML5 Canvas.
*   Метод `setType()` позволяет динамически менять тип диаграммы — полезно для переключения между видами визуализации.
*   Метод `addDataset()` добавляет набор в конец массива — порядок влияет на отображение в легенде.
*   Для составных диаграмм общая сумма вычисляется по каждой метке — сектор пропорционален вкладу метки в общую сумму.
*   Метод `drawPie()` вычисляет границы секторов до отрисовки — затем рисует белые линии для чёткости.
*   Для кольцевых диаграмм внутренний радиус вычисляется как `radius - totalRadius` — создаёт эффект кольца.
*   Метод `drawLegend()` использует шрифт 14px sans-serif — может быть переопределён через CSS.
*   Цвета легенды берутся из `dataset.borderColor` или генерируются через `getColor()` — обеспечивает соответствие.
*   Для линейных графиков пустые значения обрабатываются как 0 — могут быть изменены в `drawLine()`.
*   Для столбчатых диаграмм несколько datasets отображаются рядом в группе — не накладываются друг на друга.
*   Метод `update()` не проверяет валидность данных — ответственность на разработчике.
*   При изменении размера canvas график не перерисовывается автоматически — требуется явный вызов `render()`.
*   Для адаптивных графиков рекомендуется использовать CSS для изменения размера canvas и вызывать `render()` при resize.
*   Метод `destroy()` снимает график с учёта — после вызова `charts[id]` не существует.
*   Для работы с большими наборами данных рекомендуется ограничивать количество labels — для производительности рендеринга.
*   Метод `dataset()` с `$options` позволяет переопределить цвета — `borderColor`, `backgroundColor`, `borderWidth`.
*   Для прозрачности фона используется `backgroundColor` с альфа-каналом — генерируется через `hslToRgba()` с `a = 0.7`.
*   Метод `autolabels()` поддерживает отрицательный шаг — для убывающих последовательностей.
*   Для числовых меток используется `toString()` — преобразует числа в строки для отображения.
*   Метод `labels()` использует `array_values()` — нормализует ключи массива для последовательного доступа.
*   При компиляции конфигурация кодируется с `JSON_UNESCAPED_UNICODE` — кириллица в метках сохраняется без экранирования.
*   Объекты SimpleChart автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   Для отладки можно использовать `console.log(ltsSimpleChart.charts[id])` — просматривать конфигурацию графика в браузере.
*   Класс не реализует анимации — для анимированных переходов нужно использовать кастомную логику в `update()`.
*   Для экспорта графика можно использовать `canvas.toDataURL()` — получает изображение в формате PNG/JPEG.
*   Метод `render()` вызывается автоматически при создании — не требует явного вызова для начальной отрисовки.
*   Для обновления данных рекомендуется использовать `update()` вместо пересоздания графика — сохраняет конфигурацию.
*   Метод `setType()` полезен для переключения между видами — например, с line на bar для сравнения.
*   Для множественных графиков на странице — каждый имеет независимое хранилище в `ltsSimpleChart.charts`.
*   Метод `getColor()` использует индекс для генерации — одинаковые индексы дают одинаковые цвета.
*   Для кастомизации цветов рекомендуется передавать `$options` в `dataset()` — переопределяет автоматическую генерацию.
*   Метод `drawLegend()` рисует легенду после данных — обеспечивает отображение поверх графика.
*   Для скрытия легенды нужно переопределить `drawLegend()` в кастомной версии `SimpleChart.js`.
*   Метод `render()` проверяет тип диаграммы — для неизвестных типов выводит предупреждение в консоль.
*   Для составных диаграмм нулевые значения пропускаются — не рисуются сектора с `value === 0`.
*   Метод `drawStackedPie()` рисует границы между частями сектора — белые линии толщиной 1px.
*   Для круговых диаграмм общий радиус вычисляется как `Math.min(width, height) / 2 - padding` — вписывается в canvas.
*   Метод `drawBars()` использует `index` и `total` для позиционирования — поддерживает множественные datasets в группе.
*   Для линейных графиков масштаб Y вычисляется через `Math.ceil(maxVal / 10) * 10` — округление вверх до 10.
*   Метод `drawLine()` рисует точки после линии — точки отображаются поверх линии.
*   Для столбчатых диаграмм высота вычисляется через `val * yScale` — пропорционально значению.
*   Метод `drawPie()` использует `startAngle` для отслеживания позиции — последовательно рисует сектора.
*   Для кольцевых диаграмм толщина кольца пропорциональна сумме dataset — визуализирует вклад.
*   Метод `hslToRgba()` использует стандартный алгоритм — совместим с CSS HSL функциями.
*   Для генерации цветов используется "золотой угол" 137.508° — обеспечивает максимальное разнообразие.
*   Метод `getColor()` варьирует насыщенность 70-85% — через `(index % 4) * 5`.
*   Для светлоты используется диапазон 55-75% — через `Math.floor(index / 3) % 3 * 10`.
*   Метод `hue2rgb()` обрабатывает все диапазоны t — обеспечивает корректную конвертацию.
*   Для альфа-канала по умолчанию 0.7 — обеспечивает полупрозрачность для наложения.
*   Метод `update()` мутирует исходную конфигурацию — не создаёт копию через `Object.assign()`.
*   Для добавления dataset без перерисовки нужно вызвать `addDataset()` без последующего `render()`.
*   Метод `destroy()` удаляет canvas из DOM — требует наличия `parentNode`.
*   Для восстановления после `destroy()` нужно пересоздать объект SimpleChart через PHP.
