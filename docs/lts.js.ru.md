# Клиентское ядро (lts.js)

## Назначение
Основной JavaScript-модуль фреймворка LOTIS, автоматически подключаемый на всех страницах приложения. Обеспечивает клиентскую часть объектной модели LOTIS, реализуя взаимодействие между браузером и PHP-бэкендом. Отвечает за управление жизненным циклом клиентских объектов, передачу данных (AJAX/Fetch), синхронизацию глобального состояния, систему сигналов (Pub/Sub) и локализацию интерфейса.

## Свойства
*   **objects** — Хранилище клиентских представлений объектов (LTS-объектов) по их уникальным ID.
*   **varsstorage** — Локальное хранилище глобальных переменных, синхронизируемых с сессией сервера.
*   **requests** — Реестр callback-функций для обработки успешных ответов сервера.
*   **errors** — Реестр callback-функций для обработки ошибок запросов.
*   **busi** — Флаги блокировки повторной отправки идентичных запросов.
*   **signals** — Реестр подписчиков для системы сигналов (событийная шина).
*   **async** — Флаг режима асинхронности запросов по умолчанию.
*   **isonerequest** — Флаг ограничения на выполнение только одного активного запроса одновременно.
*   **languages** — Словари для клиентской локализации текстов.
*   **isfetch** — Флаг поддержки браузером API `fetch`.
*   **preloaderopen / preloaderclose** — Ссылки на функции управления индикатором загрузки.

## Методы
*   **get(id)** — Возвращает LTS-объект - представление объекта с его клиентскими свойствами и методами из хранилища по ID. Если объект отсутствует, создаёт его заглушку с методами-хуками (`check`, `before`, `on`), доступом к jQuery-обёртке, подпиской на события и сигналами.
*   **vars(name, values)** — Управление глобальными переменными. Позволяет получать и устанавливать значения, сохраняемые в рамках сессии.
*   **signal(name, data)** — Инициирует сигнал с указанным именем и передаёт данные всем подписчикам.
*   **onSignal(name, obj, handler)** — Подписывает объект и обработчик на прослушивание указанного сигнала.
*   **post(name, script, args)** — Отправляет POST-запрос на сервер. Обрабатывает блокировку дублей, управление прелоадером, сериализацию данных (JSON или FormData) и маршрутизацию ответа к соответствующим обработчикам `requests` или `errors`. Поддерживает синхронный и асинхронный режимы (через `fetch` или `XMLHttpRequest`).
*   **asyncpost(name, script, args)** — Асинхронная версия метода `post`, использующая `await/fetch`.
*   **page(url)** — Выполняет навигацию на новую страницу через механизм POST-запроса с передачей текущего состояния глобальных переменных.
*   **lang(lng, words)** — Регистрирует словарь переводов для указанного языка.
*   **say(word, key, lng)** — Возвращает локализованную строку. Если перевод не найден, возвращает исходное слово.
*   **request(name, func)** — Регистрирует функцию обработчика успешного ответа для запроса с указанным именем.
*   **error(name, func)** — Регистрирует функцию обработчика ошибки для запроса с указанным именем.
*   **inic() —** Инициализирует базовые обработчики (по умолчанию настроена реакция на ответ `page` для редиректа). Вызывается автоматически при загрузке скрипта.
