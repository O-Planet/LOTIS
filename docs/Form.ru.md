# Класс Form

## Назначение
Класс представления форм ввода данных фреймворка LOTIS, наследующий `Element`. Предназначен для создания интерактивных форм с поддержкой различных типов полей ввода, обработки событий отправки, валидации и синхронизации данных между сервером и клиентом. Класс автоматически создаёт объект `Events` для сервер-клиент коммуникации, предоставляет методы для генерации полей по конфигурации, организации полей в сетку (Cells/Grid) и управления значениями формы через клиентские методы. Интегрируется с `Input`, `LookupField`, `DataTable` для создания сложных форм ввода.

### Основные возможности

* **Создание полей** различных типов
* **Управление значениями** полей
* **Обработка событий** отправки
* **Валидация данных**
* **Организация структуры** формы

### Типы полей

Класс поддерживает создание следующих типов полей:

* **Текстовые поля** (text, email, tel, url)
* **Числовые поля** (number, range)
* **Дата и время** (date, time)
* **Специальные типы** (color, file, hidden)
* **Кнопки** (submit, reset, button)
* **Флажки и радио** (checkbox, radio)
* **Списки** (select)
* **Текстовые области** (textarea)
* **Поля поиска с подкачкой из базы данных** (LookupField)

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$events** | `Events` | `public` | Объект событий для сервер-клиент коммуникации. Создаётся в конструкторе с ID `"{$this->id}_events"`. Используется для обработки нажатий кнопок и отправки данных формы. |

### Наследуемые свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор формы, используется как HTML-атрибут `id` и `name`. |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$tagname** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'form'` в конструкторе. Определяет HTML-тег элемента при рендеринге. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки-поля), `'element'` (DOM-элемент), `'attr'` (HTML-атрибуты), `'clicks'` (обработчики кнопок). |

## Методы (серверная часть, PHP)

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`, создаёт объект `$events` с ID `"{$this->id}_events"`, устанавливает `tagname` в `'form'`. Если `$id` не пуст — устанавливает атрибут `name` формы. |

### Методы работы с формой

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **generate** | `mixed $objects` | `$this` | Генерирует поля формы из массива конфигураций. Принимает массив или JSON-строку с описаниями полей (`name`, `caption`, `type`, `value`, `values`, `id`, `readonly` и др.). Для каждого элемента создаёт соответствующий `Input` или `LookupField`. Поддерживает fluent interface. |
| **cells** | `string $names = ''`, `int $captionscol = 1`, `int $inputscol = 2` | `Cells` | Организует поля формы в табличную структуру через объект `Cells`. Принимает список имён полей, номер колонки для подписей и номер колонки для полей ввода. Возвращает объект `Cells` для дальнейшей настройки. |
| **grid** | `array $regions`, `Grid $grid = null` | `Grid` | Размещает поля формы в сетке `Grid` по областям. Принимает массив `{areaName => [fieldNames]}`. Если `$grid` не передан — создаёт новый. Возвращает объект `Grid` с формой внутри. |

### Работа с событиями

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **getEvent** | — | `Events` | Возвращает объект событий формы. Используется для прямой регистрации обработчиков через `$form->getEvent()->server()`. |
| **eventname** | `mixed $button` | `string` | Возвращает имя события для кнопки. Принимает ID (строку) или объект кнопки, извлекает `$button->id`. Используется внутри `event()`. |
| **event** | `mixed $button`, `callable/string/false $func` | `$this` | Регистрирует обработчик нажатия кнопки. Принимает кнопку (ID или объект) и функцию/JS-код. Если `$func === false/null` — отключает событие. Если строка — добавляет JS-код. Если callable — добавляет серверный обработчик. Устанавливает флаг в секции `'clicks'`. Поддерживает fluent interface. |
| **checkevent** | `mixed $button`, `callable/string $func` | `$this` | Регистрирует хук-валидатор перед выполнением события. Принимает кнопку и функцию/строку. Если строка — добавляет клиентский код, который должен вернуть `true` для продолжения или `false` для отмены. Если callable — серверная функция, возвращающая `true` или сообщение об ошибке. Поддерживает fluent interface. |
| **beforeevent** | `mixed $button`, `callable/string $func` | `$this` | Регистрирует хук предобработки данных перед отправкой. Принимает кнопку и функцию/строку. Может модифицировать аргументы перед отправкой на сервер. Если возвращает `FormData` или объект — данные заменяются. Поддерживает fluent interface. |
| **onevent** | `mixed $button`, `callable/string $func` | `$this` | Регистрирует хук постобработки после выполнения события. Принимает кнопку и функцию/строку. Получает результат выполнения основного обработчика для дополнительной логики. Поддерживает fluent interface. |
| **error** | `mixed $button`, `string $body` | `$this` | Регистрирует обработчик ошибки для кнопки. Принимает кнопку (ID или объект) и JS-код обработчика. Вызывает `$events->error()`. Поддерживает fluent interface. |


### Жизненный цикл

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **compile** | — | — | Генерация клиентских JavaScript-методов. Если есть обработчики кнопок в `'clicks'` — генерирует код для каждого: создаёт метод `value(name, val)`, `values(vals)`, `data()`, `clear()`. Для кнопок с обработчиками — добавляет click-обработчик с отправкой данных через `LTS.get(formId).data()`. Вызывает `parent::compile()`. |

## Методы (клиентская часть, генерируются при compile)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(form).value(name, val)** | `string name`, `mixed val` | `mixed` | Получает или устанавливает значение поля по имени. Если `val` не передан — возвращает текущее значение. Для чекбоксов возвращает `1`/`0`, для радио — значение выбранной кнопки, для `LookupField` — ID выбранной записи. |
| **LTS(form).values(vals)** | `object vals` | `object` | Получает или устанавливает значения всех полей формы. Если `vals` передан — устанавливает значения и возвращает `this`. Иначе — возвращает объект со всеми значениями полей. Для `LookupField` использует `fieldmap` для маппинга полей. |
| **LTS(form).data()** | — | `FormData` | Возвращает данные формы в формате `FormData`. Используется для отправки файлов через `Events`. Включает все поля кроме `button`, `submit`, `reset`. Для чекбоксов добавляет `1`/`0`, для радио — только выбранное значение, для `file` — файл из `input.files[0]`. |
| **LTS(form).clear()** | — | `object` | Очищает все поля формы. Вызывает `values({})` — устанавливает все значения в пустые строки, чекбоксы в `false`. |

## Метод normalizeInputValue (клиентская часть, JS)

| Тип поля | Формат ввода | Формат вывода | Описание |
|----------|--------------|---------------|----------|
| **date** | `YYYY-MM-DD` | `YYYY-MM-DD` | Проверяет валидность даты, возвращает `null` если некорректно. |
| **time** | `HH:MM` или `HH:MM:SS` | `HH:MM` или `HH:MM:SS` | Проверяет диапазон часов (0-23), минут (0-59), секунд (0-59). |
| **datetime-local** | `YYYY-MM-DDTHH:MM` | `YYYY-MM-DDTHH:MM` | Проверяет валидность даты и времени. |
| **month** | `YYYY-MM` | `YYYY-MM` | Проверяет диапазон месяцев (1-12). |
| **week** | `YYYY-Www` | `YYYY-Www` | Проверяет диапазон недель (1-53). |
| **number/range** | Число | Строка числа | Парсит через `parseFloat()`, возвращает `null` если `NaN`. |
| **url** | URL с протоколом | Полный URL | Если нет протокола — добавляет `http://`. Возвращает `null` если невалидно. |
| **email** | email@domain.tld | Строка email | Простая проверка на `@` и точку в домене. |
| **color** | `#RRGGBB` | `#rrggbb` | Проверяет формат HEX, приводит к нижнему регистру. |
| **text/password/search/tel/hidden** | Любая строка | Строка | Возвращает как есть после `trim()`. |

## Примеры использования

```php
// Создание формы
$myform = LTS::Form();

// Добавление полей
$myform->text('username', 'Логин')
     ->password('password', 'Пароль')
     ->email('email', 'Email');

// Добавление кнопок
$myform->button('close', 'Закрыть')->click('$(myform).hide()');
$clickbutton = $myform->button('clickme', 'Нажми меня');

// Привязка события к кнопке
$myform->event($clickbutton,
<<<JS
    alert(result);
JS
  )
  ->event($clickbutton, function($values) {
    return 'Привет, ' . $args['username'];
});

// Назначение хаков
$myform->checkevent($clickbutton,
<<<JS
    function (args) {
        if(args.get('username') == '' || args.get('password') == '') {
          alert('Логин и пароль не должны быть пустыми!');
          return false;
        }
        return true;
    };
JS
);

$myform->beforeevent($clickbutton, function($args) {
    // Модификация данных перед отправкой
    $args['username'] = strtoupper($args['username']);
    return $args;
});

// Организация полей в сетку Cells
$myform->cells([
    'username',
    'password',
    'email',
    'clickme, close'
]);
```

Можно генерировать форму из массива описания:

```php
// Создание формы
$myform = LTS::Form();
// Массив описаний полей формы 
$inputs = [
        ['name' => 'works', 'type' => 'table', 'dbtable' => $works, 'caption' => 'Работы'],
        ['name' => 'quantity', 'type' => 'numeric', 'caption' => 'Количество'],
        ['name' => 'ltsDataId', 'type' => 'hidden'],
        ['name' => 'save', 'type' => 'button', 'caption' => 'Ок'],
        ['name' => 'close', 'type' => 'button', 'caption' => 'Отмена']
    ];

// Генерация формы
$myform->generate($inputs);

// Определения для LookupField
$worksfield = $myform->field('works');
$worksfield->head(['name' => 'Наименование'])
    ->fieldmap(['id' => 'works', 'name' => 'name']);

// Обработчик нажатия на кнопку
$myforms->field('close')->click(
<<<JS
     $(myform).hide();
JS
);

// Привязка события к кнопке
$myform->event('save',
<<<JS
     if(result.result)
          alert(result.data);
     else
          alert('Не удалось сохранить данные!');
JS
  )
  ->event('save', function($values) {
     // Код, сохраняющий данные
     return { 'result' => true, 'data' => 'Данные сохранены'};
});

// Хуки для кнопки save
$myform->checkevent('save', function($args) {
    if(empty($args['quantity']) || $args['quantity'] <= 0) {
        return 'Количество должно быть больше нуля';
    }
    return true;
});

$myform->beforeevent('save', function($args) {
    $args['quantity'] = (int)$args['quantity'];
    $args['total'] = $args['quantity'] * $args['price'];
    return $args;
});

$myform->onevent('save',
<<<JS
function (result) {
    if(result.result) {
        LTS(myform).signal('DataSaved');
    }
    return result;
}
JS
);
```

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   Объект `$events` создаётся автоматически в конструкторе — не требует явного создания для обработки кнопок.
*   Метод `generate()` поддерживает JSON-строку или массив конфигураций — удобно для динамической генерации форм из метаданных.
*   Фиксированные ключи конфигурации: `id`, `type`, `name`, `caption`, `values`, `value`, `min`, `max`, `class`, `dbtable`.
*   Для полей с `class` в конфигурации — вызывается `addclass()` после создания.
*   Метод `cells()` возвращает объект `Cells` — позволяет дополнительно настроить сетку полей перед рендерингом.
*   Метод `cells()` создаёт строки с классами `Row_{inputnames}` — для стилизации через CSS.
*   При организации в `Cells` label и input размещаются в разных колонках — контролируется через `$captionscol` и `$inputscol`.
*   Метод `grid()` добавляет форму в `Grid` после заполнения областей — `$grid->add($this)` — полезно для сложных макетов с несколькими областями.
*   Метод `field()` ищет поля по атрибуту `name` — не по `$id` объекта `Input`. Он возвращает `false` если поле не найдено — требуется проверка перед использованием.
*   Для `LookupField` метод `field()` проверяет `$child->input->attr('name')` — учитывает вложенную структуру.
*   Метод `event()` поддерживает три режима: `false/null` (отключить), строка (JS-код), callable (серверная функция).
*   Обработчики кнопок сохраняются в секции `'clicks'` — используются при компиляции для генерации JS-кода.
*   Метод `compile()` генерирует 4 клиентских метода: `value()`, `values()`, `data()`, `clear()` — доступны через `LTS.get('{id}')` или `LTS(var)`.
*   Для кнопок `submit`/`reset` метод `compile()` генерирует `click()` метод — позволяет назначать обработчики через Events.
*   Метод `data()` возвращает `FormData` — требуется для отправки файлов через `Events`.
*   Метод `values()` поддерживает маппинг полей для `LookupField` через `$lookup->fieldmap`.
*   Для чекбоксов `values()` возвращает `1`/`0` — не `true`/`false`.
*   Для радио-кнопок `values()` возвращает значение выбранной кнопки — невыбранные игнорируются.
*   Для `LookupField` `values()` устанавливает `selected` объект и значение в input через `normalizeInputValue()`.
*   Метод `normalizeInputValue()` поддерживает 10+ типов HTML5-полей — обеспечивает корректное форматирование значений.
*   При установке значения через `value()` для `date`/`time` — проверяется валидность формата.
*   Для `url` без протокола — автоматически добавляется `http://`.
*   Для `color` — значение приводится к нижнему регистру (`#aabbcc`).
*   Метод `clear()` устанавливает все поля в пустые значения — не удаляет поля из формы.
*   Кнопки `submit` и `reset` имеют стандартное поведение браузера — могут быть переопределены через `event()`.
*   Для кнопок с обработчиками — стандартное поведение отменяется, данные отправляются через `Events`.
*   Метод `error()` регистрирует обработчики ошибок через `$events->error()` — выполняются на клиенте при неудачном ответе.
*   При компиляции для кнопок генерируется click-обработчик — собирает данные через `LTS.get(formId).data()` и отправляет через `Events`.
*   Форма автоматически получает атрибут `name` равный `$id` — требуется для идентификации на клиенте.
*   Для `radio` с `$values` — создаётся несколько `Input` с одинаковым `name` — браузер обеспечивает взаимное исключение.
*   Все методы создания полей возвращают объект поля — позволяет цепочечную настройку через fluent interface.
*   Метод `generate()` пропускает элементы без ключа `name` — требует явного указания имени поля.
*   Для полей с `readonly` в конфигурации — устанавливается атрибут `readonly='1'`.
*   Метод `cells()` собирает поля из `$this->get('childs')` — должны быть созданы до вызова `cells()`.
*   Метод `grid()` ищет поля через `getchild()` — поддерживает поиск по `tagname` и функции-предикату.
*   При размещении в `Grid` форма становится потомком области — `$this->owner` устанавливается в ID области.
*   Метод `getEvent()` возвращает `$events` — позволяет прямую регистрацию через `$form->getEvent()->server()`.
*   Обработчики событий выполняются в контексте формы — `$args` содержат данные всех полей.
*   Для файловых полей данные передаются через `FormData` — не через JSON.
*   Метод `values()` для `LookupField` использует `fieldmap` — сопоставляет имена полей таблицы с полями формы.
*   При очистке через `clear()` для `LookupField` — устанавливается `selected = null` и input очищается.
*   Для `datetime-local` поддерживаются форматы с `T` и пробелом между датой и временем.
*   Для `week` формат `YYYY-Www` — неделя от 01 до 53.
*   Для `month` формат `YYYY-MM` — месяц от 01 до 12.
*   Для радио-кнопок `value()` устанавливает `checked` на кнопку с matching значением.
*   Для `file` полей `data()` добавляет первый файл из `input.files[0]`.
*   При отправке через `Events` данные автоматически сериализуются — массивы и объекты в JSON с префиксом `<JSON>`.
*   Метод `eventname()` извлекает ID из объекта или строки — универсальный интерфейс.
*   При отключении события через `event($button, false)` — обработчик удаляется из `'clicks'`.
*   **Хуки событий:** Методы `checkevent()`, `beforeevent()`, `onevent()` регистрируют обработчики жизненного цикла события: валидация → предобработка → выполнение → постобработка.
*   **Порядок выполнения:** При нажатии кнопки: `checkevent` → `beforeevent` → `event` → `onevent`. Если `checkevent` возвращает `false` — выполнение прерывается.
*   **Модификация данных:** `beforeevent` может возвращать `FormData` или объект для замены аргументов перед отправкой на сервер.
*   **Клиентские хуки:** Если передана строка — код выполняется на клиенте, должен быть в формате `function(args) { ... }` для `checkevent`/`beforeevent` или `function(result) { ... }` для `onevent`.
*   **Серверные хуки:** Если передана callable-функция — выполняется на сервере с параметрами `($args)` для `checkevent`/`beforeevent` или `($args, $result)` для `onevent`.
*   **Возврат из checkevent:** Должен возвращать `true` для продолжения или `false`/строку для отмены с сообщением об ошибке.
*   **Возврат из beforeevent:** Может возвращать модифицированные `$args` (массив, объект или `FormData`) или ничего для использования исходных данных.
*   **Возврат из onevent:** Получает результат основного обработчика — может модифицировать или логировать, возвращаемое значение передаётся клиенту.
