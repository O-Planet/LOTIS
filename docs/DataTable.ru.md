# Класс DataTable

## Назначение
Класс представления интерактивных таблиц данных фреймворка LOTIS, наследующий `Element`. Предназначен для создания динамических таблиц с поддержкой сортировки, фильтрации, редактирования ячеек, добавления/удаления строк и синхронизации данных между сервером и клиентом. Состоит из серверной части (DataTable.php) для определения структуры и генерации метаданных, и клиентской части (DataTable.js) для управления таблицей в браузере. Класс автоматически генерирует JavaScript-методы для работы с таблицей через `LTS.get('{id}')`, поддерживает скрытые поля, кастомизацию отображения через хуки и интеграцию с формами редактирования.

## Свойства (серверная часть, PHP)

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$head** | `array` | `public` | Ассоциативный массив заголовков колонок. Ключи — имена полей, значения — отображаемые названия. Устанавливается через `head()`. |
| **$rows** | `array` | `public` | Массив данных строк таблицы. Каждый элемент — ассоциативный массив или объект с данными строки. Устанавливается через `data()` или `append()`. |
| **$hidden** | `array` | `public` | Массив имён скрытых полей. Эти поля не отображаются в таблице, но сохраняются в data-атрибутах строк для доступа на клиенте. Устанавливается через `hidden()`. |
| **$sorted** | `mixed` | `public` | Поля для сортировки. Может быть строкой (имя поля) или массивом строк. Устанавливается через `sort()`. |
| **$norows** | `bool` | `public` | Флаг отложенного рендеринга строк. Если `true` — таблица создаётся без строк при загрузке, данные добавляются позже через клиентские методы. По умолчанию `true`. |
| **$out** | `string` | `public` | JavaScript-код хука для кастомизации строки. Вызывается при создании каждой строки таблицы. Устанавливается через `out()`. |
| **$classname** | `string` | `public` | Наследуется от `Element`. Автоматически устанавливается в `'data-table'` в конструкторе. |
| **$tagname** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'table'` в конструкторе. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор таблицы, используется для связи с клиентским объектом в `ltsDataTable`. |

## Методы (серверная часть, PHP)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`, устанавливает `classname` в `'data-table'`, `tagname` в `'table'`, `norows` в `true`, `out` в пустую строку. |
| **hidden** | `string $hidden` | `$this` | Устанавливает список скрытых полей. Принимает строку с именами полей через запятую. Разбивает через `\LTS::explodestr()`, сохраняет в `$hidden`. Поля будут сохранены в data-атрибутах строк но не отображаться в ячейках. Поддерживает fluent interface. |
| **sort** | `string $name` | `$this` | Устанавливает поля для сортировки. Принимает имя поля или массив имён. Сохраняет в `$sorted`. При компиляции генерирует JS-код для автоматической сортировки при загрузке. Поддерживает fluent interface. |
| **head** | `array $head` | `$this` | Устанавливает заголовки колонок. Принимает ассоциативный массив (ключ — имя поля, значение — заголовок). Сохраняет в `$head`. Используется при генерации `<thead>`. Поддерживает fluent interface. |
| **data** | `array $rows` | `$this` | Устанавливает данные строк таблицы. Принимает массив массивов или объектов. Сохраняет в `$rows`. При компиляции кодируется в JSON и передаётся клиенту. Поддерживает fluent interface. |
| **out** | `string $func` | `$this` | Устанавливает JavaScript-хук для кастомизации строки. Если код не начинается с `function` — оборачивает в `function (row, data) { ... }`. Хук вызывается при создании каждой строки, позволяет модифицировать DOM или добавлять обработчики. Поддерживает fluent interface. |
| **append** | `array $values` | `$this` | Добавляет одну строку в массив `$rows`. Принимает ассоциативный массив данных строки. Используется для динамического добавления данных на сервере до рендеринга. Поддерживает fluent interface. |
| **add** | `mixed $name`, `mixed $value = null` | `$this` | Переопределённый метод добавления данных. Поддерживает три режима: (1) Если оба параметра — массивы с числовыми ключами — первый как `$head`, второй как `$rows`. (2) Если один массив с числовыми ключами — как `$rows`. (3) Иначе — вызывает родительский `Element::add()`. Определяет тип массива через `isArrayOfArraysWithNumericKeys()`. Поддерживает fluent interface. |
| **isArrayOfArraysWithNumericKeys** | `array $array` | `bool` | Статический вспомогательный метод. Проверяет, является ли массив массивом массивов/объектов с числовыми ключами. Используется в `add()` для определения режима добавления данных. |
| **rowclick** | `Form $form` | `$this` | Привязывает открытие формы редактирования к клику по строке. Генерирует клиентский метод `rowclick(row)` который: (1) Извлекает `ltsDataId` из строки. (2) При клике на колонку выбора — переключает флаг `sel`. (3) При клике на другие колонки — загружает данные строки в форму через `LTS({form}).values()`, сохраняет `ltsDataId` в атрибут формы, отправляет сигнал `{id}_RowClick`. Поддерживает хук `loadrowclick` для предобработки данных. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Кодирует `$head`, `$rows`, `$hidden` в JSON. Вызывает `compilemethod()` для создания методов: `ltsDataTable.add()`, `head()`, `attr()`, `rows()`, `Row()`, `findRows()`, `sort()`, `create()`, `values()`, `clear()`, `append()`, `filter()`, `fieldvalues()`, `out()`. Если установлен `$sorted` — добавляет JS-код сортировки в область `'ready'`. Если установлен `$out` — добавляет хук. В конце вызывает `ltsDataTable.create()` для рендеринга таблицы. Вызывает `parent::compile()`. |

## Объект ltsDataTable (клиентская часть, JS)

| Свойство | Тип | Описание |
|----------|-----|----------|
| **rows** | `object` | Хранилище данных строк по ID таблицы. `rows[id]` — массив объектов строк. |
| **heads** | `object` | Хранилище заголовков колонок по ID таблицы. `heads[id]` — ассоциативный массив. |
| **hidden** | `object` | Хранилище скрытых полей по ID таблицы. `hidden[id]` — массив имён полей. |
| **sorted** | `object` | Хранилище настроек сортировки по ID таблицы. `sorted[id]` — массив полей с типами. |
| **hucks** | `object` | Хранилище хуков для кастомизации строк. `hucks[id]` — функция. |
| **dataids** | `object` | Счётчики ID строк по таблицам. `dataids[id]` — последний присвоенный `ltsDataId`. |

## Методы (клиентская часть, JS)

Эти методы становятся доступны на клиенте через `LTS.get('{id}')` после компиляции. Фактическая логика выполняется объектом `ltsDataTable`.

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(table).head(data)** | `array data` | `array` | Устанавливает или получает заголовки колонок. Если `data` передан — обновляет `ltsDataTable.heads[id]`. Иначе — возвращает текущие заголовки. |
| **LTS(table).attr(data)** | `array data` | `array` | Устанавливает или получает список скрытых полей. Если `data` передан — обновляет `ltsDataTable.hidden[id]`. Иначе — возвращает текущий список. |
| **LTS(table).rows(data)** | `array data` | `array` | Устанавливает или получает данные всех строк. Если `data` передан — создаёт глубокую копию через `ltsDataTable.deepCopyArray()` и сохраняет в `ltsDataTable.rows[id]`. Иначе — возвращает текущие данные. |
| **LTS(table).Row(id)** | `mixed id` | `jQuery` | Возвращает jQuery-объект строки по ID. `id` может быть числом (`ltsDataId`) или объектом строки (извлекает `ltsDataId`). Селектор: `#{id} tbody tr[data-id="{id}"]`. |
| **LTS(table).findRows(attr)** | `mixed attr` | `jQuery` | Находит строки по фильтру. Поддерживает: (1) HTML-элемент `tr` — возвращает одну строку. (2) jQuery-объект — фильтрует по `data-id`. (3) Массив объектов — фильтрует по `ltsDataId`. (4) Объект `{field: value}` — фильтрует по значениям полей. Возвращает jQuery-объект найденных строк. |
| **LTS(table).sort(name, napr)** | `mixed name`, `int napr` | — | Сортирует таблицу. Если `name` — массив — устанавливает настройки сортировки в `ltsDataTable.sorted[id]`. Если строка — сортирует по полю с направлением `napr` (1 = ASC, -1 = DESC). Определяет тип данных (int, float, date, string) из `data-type` атрибута заголовка. Пересортировывает `ltsDataTable.rows[id]` и обновляет DOM через `reorderTbody()`. |
| **LTS(table).create(data)** | `array data` | — | Создаёт таблицу в DOM. Если `data` передан — устанавливает строки. Иначе — очищает `ltsDataTable.rows[id]`. Вызывает `ltsDataTable.create(id)` для генерации `<thead>` и `<tbody>`. |
| **LTS(table).values(attr, values)** | `mixed attr`, `object values` | `array` | Получает или обновляет значения строк. Поддерживает 7 режимов: (1) Без параметров — возвращает все строки. (2) HTML-элемент — фильтрует по `data-id`, обновляет если передан `values`. (3) jQuery-объект — фильтрует по `data-id`, обновляет если передан `values`. (4) Массив объектов — фильтрует по `ltsDataId`, обновляет если передан `values`. (5) Объект `{field: value}` — фильтрует по значениям полей, обновляет если передан `values`. (6) Только `values` — обновляет все строки. (7) Только `attr` — возвращает отфильтрованные строки. При обновлении модифицирует ячейки по классам `.Column_{field}` и скрытые атрибуты. |
| **LTS(table).clear(attr)** | `mixed attr` | — | Удаляет строки из таблицы. Если `attr` не передан — удаляет все строки, очищает `ltsDataTable.rows[id]` и `dataids[id]`. Иначе — фильтрует строки для удаления (HTML-элемент, jQuery, массив объектов, объект-фильтр), удаляет из `ltsDataTable.rows[id]` и из DOM. |
| **LTS(table).append(values)** | `array values` | — | Добавляет новые строки в таблицу. Принимает массив объектов строк. Для каждой строки создаёт новый `ltsDataId`, добавляет в `ltsDataTable.rows[id]`, генерирует HTML `<tr>` и добавляет в `<tbody>`. Вызывает хук `hucks[id]` если установлен. |
| **LTS(table).filter(values)** | `object values` | — | Фильтрует отображение строк. Принимает объект правил фильтрации `{field: value}`. Вызывает `ltsDataTable.filterAndApply()` — скрывает строки не прошедшие фильтр через `display: none`, показывает прошедшие. Пустой фильтр показывает все строки. |
| **LTS(table).fieldvalues(field, selectedrows)** | `string field`, `mixed selectedrows` | `array` | Возвращает уникальные значения поля из выбранных строк. Если `selectedrows` не передан — из всех строк. `selectedrows` может быть: массивом объектов, jQuery-объектом, HTML-элементом. Извлекает значения, удаляет дубликаты через `Set`, возвращает массив. |
| **LTS(table).out(f)** | `function f` | — | Устанавливает хук для кастомизации строк. Сохраняет функцию в `ltsDataTable.hucks[id]`. Хук вызывается при создании каждой строки с параметрами `($row, $data)`. |
| **LTS(table).rowclick(row)** | `HTMLElement row` | — | Обработчик клика по строке (генерируется методом `rowclick()`). Извлекает `ltsDataId`, при клике на колонку выбора — переключает `sel`, при клике на другие колонки — загружает данные в форму, сохраняет `ltsDataId` в атрибут формы, отправляет сигнал `{id}_RowClick`. |

## Глобальные методы ltsDataTable (прямой вызов)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **ltsDataTable.add(id, head, rows, hidden)** | `string id`, `array head`, `array rows`, `array hidden` | — | Регистрирует таблицу в хранилище. Заполняет `heads[id]`, `rows[id]`, `hidden[id]`, `sorted[id]`, `hucks[id]`, `dataids[id]`. Вызывается автоматически при компиляции. |
| **ltsDataTable.create(id, norows)** | `string id`, `bool norows` | — | Создаёт HTML-таблицу в DOM. Генерирует `<thead>` с заголовками, `<tbody>` со строками. Добавляет классы `Column_{fieldname}` к ячейкам, `ltsSorted` к сортируемым заголовкам, `ltsEditable` к редактируемым ячейкам. Вызывает хук `hucks[id]` для каждой строки. |
| **ltsDataTable.append(id, dataObject)** | `string id`, `array dataObject` | — | Добавляет строки в существующую таблицу. Создаёт глубокие копии через `deepCopyArray()`, присваивает `ltsDataId`, добавляет в `rows[id]` и в DOM. |
| **ltsDataTable.lastdataid(id)** | `string id` | `int/false` | Возвращает последний присвоенный `ltsDataId` для таблицы. |
| **ltsDataTable.sort(id, name, direction)** | `string id`, `string name`, `int direction` | — | Сортирует строки таблицы по полю. Определяет тип данных из `data-type` атрибута, парсит значения (int, float, date, string), сортирует массив `rows[id]`, обновляет DOM через `reorderTbody()`. |
| **ltsDataTable.reorderTbody(id)** | `string id` | — | Пересобирает `<tbody>` в соответствии с порядком строк в `rows[id]`. Сохраняет строки в map по `data-id`, очищает `<tbody>`, восстанавливает в новом порядке. |
| **ltsDataTable.deepCopyArray(originalArray)** | `array originalArray` | `array` | Создаёт глубокую копию массива. Использует `structuredClone()` если доступен, иначе `jQuery.extend(true, ...)`, иначе `JSON.parse(JSON.stringify(...))`. |
| **ltsDataTable.filter(id, rules)** | `string id`, `object rules` | `array` | Фильтрует строки по правилам. Возвращает массив строк прошедших фильтр. Поддерживает: пустые строки (всегда true), RegExp, даты, подстроки, числа, строгое сравнение. |
| **ltsDataTable.applyFilter(id, filtered)** | `string id`, `array filtered` | — | Применяет фильтр к DOM. Показывает строки из `filtered` через `show()`, скрывает остальные через `hide()`. |
| **ltsDataTable.filterAndApply(id, rules)** | `string id`, `object rules` | — | Комбинирует фильтрацию и применение. Если фильтр пустой — вызывает `clearFilter()`. Иначе — фильтрует и применяет. |
| **ltsDataTable.clearFilter(id)** | `string id` | — | Сбрасывает фильтр — показывает все строки таблицы. |

## Специальные классы и атрибуты

| Класс/Атрибут | Описание |
|---------------|----------|
| **`.data-table`** | Базовый класс таблицы — добавляется автоматически. |
| **`.Column_{fieldname}`** | Класс ячейки и заголовка — используется для доступа к колонке. |
| **`.ltsSorted`** | Класс сортируемого заголовка — добавляет обработчик клика для сортировки. |
| **`.ltsEditable`** | Класс редактируемой ячейки — добавляет inline-редактирование по клику. |
| **`.ltsRowDelbutton`** | Класс кнопки удаления строки — вызывает `ltsDataTable.clear()` или пользовательский обработчик. |
| **`data-id`** | Атрибут строки — хранит `ltsDataId` для идентификации. |
| **`data-field`** | Атрибут ячейки — хранит имя поля. |
| **`data-type`** | Атрибут заголовка — хранит тип данных для сортировки (int, float, date, string). |
| **`data-sort-order`** | Атрибут заголовка — хранит направление сортировки (1 = ASC, -1 = DESC). |
| **`ltsDataId`** | Свойство строки — уникальный ID строки в клиентском хранилище. |

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   **Зависимость от DataTable.js:** Для корректной работы клиентских методов необходимо подключение файла `DataTable.js`, который содержит реализацию объекта `ltsDataTable`. PHP-класс `DataTable` только генерирует конфигурацию и вызовы методов.
*   Все методы настройки таблицы возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Флаг `$norows` по умолчанию `true` — таблица создаётся без строк, данные добавляются позже через клиентские методы. Это полезно для больших таблиц чтобы не перегружать начальную загрузку.
*   Скрытые поля через `hidden()` сохраняются в data-атрибутах строк — доступны на клиенте через `LTS(table).values()` но не отображаются в ячейках.
*   Метод `rowclick()` автоматически генерирует обработчик клика — при клике на строку данные загружаются в указанную форму, сохраняется `ltsDataId` для последующего обновления.
*   Хук `loadrowclick` может быть определён на клиенте для предобработки данных перед загрузкой в форму — вызывается внутри `rowclick()`.
*   Метод `compile()` генерирует 13 клиентских методов через `compilemethod()` — все они становятся доступны через `LTS.get('{id}')` на клиенте.
*   Сортировка через `sort()` автоматически определяет тип данных из атрибута `data-type` — поддерживает int, float, date, string.
*   Пустые значения при сортировке перемещаются в конец списка независимо от направления.
*   Метод `values()` поддерживает 7 режимов работы — от получения всех строк до фильтрации и обновления по сложным условиям.
*   При обновлении через `values()` модифицируются ячейки по классам `.Column_{fieldname}` — HTML-контент обновляется через `.html()`.
*   Метод `filter()` использует умное сравнение — пустые строки всегда совпадают, даты парсятся в формате `DD.MM.YYYY` или `YYYY-MM-DD`, RegExp поддерживается.
*   Глубокое копирование через `deepCopyArray()` использует `structuredClone()` если доступен — обеспечивает корректное клонирование вложенных объектов.
*   Метод `reorderTbody()` не пересоздаёт строки — только меняет порядок в DOM, что эффективнее для больших таблиц.
*   Обработчики событий для сортировки, редактирования и удаления регистрируются через `$(document).on()` — работают с динамически добавленными строками.
*   При редактировании ячейки с классом `.ltsEditable` — создаётся `<input>` на лету, при `blur` или `Enter` — сохраняется через `ltsDataTable.values()` и отправляется сигнал `{id}_ElementSave`.

*   Метод `fieldvalues()` возвращает уникальные значения — полезно для генерации фильтров и статистики.
*   Класс не реализует пагинацию на сервере — для больших наборов данных нужно использовать фильтрацию и загрузку порциями через Events.
*   При компиляции данные `$head`, `$rows`, `$hidden` кодируются в JSON — сложные типы данных сериализуются автоматически.
*   Объекты DataTable автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   Для кастомизации отображения строк используется хук `out()` — функция вызывается для каждой строки с параметрами `($row, $data)`.
*   Метод `add()` автоматически определяет тип данных — массивы с числовыми ключами считаются строками, с ассоциативными — заголовками.

*   Сортировка по датам поддерживает форматы `DD.MM.YYYY HH:MM:SS` и `YYYY-MM-DD` — парсится через `tryParseDate()`.
*   При удалении строк через `clear()` данные удаляются и из `ltsDataTable.rows[id]` и из DOM — обеспечивается синхронизация хранилища и отображения.
*   Для работы с выбранными строками используется свойство `sel` — переключается при клике на колонку выбора.
*   Класс совместим с DataView — DataTable используется как компонент списка записей в составе DataView.
*   Метод `filterAndApply()` автоматически сбрасывает фильтр при пустых значениях — показывает все строки.
*   Для доступа к строке по `ltsDataId` используется метод `Row()` — возвращает jQuery-объект для манипуляций.
*   При добавлении строк через `append()` автоматически генерируется новый `ltsDataId` — счётчик хранится в `ltsDataTable.dataids[id]`.

*   При сортировке направление сохраняется в атрибуте `data-sort-order` — визуально отображается через CSS.
*   Для работы с файлами и бинарными данными в ячейках рекомендуется использовать скрытые поля через `hidden()`.
*   Класс не реализует экспорта данных — для этого нужно использовать клиентские методы `rows()` и серверную обработку через Events.
*   Метод `valuesCompare()` используется для фильтрации — поддерживает сравнение дат, строк, чисел, RegExp.

*   Для интеграции с системой событий Events можно использовать сигналы `{id}_RowClick`, `{id}_ElementSave`, `{id}_RowsDel`.

*   Метод `sort()` поддерживает множественную сортировку — можно передать массив полей с направлениями.

*   При удалении строк через `clear()` с фильтром — удаляются только строки прошедшие фильтр.
