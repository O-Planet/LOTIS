# Трейт TypeChecker

## Назначение
Вспомогательный трейт фреймворка LOTIS, предоставляющий механизмы логирования ошибок валидации типов и значений аргументов. Используется для обеспечения строгой типизации в методах классов фреймворка. Все ошибки передаются в центральный класс отладки `Debug`, что позволяет унифицировать обработку и вывод сообщений об ошибках. Трейт подключается к классам, требующим валидации входных данных (например, `Construct`), и предоставляет два защищённых метода для регистрации ошибок без прерывания выполнения кода.

## Свойства
Трейт не содержит собственных свойств.

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **typeError** | `string $method`, `string $paramName`, `string $expected`, `string $actual` | — | Логирует ошибку несоответствия типа аргумента. Принимает имя метода, имя параметра, ожидаемый тип и фактический тип переданного значения. Вызывает `Debug::typeError()` для записи ошибки в лог. Используется при валидации типов параметров методов. |
| **valueError** | `string $method`, `string $paramName`, `string $message`, `mixed $value = null` | — | Логирует ошибку недопустимого значения аргумента. Принимает имя метода, имя параметра, сообщение об ошибке и опционально само значение. Вызывает `Debug::valueError()` для записи ошибки в лог. Используется при валидации диапазонов, форматов или бизнес-правил для параметров. |

## Примечания
*   Трейт использует класс `Debug` для фактической записи ошибок — сам не реализует логику логирования.
*   Оба метода объявлены как `protected` — доступны только внутри классов, подключивших трейт.
*   Методы не бросают исключения и не прерывают выполнение кода — только регистрируют ошибку для последующего анализа.
*   Параметр `$value` в `valueError()` опционален — может быть передан для детальной отладки.
*   Трейт подключается через `use TypeChecker;` в теле класса (например, в `Construct`).
*   Статический контекст `static::class` используется для автоматического определения имени класса, вызвавшего метод ошибки.
