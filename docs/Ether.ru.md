# Интерфейс Ether

## Назначение
Базовый интерфейс фреймворка LOTIS, определяющий контракт жизненного цикла для всех объектов системы. Все классы фреймворка реализуют этот интерфейс, что гарантирует единообразие процессов инициализации, компиляции метаданных, рендеринга HTML и регистрации объектов в пространстве имён (Universe). Интерфейс описывает пять ключевых этапов существования объекта: конструирование, предварительную обработку, создание DOM-элемента, добавление в глобальное пространство и рекурсивную обработку потомков.

## Методы

| Метод | Параметры | Описание |
|-------|-----------|----------|
| **__construct** | `string $id = ''` | Конструктор объекта. Принимает уникальный идентификатор элемента. Если ID не передан, система генерирует его автоматически. Инициализирует базовые свойства объекта перед началом жизненного цикла. |
| **compile** | — | Этап предварительной обработки. Выполняет валидацию, подготовку внутренних структур данных и вычисление производных значений перед рендерингом. Вызывается перед созданием HTML-представления. |
| **create** | `Space $space`, `Element $parent = null` | Полный цикл создания объекта. Координирует последовательное выполнение методов `shine()`, `childs()` и `addinspace()`. Принимает объект пространства (Space) для рендеринга и опционально ссылку на родительский элемент иерархии. |
| **shine** | `Element $parent = null` | Непосредственное создание DOM-элемента. Генерирует HTML-код текущего объекта, назначает атрибуты (ID, классы, стили) и возвращает ссылку на созданный элемент. Не обрабатывает дочерние элементы. |
| **addinspace** | — | Регистрация объекта в глобальном пространстве имён (Universe). Добавляет ссылку на объект в реестр Space, делая его доступным для поиска по ID и обеспечивая связь между серверной и клиентской частями приложения. |
| **childs** | — | Рекурсивная обработка потомков. Инициирует создание и регистрацию всех дочерних элементов текущего объекта. Вызывается после `shine()` для соблюдения порядка вложенности в DOM-дереве. |

## Примечания
*   Интерфейс не содержит свойств — они определяются в конкретных реализациях классов.
*   Метод `create()` является фасадом, инкапсулирующим стандартную последовательность: `shine()` → `childs()` → `addinspace()`.
*   Нарушение порядка вызова методов жизненного цикла может привести к некорректной работе связки клиент-сервер.
*   Все классы UI-компонентов (Div, Form, Button и др.) и функциональных объектов (JS, CSS, Events) реализуют данный интерфейс.
