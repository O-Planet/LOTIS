# Интерфейс Ether

## Назначение
Базовый интерфейс фреймворка LOTIS, определяющий контракт жизненного цикла для всех объектов системы. Все классы фреймворка реализуют этот интерфейс, что гарантирует единообразие процессов инициализации, компиляции метаданных, рендеринга HTML и регистрации объектов во вселенной (Space). Интерфейс описывает пять ключевых этапов существования объекта: конструирование, предварительную обработку, создание метаданных, добавление метаданных во вселенную и рекурсивную обработку потомков.

## Жизненный цикл объектов

Все объекты во вселенной LOTIS проходят следующий жизненный цикл:

1. **compile()** — предварительная обработка
2. **shine()** — создание метаданных
3. **addinspace()** — регистрация метаданных во вселенной
4. **childs()** — обработка дочерних объектов

## Методы

| Метод | Параметры | Описание |
|-------|-----------|----------|
| **__construct** | `string $id = ''` | Конструктор объекта. Принимает уникальный идентификатор. Инициализирует базовые свойства объекта перед началом жизненного цикла. |
| **compile** | — | Этап предварительной обработки перед созданием метаданных объекта и поменщением их во вселенную. Выполняет валидацию, подготовку внутренних структур данных и вычисление производных значений перед рендерингом.  |
| **create** | `Space $space`, `Element $parent = null` | Полный цикл создания объекта. Координирует последовательное выполнение методов `shine()`, `childs()` и `addinspace()`. Принимает объект пространства (Space) для рендеринга и опционально ссылку на родительский элемент иерархии. |
| **shine** | `Element $parent = null` | Непосредственное создание метаданных объекта. Не обрабатывает дочерние объекты. |
| **addinspace** | — | Размещение метаданных во вселенной. |
| **childs** | — | Рекурсивная обработка потомков. Инициирует создание и регистрацию всех дочерних элементов текущего объекта. Вызывается после `shine()` для соблюдения порядка вложенности в дереве объектов. |

## Примечания
*   Все классы UI-компонентов (Div, Form, Button и др.) и функциональных объектов (JS, CSS, Events) реализуют данный интерфейс.
*   Метод `create()` является фасадом, инкапсулирующим стандартную последовательность: `shine()` → `childs()` → `addinspace()`.
*   Нарушение порядка вызова методов жизненного цикла может привести к некорректной работе связки клиент-сервер.
