# Класс DataSync

## Назначение
Класс синхронизации данных фреймворка LOTIS, наследующий `Quark`. Предназначен для организации синхронизации данными между клиентом и сервером в реальном времени. Устанавливает связь между табюлицей базы данных `dbtable` и ее отображением в `datatable` пользователя. Поддерживает автоматическую подгрузку записей через `IntersectionObserver`, отслеживание изменений записей между клиентами, блокировку записей на время редактирования и фиксацию изменений в специальной таблице-менеджере. Интегрируется с `DataTable` для отображения данных, `Events` и `Vars` для клиент-серверной коммуникации.

### Режимы использования

* **Подкачка данных в реальном времени** - позволяет добавлять данные из `dbtable` в `datatable` порциями при скроллинге таблицы внутри `wrapper`.
* **Многопользовательский режим** - организует обновление в реальном времени `datatable` пользователя, если кем-то произведены изменения в таблице бахы данных `dbtable`. Позволяет блокировать запись из `dbtable` при редактировании и сообщать остальным пользователям приложения о произведенных изменениях.

Для организации многопользовательского режима работы требуется создание в базе данных таблицы-менеджера, фиксирующего изменения записей. Это разовая операция, которая выполняется при вызове метода `initmanager` объекта `DataSync` с установленным в true вторым парамкетром.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$datatable** | `DataTable/null` | `public` | Ссылка на объект DataTable для отображения данных. Устанавливается через `datatable()`. |
| **$dbtable** | `MySqlTable/null` | `public` | Ссылка на таблицу базы данных. Устанавливается через `dbtable()`. Используется в запросах через `QueryBuilder`. |
| **$autochange** | `bool` | `public` | Флаг разрешения отслеживания изменений. По умолчанию `false`. Управляется через `autochange()`. |
| **$autoupload** | `bool` | `public` | Флаг разрешения автоподгрузки. По умолчанию `false`. Управляется через `autoupload()`. |
| **$limit** | `int` | `public` | Количество записей для одновременной загрузки. По умолчанию `100`. Используется при автоподгрузке через `upload()`. |
| **$conditions** | `array` | `public` | Условия, установленные на сервере. Передаются в `QueryBuilder` через `applycondition()`. Устанавливаются через `condition()`. |
| **$usersconditions** | `mixed/null` | `public` | Условия пользователя для фильтрации и сортировки. Назначаются на клиенте и сохраняются в LTS-объекте. Передаются автоматически при вызове `upload()`. |
| **$system** | `mixed/null` | `public` | Условия внутренних функций, в частности для `LIMIT`. Управляется автоматически при автоподгрузке. |
| **$datarequest** | `callable` | `public` | Функция загрузки данных. По умолчанию — запрос к `$dbtable` с условиями. Переопределяется через `datarequest()`. |
| **$changeregime** | `int` | `public` | Режим обновления изменённых записей: `1` — автоматическое, `0` — не обновлять. Требуется для ручной обработки полученных изменений на клиенте. Пришедшие измененные записи можно получить в `LTS(datasync).changet`, а удаленные записи - в `LTS(datasync).deleted`. Состав полей записей определяется установленными условиями на сервере `conditions`, на клиенте `usersconditions` и методом `datarequest', читающим записи из таблицы базы данных. |
| **$ltstablesmanager** | `MySqlTable/null` | `public` | Таблица-менеджер изменений в базе данных. Создаётся через `initmanager()`. Хранит информацию об изменённых записях. |
| **$events** | `Events` | `public` | Объект событий для сервер-клиент коммуникации. Создаётся в конструкторе с ID `"{$this->id}_Events"`. |
| **$vars** | `Vars` | `public` | Объект глобальных переменных для хранения состояния. Создаётся с ID `"ltsSync_{$this->id}"`. Хранит `begin`, `next`, `term`, `client`, `recid`. |
| **$timer** | `int` | `public` | Интервал опроса изменений на сервере в миллисекундах. По умолчанию `5000`. Используется в `setInterval()` на клиенте. |
| **$wrapper** | `mixed/null` | `public` | Элемент, относительно которого отслеживается появление последней строки через `IntersectionObserver`. По умолчанию `$this->dropdown`. |
| **$recidname** | `string` | `public` | Имя поля, в котором хранится ID записи в DataTable. По умолчанию `'id'`. |

## Методы (серверная часть, PHP)

### Констрктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Quark`, устанавливает `$type` в `'DataSync'`. Создаёт объекты `$events`, `$vars`. Инициализирует `$datarequest` функцией загрузки из `$dbtable`. |

### Инициализация

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **initmanager** | `MySql $base`, `bool $create = false` | `$this` | Инициализирует таблицу-менеджер изменений `ltstablesmanager`. Создаёт поля: `name` (имя таблицы), `recid` (ID записи), `client` (автор изменений), `created` (дата), `type` (тип операции: 0=удалено, 1=изменено). Если `$create === true` — создаёт таблицу - менеджер изменений в БД. |
| **dbtable** | `MySqlTable $dbtable` | `$this` | Привязывает таблицу базы данных. Сохраняет в `$dbtable`. Используется в дефолтной функции поиска. Поддерживает fluent interface. |
| **datatable** | `DataTable $datatable` | `$this` | Привязывает объект DataTable. Сохраняет в `$datatable`. Используется для отображения данных на клиенте. Поддерживает fluent interface. |
| **wrapper** | `mixed $el` | `$this` | Устанавливает элемент для отслеживания через `IntersectionObserver`. Может быть строкой (ID), объектом с `$id` или DOM-элементом. Сохраняет в `$wrapper`. Поддерживает fluent interface. |
| **datarequest** | `callable $f` | `$this` | Переопределяет функцию загрузки данных. Функция должна принимать `$ids` (массив ID) и возвращать массив записей. Привязывает контекст через `bindTo($this)`. |

### Управление синхронизацией

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **autochange** | `bool $reg = true` | `$this` | Включает или выключает отслеживание изменений. Устанавливает `$autochange`. Поддерживает fluent interface. |
| **autoupload** | `bool $reg = true` | `$this` | Включает или выключает автоподгрузку данных. Устанавливает `$autoupload`. Поддерживает fluent interface. |
| **condition** | `mixed $key`, `mixed $value = null` | `$this` | Добавляет условие отбора записей в таблице базы данных. Если `$value === null` и `$key` — массив — добавляет все пары ключ-значение. Иначе — добавляет одну пару. Сохраняет в `$conditions`. Поддерживает fluent interface. |
| **begchange** | — | `$this` | Устанавливает максимальный ID в таблице-менеджере при начале работы. Запрашивает последний `id` из `$ltstablesmanager` и сохраняет в `$vars->recid`. Используется для отслеживания новых изменений. |

### Автоподгрузка

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **reset** | — | `$this` | Сбрасывает счётчики автоподгрузки. Устанавливает `$vars->begin = 0`, `$vars->next = 1`. Вызывает `$vars->store()` для сохранения в сессию. Поддерживает fluent interface. |
| **upload** | — | `array` | Загружает очередную порцию записей при автоподгрузке. Получает `begin`, `next` из `$vars`. Устанавливает `LIMIT` через `$this->system`. Вызывает `$this->datarequest`. Обновляет `$vars->begin` и `$vars->next`. Вызывает `$vars->store()`. Возвращает массив записей или пустой массив. Метод используется в серверной части события, отвечающего за автоподгрузку данных. |

### Многопользовательский режим

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **fixchange** | `array $arr`, `int $type = 1` | `$this` | Фиксирует изменения записей в таблице-менеджере. `$arr` — массив ID записей, `$type` — тип операции (1=изменено/добавлено, 0=удалено). Для каждого ID создаёт запись в `$ltstablesmanager` с текущим клиентом и временем. |
| **setlock** | `mixed $recid` | `$this` | Устанавливает блокировку на запись (type=2 в менеджере). Проверяет существующие блокировки, создаёт или обновляет запись в `$ltstablesmanager`. Поддерживает массив ID для множественной блокировки. |
| **removelock** | `mixed $recid` | `$this` | Снимает блокировку с записи. Удаляет запись из `$ltstablesmanager` с `type=2` и matching `recid`. Поддерживает массив ID. |
| **checklock** | `mixed $recid` | `bool` | Проверяет, заблокирована ли запись. Запрашивает запись из `$ltstablesmanager` с `type=2` и matching `recid`. Возвращает `true` если не заблокирована, `false` если заблокирована. |
| **getchange** | — | `array` | Получает массив изменённых записей от других клиентов. Запрашивает записи из `$ltstablesmanager` с `client != текущий` и `id > last_recid`. Разделяет на `$changed` (тип 1) и `$deleted` (тип 0). Возвращает `['result' => bool, 'changed' => array, 'deleted' => array]`. Метод используется в серверной части события, отвечающего за организацию многопользовательского режима. |

### Вспомогательные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **applycondition** | `QueryBuilder $query`, `array $cond` | `$this` | Применяет условия к объекту `QueryBuilder`. Поддерживает ключи: `FIELDS`, `WHERE`, `ORDER`, `LIMIT`, `GROUP`. Игнорирует пустые значения. Вызывается внутри `datarequest()` и `upload()`. |
| **checktime** | — | `$this` | Очищает таблицу-менеджер от устаревших записей старше 120 секунд. Вызывается автоматически в `getchange()`. Сохраняет время следующей проверки в `$vars->checktime`. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Если `$autoupload == true` — создаёт событие `upload`, регистрирует `IntersectionObserver`, генерирует методы `observe()`, `unobserve()`, `reset()`, `upload()`. Если `$autochange == true` — создаёт события `setlock`, `removelock`, `checklock`, `getchange`, `fixchange`, регистрирует `setInterval` для опроса изменений каждые `$timer` мс. Добавляет `$vars` в потомки. Вызывает `parent::compile()`. |

## Методы LTS-объекта (клиентская часть)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(datasync).observe(row)** | `jQuery/Element/string row` | `this` | Начинает отслеживание элемента через `IntersectionObserver`. Если `row` — jQuery — извлекает DOM-элемент. Если строка — находит по ID. Если элемент — использует напрямую. Добавляет элемент в observer. Сохраняет в `this.observerow`. |
| **LTS(datasync).unobserve(row)** | `jQuery/Element/string row` | `this` | Прекращает отслеживание элемента. Если `row` не передан — удаляет `this.observerow`. Если передан — находит элемент и вызывает `unobserve()`. Сбрасывает `this.observerow` если элемент совпадает. |
| **LTS(datasync).upload(usersconditions, reset)** | `object usersconditions`, `bool reset` | `this` | Запускает загрузку очередной порции данных. Вызывает серверное событие `upload` с параметрами. Если `reset === true` — сбрасывает счётчики перед загрузкой. |
| **LTS(datasync).reset(usersconditions)** | `object usersconditions` | `this` | Сбрасывает таблицу, изменяет локальные условия и загружает первый пул данных. Устанавливает `autochange = false`, вызывает `DataTable.clear()`, устанавливает `usersconditions`, вызывает `upload(usersconditions, true)`. |
| **LTS(datasync).condition(usersconditions)** | `object usersconditions` | `this` | Устанавливает локальные условия отбора и сортировки записей для автоподкачки. Сохраняет в `this.usersconditions`. |
| **LTS(datasync).fixchange(changed, deleted)** | `array changed`, `array deleted` | `this` | Передаёт данные об ID изменённых и удалённых записей на сервер. Вызывает серверное событие `fixchange` с массивами ID. |
| **LTS(datasync).setlock(recid)** | `mixed recid` | `this` | Устанавливает блокировку на запись. Проверяет локальный кэш `__locks`, добавляет recid в Set, вызывает серверное событие `setlock`. |
| **LTS(datasync).removelock(recid)** | `mixed recid` | `this` | Снимает блокировку с записи. Удаляет recid из локального кэша `__locks`, вызывает серверное событие `removelock`. |
| **LTS(datasync).checklock(recid)** | `mixed recid` | `bool` | Проверяет, заблокирована ли запись. Вызывает серверное событие `checklock`, возвращает результат из `LTS.get(events).__checklock`. |

## События (серверная часть)

| Событие | Параметры | Возвращает | Описание |
|---------|-----------|------------|----------|
| **upload** | `array $args` с ключами `'usersconditions'`, `'reset'` | `array ['result' => bool, 'data' => array]` | Серверный обработчик автоподгрузки. Если `$reset` — вызывает `reset()`. Если `$usersconditions` — сохраняет в `$this->usersconditions`. Вызывает `upload()`, возвращает данные. |
| **setlock** | `array $args` с ключом `'recid'` | `bool` | Серверный обработчик блокировки записи. Вызывает `$this->setlock($args['recid'])`. Возвращает `true`. |
| **removelock** | `array $args` с ключом `'recid'` | `bool` | Серверный обработчик снятия блокировки. Вызывает `$this->removelock($args['recid'])`. Возвращает `true`. |
| **checklock** | `array $args` с ключом `'recid'` | `bool` | Серверный обработчик проверки блокировки. Вызывает `$this->checklock($args['recid'])`. Возвращает результат проверки. |
| **getchange** | `array $args` | `array ['result' => bool, 'changed' => array, 'deleted' => array]` | Серверный обработчик получения изменений. Вызывает `$this->getchange()`. Возвращает массивы изменённых и удалённых записей. |
| **fixchange** | `array $args` с ключами `'changed'`, `'deleted'` | `bool` | Серверный обработчик фиксации изменений. Вызывает `$this->fixchange($changed, 1)` и `$this->fixchange($deleted, 0)`. Возвращает `true`. |

## События (клиентская часть)

| Событие | Параметры | Описание |
|---------|-----------|----------|
| **upload** | `array result` | Клиентский обработчик ответа автоподгрузки. Если `result` не пуст — вызывает `DataTable.append(result)`. Если записей `$limit` — продолжает наблюдение за последней строкой. |
| **getchange** | `array result` | Клиентский обработчик ответа получения изменений. Если `$changeregime === 1` — автоматически обновляет строки в DataTable через `values()` и удаляет через `clear()`. Если `$changeregime === 0` — сохраняет в `LTS(datasync).changed` и `LTS(datasync).deleted`. Отправляет сигнал `{id}_Change`. |
| **{id}_Change** | — | Сигнал отправляется при получении изменений. Можно подписаться через `LTS.onSignal('{id}_Change', handler)`. |
| **{id}_Upload** | — | Сигнал отправляется при завершении автоподгрузки. Можно подписаться через `LTS.onSignal('{id}_Upload', handler)`. |

## Примеры использования

```php
// Создание DataSync
$datasync = LTS::DataSync('OrdersSync', $maindiv)
    ->initmanager($mybase)
    ->dbtable($orders)
    ->datatable($orderstable)
    ->wrapper($content);

// Включение многопользовательского режима
$datasync->autochange();
// Включение подкачки данных
$datasync->autoupload();

// Установка условий на сервере
$datasync->condition([
    'WHERE' => ['active' => 1, 'user.blocked' => 0],
    'ORDER' => '-date'
]);
```

### Пример ручной обработки изменений на клиенте 

```php
$datasync->changeregime = 0;

$datatable->signal('Change', 
<<<JS
function () {
    let changedrows = LTS(datasync).changed;    // [{...}, {...}, ...] -  строки для DataTable
    let deletedids = LTS(datasync).deleted;     // [id1, id2, ...] - id-ы удаленных записей
    ...
    LTS(datatable).filter([...]);      // Применение фильтра к вновь добавленным строкам
    LTS(datatable).sort(sortname, sortnapr);    // Сортировка
}
JS
, $datasync);
```

## Примечания
*   Класс расширяет `Quark`, поэтому наследует методы управления потомками (`add()`, `get()`, `set()`, `del()`).
*   Класс расширяет `Construct`, поэтому поддерживает хуки жизненного цикла (`$check`, `$before`, `$on`).
*   **Автоподгрузка:** При `$autoupload == true` используется `IntersectionObserver` для отслеживания появления последней строки DataTable. При пересечении — вызывается событие `upload()` для загрузки следующей порции.
*   **Отслеживание изменений:** При `$autochange == true` — каждые `$timer` мс (по умолчанию 5000) вызывается `getchange()` для получения изменений от других клиентов.
*   **Таблица-менеджер:** Таблица `ltstablesmanager` хранит информацию об изменениях: имя таблицы, ID записи, клиент-автор, время, тип операции. Очищается от записей старше 120 секунд.
*   **Блокировки:** Записи блокируются через `setlock()` с `type=2` в менеджере. Предотвращает одновременное редактирование одной записи несколькими клиентами.
*   **Клиент ID:** Уникальный ID клиента генерируется при создании через `bin2hex(random_bytes(16))` и сохраняется в `$vars->client`.
*   **Режимы обновления:** `$changeregime = 1` — автоматическое обновление DataTable при получении изменений. `$changeregime = 0` — изменения помещаются в `LTS(datasync).changed` и `LTS(datasync).deleted` для ручной обработки.
*   **Сигналы:** Сигналы `{id}_Change` и `{id}_Upload` отправляются при получении изменений и завершении автоподгрузки соответственно.
*   **Переменные состояния:** Объект `$vars` с ID `"ltsSync_{$this->id}"` хранит `begin` (смещение), `next` (флаг продолжения), `term` (поисковый запрос), `client` (ID клиента), `recid` (последний ID изменения), `checktime` (время следующей очистки).
*   **IntersectionObserver:** Создаётся при компиляции если `$autoupload == true`. Наблюдает за `$wrapper` или последней строкой таблицы. При пересечении — вызывает `upload()`.
*   **Функция загрузки:** `$datarequest` по умолчанию запрашивает данные из `$dbtable` с условиями. Может быть переопределена через `datarequest()` для кастомной логики (внешние API, кэш и др.).
*   **Условия:** Через `condition()` можно добавить постоянные условия — фильтрацию по статусу, пользователю, дате. Применяются ко всем запросам.
*   **Сброс:** Метод `reset()` сбрасывает счётчики автоподгрузки — вызывается при новом поиске, при загрузке страницы.
*   **Очистка:** Метод `checktime()` автоматически вызывается в `getchange()` — удаляет старые записи из менеджера изменений.
*   **Блокировки в кэше:** На клиенте блокировки кэшируются в `LTS.get(events).__locks` (Set) — для предотвращения повторных запросов на блокировку.
*   **Серверная синхронизация:** При `getchange()` изменения от других клиентов запрашиваются с `client != текущий` и `id > last_recid` — только новые изменения.
*   **Фиксация изменений:** Метод `fixchange()` должен вызываться после успешного сохранения записи на сервере — для уведомления других клиентов.
*   **Проверка блокировки:** Метод `checklock()` возвращает `true` если запись не заблокирована — удобно для валидации перед редактированием.
*   **Множественные блокировки:** Методы `setlock()` и `removelock()` поддерживают массив ID — для блокировки нескольких записей одновременно.
*   **Клиентские методы:** Методы `observe()`, `unobserve()`, `reset()`, `upload()`, `condition()`, `fixchange()`, `setlock()`, `removelock()`, `checklock()` генерируются через `compilemethod()` — доступны через `LTS.get('{id}')` или LTS(var), где var - глобальная PHP-переменная, ссылающаяся на объект DataSync.
*   **Таймер опроса:** Интервал `$timer` по умолчанию 5000 мс — может быть изменён до компиляции для более частого или редкого опроса.
*   **Имя поля ID:** Свойство `$recidname` по умолчанию `'id'` — может быть изменено если в DataTable ID хранится в другом поле (например, `'code'`).
*   **Wrapper элемент:** Через `wrapper()` можно указать контейнер для отслеживания — полезно при кастомной вёрстке.
*   **Observer опции:** При указании `$wrapper` — передаётся в `IntersectionObserver` как `root` опция.
*   **Хранение:** `$vars->store()` вызывается после каждого `upload()` — состояние сохраняется в сессию.
*   **Восстановление:** При загрузке страницы — `$vars->restore()` — восстанавливает `begin`, `next`, `recid` из сессии.
