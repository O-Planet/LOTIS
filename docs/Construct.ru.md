# Класс Construct

## Назначение
Базовый абстрактный класс фреймворка LOTIS, реализующий интерфейс `Ether`. Является родителем для всех объектов системы (UI-компонентов, функциональных классов, объектов предметной области). Реализует полный жизненный цикл объекта: от конструирования до регистрации в глобальном пространстве имён (Universe). Предоставляет механизм хуков (hooks) для внедрения пользовательской логики на ключевых этапах создания объекта и его потомков. Все классы фреймворка наследуются от `Construct`, что обеспечивает единообразие архитектуры и процессов рендеринга.

## Свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$space** | `Space` | Ссылка на объект пространства (Universe), в котором создаётся текущий объект. Обеспечивает доступ к глобальному контексту приложения. |
| **$id** | `string` | Уникальный идентификатор объекта. Используется для генерации HTML-атрибута `id`, CSS-селекторов и связи с клиентским представлением в `lts.js`. |
| **$type** | `string` | Тип объекта. Определяет поведение при рендеринге. Значение `'none'` указывает на абстрактный объект, не имеющий визуального представления. |
| **$shineobject** | `bool` | Флаг необходимости рендеринга объекта как отдельного DOM-элемента. |
| **$owner** | `string/null` | Идентификатор переопределённого владельца. Позволяет изменить родителя в иерархии DOM по сравнению с контекстом создания. |
| **$check** | `callable/null` | Хук-валидатор. Вызывается перед созданием элемента. Если возвращает `false`, процесс создания прерывается. |
| **$before** | `callable/null` | Хук предобработки. Выполняется перед рендерингом элемента (метод `shine()`). |
| **$on** | `callable/null` | Хук постобработки. Выполняется после рендеринга элемента, до регистрации в пространстве имён. |
| **$checkchilds** | `callable/null` | Хук-валидатор для потомков. Если возвращает `false`, создание дочерних элементов пропускается. |
| **$beforechilds** | `callable/null` | Хук предобработки потомков. Выполняется перед рекурсивным созданием дочерних элементов. |
| **$onchilds** | `callable/null` | Хук завершения. Выполняется после добавления всех потомков в пространство имён. |
| **$spaceid** | `static int` | Статический счётчик для автоматической генерации уникальных ID объектов. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Если ID не передан, генерирует уникальный идентификатор вида `'O' + номер` на основе статического счётчика `$spaceid`. Инициализирует свойства по умолчанию. |
| **compile** | — | — | Этап предварительной компиляции метаданных. В базовой реализации пустой, переопределяется в дочерних классах для подготовки данных перед рендерингом. |
| **create** | `Space $space`, `Element $parent = null` | `$this` | Основной метод жизненного цикла. Координирует последовательность: проверка `$check` → хук `$before` → `shine()` → хук `$on` → `addinspace()` → проверка `$checkchilds` → хук `$beforechilds` → `childs()` → хук `$onchilds`. Возвращает `$this` для поддержки fluent interface. |
| **shine** | `Element $parent = null` | — | Создание DOM-элемента. В базовой реализации пустой, переопределяется в дочерних классах для генерации HTML и назначения атрибутов. |
| **childs** | — | — | Рекурсивное создание и регистрация дочерних элементов. В базовой реализации пустой, переопределяется в контейнерах (Div, Form, Grid и др.). |
| **addinspace** | — | — | Регистрация объекта в глобальном пространстве имён (Universe). Делает объект доступным для поиска по ID и обеспечивает связь сервер-клиент. |

## Примечания
*   Класс использует трейт `TypeChecker` для валидации типов данных.
*   Метод `create()` является точкой входа для запуска жизненного цикла объекта. Прямой вызов `shine()`, `childs()` или `addinspace()` не рекомендуется.
*   Хуки (`$check`, `$before`, `$on` и др.) могут быть назначены как PHP-функции, так и анонимные функции (closure).
*   Все хуки выполняются без параметров — данные передаются через контекст объекта (`$this`).
*   При `$type == 'none'` объект не рендерится, но может выполнять функциональную логику (например, `JS`, `CSS`, `Events`).
*   Свойство `$owner` позволяет реализовать сложные сценарии вложенности, когда физический родитель в DOM отличается от логического владельца в коде.
