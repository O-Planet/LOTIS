# Класс Construct

## Назначение
Базовый класс фреймворка LOTIS, реализующий интерфейс `Ether`. Является родителем для всех объектов системы (UI-компонентов, функциональных классов, объектов предметной области). Реализует полный жизненный цикл объекта: от конструирования метаданных до регистрации во вселенной (Space). Предоставляет механизм хуков (hooks) для внедрения пользовательской логики на ключевых этапах создания объекта и его потомков. Все классы фреймворка наследуются от `Construct`, что обеспечивает единообразие архитектуры и процессов рендеринга.

## Свойства

### Основные свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$space** | `Space` | Ссылка на объект вселенной (Space), в котором создаётся текущий объект. Обеспечивает доступ к глобальному контексту приложения. |
| **$id** | `string` | Уникальный идентификатор объекта. Используется для генерации HTML-атрибута `id`, CSS-селекторов и связи с клиентским представлением: `LTS-объектом`. **ВАЖНО!** Если id объекта в конструкторе явно не задан, то ему будет присвоен уникальный id автоматически. |
| **$type** | `string` | Тип объекта. Определяет поведение при рендеринге. Значение `'none'` указывает на абстрактный объект, не имеющий визуального представления. |
| **$shineobject** | `bool` | Флаг необходимости создания объекта методом create со всеми его потомками и помещения во вселенную. |
| **$owner** | `string/null` | Идентификатор переопределённого владельца. Позволяет изменить родителя в иерархии DOM по сравнению с контекстом создания. |
| **$spaceid** | `static int` | Статический счётчик для автоматической генерации уникальных ID объектов. |

### События жизненного цикла

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$check** | `callable/null` | Хук-валидатор. Вызывается перед созданием элемента методом create. Если возвращает `false`, процесс создания прерывается. |
| **$before** | `callable/null` | Хук предобработки. Выполняется перед рендерингом элемента (метод `shine()`). |
| **$on** | `callable/null` | Хук постобработки. Выполняется после рендеринга элемента, до регистрации во вселенной. |
| **$checkchilds** | `callable/null` | Хук-валидатор для потомков. Если возвращает `false`, создание дочерних объектов пропускается. |
| **$beforechilds** | `callable/null` | Хук предобработки потомков. Выполняется перед рекурсивным созданием дочерних объектов. |
| **$onchilds** | `callable/null` | Хук завершения. Выполняется после добавления всех потомков во вселенную. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Если ID не передан, генерирует уникальный идентификатор вида `'O' + номер` на основе статического счётчика `$spaceid`. Инициализирует свойства по умолчанию. |
| **compile** | — | — | Этап предварительной компиляции метаданных. В базовой реализации пустой, переопределяется в дочерних классах для подготовки данных перед рендерингом. |
| **create** | `Space $space`, `Element $parent = null` | `$this` | Основной метод жизненного цикла. Координирует последовательность: проверка `$check` → хук `$before` → `shine()` → хук `$on` → `addinspace()` → проверка `$checkchilds` → хук `$beforechilds` → `childs()` → хук `$onchilds`. Возвращает `$this` для поддержки fluent interface. |
| **shine** | `Element $parent = null` | — | Создание метаданных объекта. В базовой реализации пустой, переопределяется в дочерних классах. |
| **childs** | — | — | Рекурсивное создание и регистрация дочерних объектов. В базовой реализации пустой, переопределяется в контейнерах (Div, Form, Grid и др.). |
| **addinspace** | — | — | Регистрация объекта во вселенной. |

## Рекомендации по использованию

При переопределении класса необходимо:
* Реализовать метод shine() для создания конкретного элемента
* При необходимости переопределить childs() для работы с потомками
* Настроить хуки жизненного цикла через свойства-обработчики
* Учитывать порядок выполнения методов

## Типичные сценарии использования

```php
class MyComponent extends Construct
{
    public __cobstruct($id = '')
    {
        // Код до вызова конструктора базового класса
        parent::__construct($id);
        // Код после вызова конструктора базового класса
    } 
  
    public function shine($parent)
    {
        // Логика создания метаданных объекта
    }

    public function childs()
    {
        // Логика обработки дочерних объектов
    }

    public function addinspace()
    {
        // Логика размещения метаданных во вселенной
    }
}
```

## Примечания
*   Класс использует трейт `TypeChecker` для валидации типов данных.
*   Метод `create()` является точкой входа для запуска жизненного цикла объекта. Прямой вызов `shine()`, `childs()` или `addinspace()` не рекомендуется.
*   Хуки (`$check`, `$before`, `$on` и др.) могут быть назначены как PHP-функции, так и анонимные функции (closure).
*   Все хуки выполняются без параметров — данные передаются через контекст объекта (`$this`).
*   При `$type == 'none'` объект не рендерится, но может выполнять функциональную логику (например, `JS`, `CSS`, `Events`).
*   Свойство `$owner` позволяет реализовать сложные сценарии вложенности, когда физический родитель в DOM отличается от логического владельца в коде.
