# Класс Quark

## Назначение
Базовый класс фреймворка LOTIS, наследующий `Construct`. Представляет собой контейнерный объект с расширенными возможностями управления дочерними элементами и внутренним хранилищем данных. Является родителем для всех UI-компонентов, способных содержать другие объекты (Div, Form, Grid, Tabs и др.). Реализует механизм архивации потомков по секциям, предоставляет методы для поиска, фильтрации, перемещения и управления дочерними элементами. Также отвечает за генерацию и управление клиентским JavaScript-кодом, связанным с объектом.

## Свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$tagname** | `string` | Имя HTML-тега, который будет создан методом `shine()`. По умолчанию пустая строка — переопределяется в дочерних классах. |
| **$classname** | `string` | CSS-класс объекта. Используется для стилизации элемента при рендеринге. |
| **$caption** | `string` | Текстовая подпись объекта. Может отображаться как заголовок, label или содержимое элемента. |
| **$storage** | `array` | Многомерный массив для хранения внутренних данных объекта. Содержит секции: `'childs'` (потомки), `'element'` (DOM-элемент), а также пользовательские секции. |
| **$jscreate** | `JS/null` | Ссылка на JS-объект, скрипты которого должны быть размещены в секции ready скриптов клиентского приложения гарантированно после создания и инициализации текущего объекта. Инициализируется выполняется при первом обращении через метод `js('create')`. |

## Методы

### Методы управления объектами

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Construct`, инициализирует секции хранилища `'childs'` и `'element'` пустыми массивами. |

#### Базовые операции  

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **setid** | `string $id` | `$this` | Устанавливает новый идентификатор объекта. Поддерживает fluent interface. |
| **set** | `string $section`, `string $name`, `mixed $value = null` | `$this` | Добавляет данные в указанную секцию хранилища. Если `$name` указан, но `$value` не передан — добавляет `$name` как элемент массива. Если переданы оба — создаёт пару ключ-значение. Если секция не существует — создаёт её. |
| **get** | `string $section`, `string $name = null` | `mixed/false` | Получает данные из хранилища. Если `$name` не указан — возвращает всю секцию (массив). Если указан — возвращает значение по ключу. Возвращает `false`, если секция или ключ не найдены. |
| **add** | `string $name`, `mixed $value = null` | `$this` | Добавляет объект в секцию `'childs'`. Является обёрткой над `set('childs', ...)`. Поддерживает fluent interface. |
| **addmany** | `...$values` | `$this` | Добавляет несколько потомков одновременно. Принимает переменное количество аргументов, каждый из которых добавляется через `add()`. |
| **del** | `mixed $child` | `$this` | Удаляет потомка из секции `'childs'`. Принимает ID (строку) или объект. Если передан объект — извлекает его ID. Находит объект в массиве и удаляет по ключу. |
| **addin** | `mixed $pos`, `string $name`, `mixed $value = null` | `$this` | Добавляет потомка на конкретную позицию. `$pos` может быть: целым числом (1-based индекс), объектом (вставка перед ним), строкой (ID объекта). Если позиция некорректна — добавляет в конец. Перестраивает массив потомков с учётом новой позиции. |

### Управление дочерними элементами

#### Поиск и фильтрация

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **child** | `string $id` | `Construct/false` | Ищет и возвращает первого потомка с указанным ID. Выполняет линейный поиск по секции `'childs'`. Возвращает `false`, если не найден. |
| **getchild** | `mixed $type`, `mixed $par = null` | `Construct/false` | Возвращает первого потомка указанного типа с опциональным фильтром. Является обёрткой над `getchilds()` с ограничением результата до 1 элемента. |
| **getchilds** | `mixed $type`, `mixed $par = null`, `int $kol = 0` | `array/false` | Возвращает массив потомков указанного типа с фильтрацией. `$type` — строка или массив типов (проверка по свойству `$type`). `$par` — фильтр: `null` (без фильтра), массив (проверка свойств/атрибутов), callable (пользовательская функция). `$kol` — лимит количества (0 — без лимита). |
| **move** | `Quark $newparent`, `mixed $obj` | `$this` | Перемещает потомка к новому родителю. Принимает объект-родитель и потомка (ID или объект). Добавляет потомка в `$newparent`, удаляет из текущего хранилища. Если объект не найден в потомках, но передан как объект — просто добавляет в нового родителя. |

### JavaScript-интеграция

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **js** | `string $area = 'script'` | `JS` | Возвращает или создаёт объект JavaScript для указанной области. Проверяет существующие JS-объекты среди потомков. Для области `'create'` — создаёт лениво `$jscreate` с областью `'ready'`. Для других областей — создаёт новый объект `JS`, добавляет в потомки и возвращает. |

### Жизненный цикл

#### Основные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **childs** | — | — | Рекурсивное создание потомков. Перебирает секцию `'childs'`, для каждого объекта типа `Construct` с флагом `$shineobject == true` вызывает `create()` с передачей текущего пространства и ID родителя. |
| **compile** | — | — | Предварительная компиляция потомков. Вызывает `compile()` для всех потомков, имеющих этот метод. Если существует `$jscreate` — добавляет его в потомки, устанавливает область `'ready'` и компилирует. |
| **addinspace** | — | — | Регистрация DOM-элемента в пространстве имён. Получает элемент из секции `'element'`, если массив не пуст — регистрирует в `$space` по секции `'elements'` с ключом `$this->id`. |

### Структура хранилища

**$storage** содержит следующие секции:

* **childs** — коллекция дочерних элементов
* **element** — информация о создаваемом элементе
* Пользовательские секции для хранения данных

## Примечания
*   Класс расширяет функциональность `Construct`, добавляя возможность хранения и управления коллекциями дочерних объектов.
*   Хранилище `$storage` организовано по секциям — это позволяет разделять потомков, DOM-элементы и пользовательские данные.
*   Метод `getchilds()` поддерживает три типа фильтрации: по типу объекта, по свойствам/атрибутам, по пользовательской функции-предикату.
*   Метод `addin()` реализует сложную логику вставки с перестроением массива — позволяет точно контролировать порядок потомков в DOM.
*   JavaScript-объекты, создаваемые через `js()`, автоматически добавляются в потомки и компилируются вместе с родителем.
*   Специальный объект `$jscreate` для области `'create'` создаётся лениво — только при первом обращении, что оптимизирует использование памяти.
*   Все методы управления потомками (`add`, `del`, `addin`, `move`) возвращают `$this` для поддержки fluent interface.
*   Метод `addinspace()` регистрирует только DOM-элементы, а не всех потомков — регистрация потомков происходит при их собственном вызове `addinspace()`.
*   Класс не реализует метод `shine()` — эта ответственность делегирована дочерним классам (Div, Form, Button и др.).
*   При перемещении объекта через `move()` сохраняется ссылка на сам объект — меняется только принадлежность к родителю в хранилище.
