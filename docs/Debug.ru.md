# Класс Debug

## Назначение
Вспомогательный класс фреймворка LOTIS, предоставляющий унифицированный интерфейс для логирования ошибок валидации аргументов. Является фасадом над классом `Logger`, специализирующимся на регистрации ошибок типов и значений параметров методов. Используется преимущественно трейтом `TypeChecker` для автоматической записи нарушений типизации без прерывания выполнения кода. Класс работает в статическом режиме (паттерн Singleton), не допускает создания экземпляров.

## Свойства
Класс не содержит собственных свойств — все данные передаются непосредственно в метод `Logger::log()`.

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **typeError** | `string $class`, `string $method`, `string $paramName`, `string $expected`, `mixed $actual` | — | Логирует ошибку несоответствия типа аргумента. Формирует структурированное сообщение с именем класса, метода, параметра, ожидаемого типа и фактического типа/значения. Для объектов указывается имя класса, для скалярных типов — тип и значение. Вызывает `Logger::error()` с контекстом диагностики. |
| **valueError** | `string $class`, `string $method`, `string $paramName`, `string $message`, `mixed $value = null` | — | Логирует ошибку недопустимого значения аргумента. Принимает пользовательское сообщение об ошибке и опционально само значение параметра. Вызывает `Logger::error()` с контекстом диагностики. Используется при валидации бизнес-правил, диапазонов, форматов данных. |

## Структура сообщений

### Для typeError

* Записывает:
  * Класс
  * Метод
  * Имя параметра
  * Ожидаемый тип
  * Фактический тип
  * Значение параметра

### Для valueError

* Записывает:
  * Класс
  * Метод
  * Имя параметра
  * Сообщение об ошибке
  * Значение параметра

## Рекомендации по использованию

* Используйте **typeError()** при несоответствии типов параметров
* Применяйте **valueError()** для проверки корректности значений
* Передавайте максимально подробную информацию об ошибке
* Используйте эти методы в местах проверки входных параметров

## Примеры использования

```php
// Пример логирования ошибки типа
Debug::typeError(
    'UserController',
    'createUser',
    'email',
    'string',
    12345
);

// Пример логирования ошибки значения
Debug::valueError(
    'ProductModel',
    'savePrice',
    'price',
    'Цена должна быть положительной',
    -100
);
```

## Примечания
*   Класс запрещает создание экземпляров через `__construct`, `__clone`, `__wakeup` (объявлены private).
*   Все методы объявлены как `public static` — вызываются без создания объекта: `Debug::typeError(...)`.
*   Класс не реализует собственную логику логирования — делегирует запись в `Logger`.
*   Для не скалярных значений (массивы, объекты, ресурсы) в поле `value` выводится только тип данных в скобках, например `(array)`, `(stdClass)`.
*   Для скалярных значений (string, int, float, bool) выводится фактическое значение для упрощения отладки.
*   Основные потребители класса — трейт `TypeChecker` и классы, выполняющие ручную валидацию входных данных.
*   Ошибки записываются с уровнем `ERROR` и включают полный контекст вызова (класс, метод, параметр, ожидаемое/фактическое значение).
*   Класс подключается автоматически через `TypeChecker` во всех классах, наследующих `Construct`.
