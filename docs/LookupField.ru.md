# Класс LookupField

## Назначение
Класс поля поиска с автодополнением фреймворка LOTIS, наследующий `Div`. Предназначен для создания интерактивных полей ввода с выпадающим списком результатов поиска, поддержкой автоподгрузки данных при скролле, привязки к таблице базы данных и синхронизации состояния между сервером и клиентом. Класс объединяет несколько компонентов: `Input` (поле ввода), `Div` (выпадающий блок), `DataTable` (таблица результатов), `Events` (сервер-клиент коммуникация) и `Vars` (хранение состояния). Поддерживает кастомизацию запросов поиска, маппинг полей, фильтрацию результатов и интеграцию с системой сигналов для уведомления о выборе значения.

### Основные возможности

* **Поиск по базе данных** с автодополнением
* **Выпадающее меню** с результатами
* **Автоматическая подкачка** данных
* **Сохранение состояния** поиска
* **Кастомная загрузка** данных
* **Поддержка DataTable**

## Свойства

### Интегрируемые объекты

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$dbtable** | `MySqlTable/null` | `public` | Ссылка на таблицу базы данных для поиска. Устанавливается через `dbtable()`. Используется в запросах поиска через `QueryBuilder`. |
| **$events** | `Events` | `public` | Объект событий для сервер-клиент коммуникации. Создаётся в конструкторе. Используется для обработки поиска и автоподгрузки. |
| **$table** | `DataTable` | `public` | Объект таблицы результатов поиска. Создаётся в конструкторе с ID `"{$this->id}_results"`. Отображает найденные записи. |
| **$input** | `Input` | `public` | Объект поля ввода. Создаётся в конструкторе с ID `"{$this->id}_input"`. Добавляется в потомки компонента. |
| **$vars** | `Vars` | `public` | Объект глобальных переменных для хранения состояния. Создаётся с ID `"ltsSync_{$this->id}"`. Хранит `begin`, `next`, `term`. |
| **$wrapper** | `mixed/null` | `public` | Элемент для отслеживания появления последней строки через `IntersectionObserver`. По умолчанию `$this->dropdown`. Устанавливается через `wrapper()`. |
| **$dropdown** | `Div` | `public` | Объект выпадающего блока. Создаётся в конструкторе с ID `"{$this->id}_dropdown"`. Содержит таблицу результатов. По умолчанию скрыт через CSS. |

### Флаги

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$dropdownenable** | `bool` | `public` | Флаг использования выпадающего блока. По умолчанию `true`. Если `false` — таблица отображается без обёртки dropdown. |
| **$binddatatable** | `bool` | `public` | Флаг привязки `DataTable` к `Div`. По умолчанию `true`. Управляет добавлением таблицы в потомки. |
| **$autoupload** | `bool` | `public` | Флаг разрешения автоподгрузки данных. По умолчанию `true`. Если `true` — используется `IntersectionObserver` для загрузки при скролле. |

### Параметры для формирования запроса к базе данных

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$searchfield** | `string` | `public` | Имя поля в базе данных для поиска. По умолчанию `'name'`. Может включать имя таблицы (`table.field`). Устанавливается через `searchfield()`. |
| **$tablefield** | `string` | `public` | Псевдоним поля для отображения в таблице результатов. По умолчанию `'name'`. Устанавливается автоматически при вызове `searchfield()` с алиасом. |
| **$conditions** | `array` | `public` | Дополнительные условия для запроса поиска. Передаются в `QueryBuilder` через `applycondition()`. Устанавливаются через `condition()`. |
| **$datarequest** | `callable/null` | `public` | Кастомная функция загрузки данных. По умолчанию — поиск по `$dbtable` с `$searchfield`. Переопределяется через `datarequest()`. |
| **$fieldmap** | `array/null` | `public` | Сопоставление полей `DataTable` с полями метода `values`. Формат: `['datatable_field' => 'values_field']`. Устанавливается через `fieldmap()`. |
| **$system** | `array/null` | `public` | Внутренние условия для запросов (LIMIT, OFFSET и др.). Управляется автоматически при автоподгрузке. |
| **$limit** | `int` | `public` | Количество записей для одновременной загрузки. По умолчанию `100`. Используется при автоподгрузке через `upload()`. |

## Методы (серверная часть, PHP)

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Div`, добавляет класс `'lookup-field'`. Создаёт объекты `$input`, `$dropdown`, `$table`, `$events`, `$vars`. Устанавливает дефолтную функцию поиска через `datarequest()`. Добавляет `$events` и `$input` в потомки. |

### Методы настройки связи с БД

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **dbtable** | `MySqlTable $dbtable` | `$this` | Устанавливает таблицу базы данных для поиска. Сохраняет в `$dbtable`. Используется в дефолтной функции поиска. Поддерживает fluent interface. |
| **searchfield** | `string $field` | `$this` | Устанавливает поле для поиска. Поддерживает алиасы через `AS` (`table.field as alias`). Если есть точка и алиас — разделяет на `$searchfield` и `$tablefield`. Иначе — устанавливает оба в одно значение. Поддерживает fluent interface. |
| **condition** | `mixed $key`, `mixed $value = null` | `$this` | Добавляет условие поиска. Если `$value === null` и `$key` — массив — добавляет все пары ключ-значение. Иначе — добавляет одну пару. Сохраняет в `$conditions`. Поддерживает fluent interface. |
| **datarequest** | `callable $f` | `$this` | Устанавливает кастомную функцию загрузки данных. Функция должна принимать `$term` (строка поиска) и возвращать массив записей. Привязывает контекст через `bindTo($this)`. Переопределяет дефолтную функцию поиска. |
| **head** | `array $arr` | `$this` | Устанавливает заголовки колонок таблицы результатов. Вызывает `$this->table->head($arr)`. Поддерживает fluent interface. |
| **fieldmap** | `array $prot` | `$this` | Устанавливает сопоставление полей. Сохраняет в `$fieldmap`. Используется на клиенте для маппинга данных между `DataTable` и формой. Поддерживает fluent interface. |

### Методы работы с данными

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **search** | `string $term` | `array` | Выполняет поиск по термину. Вызывает `$this->datarequest` через `call_user_func()`. Возвращает массив результатов поиска. |
| **reset** | — | `$this` | Сбрасывает счётчики автоподгрузки. Устанавливает `$vars->begin = 0`, `$vars->next = 1`. Вызывает `$vars->store()` для сохранения в сессию. Поддерживает fluent interface. |
| **upload** | — | `array` | Загружает очередную порцию записей при автоподгрузке. Получает `begin`, `next`, `term` из `$vars`. Устанавливает `LIMIT` через `$this->system`. Вызывает `$this->datarequest`. Обновляет `$vars->begin` и `$vars->next`. Вызывает `$vars->store()`. Возвращает массив записей или пустой массив. |

### Настройка отслеживания изменений

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **wrapper** | `mixed $el` | `$this` | Устанавливает элемент для отслеживания через `IntersectionObserver`. Может быть строкой (ID), объектом с `$id` или DOM-элементом. Сохраняет в `$wrapper`. Поддерживает fluent interface. |

### Дополнительные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **label** | `string $caption = null` | `Html/bool` | Управляет связанным label-элементом. Если `$caption` передан — создаёт новый `Html('label')` с атрибутом `for` равным `$input->id`. Если существует — обновляет `$caption`. Если не существует и `$caption === null` — возвращает `false`. |
| **applycondition** | `QueryBuilder $query`, `array $cond` | `$this` | Применяет условия к объекту `QueryBuilder`. Поддерживает ключи: `FIELDS`, `WHERE`, `ORDER`, `LIMIT`, `GROUP`. Игнорирует пустые значения. Вызывается внутри `datarequest()` и `upload()`. |

### Жизненны цикл

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **compile** | — | — | Генерация клиентских JavaScript-методов. Создаёт события `search` и `upload` на сервере и клиенте. Генерирует методы: `observe()`, `unobserve()`, `reset()`, `select()`, `clear()`. Добавляет обработчики событий jQuery для `focus`, `input`, `click`. Если `$autoupload == true` — добавляет `IntersectionObserver` для автоподгрузки. Добавляет `$vars` в потомки. Вызывает `parent::compile()`. |

## Клиентские методы, доступные через LTS-объект

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(lookup).search(term)** | `string term` | — | Вызывает серверное событие `search` с термином поиска. Получает результаты через `Events`, передаёт в `DataTable.create()`. Показывает/скрывает dropdown в зависимости от результатов. |
| **LTS(lookup).value(term)** | `string term` | `mixed` | Получает или устанавливает значение поля ввода. Вызывает `LTS(input).value(term)`. Используется для программного изменения значения. |
| **LTS(lookup).observe(row)** | `jQuery/Element/string row` | `this` | Начинает отслеживание элемента через `IntersectionObserver`. Если `row` — jQuery — извлекает DOM-элемент. Если строка — находит по ID. Если элемент — использует напрямую. Добавляет элемент в observer. Сохраняет в `this.observerow`. |
| **LTS(lookup).unobserve(row)** | `jQuery/Element/string row` | `this` | Прекращает отслеживание элемента. Если `row` не передан — удаляет `this.observerow`. Если передан — находит элемент и вызывает `unobserve()`. Сбрасывает `this.observerow` если элемент совпадает. |
| **LTS(lookup).reset()** | — | `this` | Сбрасывает переменные автоподгрузки. Вызывает `LTS.vars('ltsSync_{id}').set('begin', 0).set('next', 1)`. Вызывает `this.unobserve()`. Возвращает `this` для цепочечных вызовов. |
| **LTS(lookup).select(row)** | `object row` | — | Сохраняет выбранную строку. Устанавливает `this.selected = row`. Если `row === null` — очищает выбор. Используется при клике по строке таблицы. |
| **LTS(lookup).clear()** | — | — | Очищает поле и таблицу. Устанавливает `this.selected = null`. Очищает значение input через `jQuery('#{inputId}').val('')`. Вызывает `DataTable.clear()`. Вызывает `this.reset()`. Скрывает dropdown. |
| **LTS(lookup).upload()** | — | — | Вызывает серверное событие `upload` для автоподгрузки. Получает результаты через `Events`, передаёт в `DataTable.append()`. Если результатов меньше `$limit` — прекращает наблюдение. |

## События и сигналы

| Событие | Параметры | Описание |
|---------|-----------|----------|
| **search(term)** | `string term`` | Поиск по слову. Если `$autoupload == true` — сохраняет `term` в `$vars`, вызывает `upload()`. Иначе — вызывает `search($term)`. Возвращает результаты в формате `['result' => true, 'data' => [...]]`. На клиенте, если `result.result == true` и `result.data.length > 0` — вызывает `DataTable.create(result.data)`, показывает dropdown. Иначе — очищает таблицу, скрывает dropdown. |
| **upload** | `array result` | Автоподгрузка. На сервере вызывает `upload()`, возвращает массив записей. Если записей нет — возвращает `false`. Используется только при `$autoupload == true`.Клиентский обработчик ответа автоподгрузки. На клиенте, если `result` не пуст — вызывает `DataTable.append(result)`. Если записей `$limit` — продолжает наблюдение за последней строкой. |
| **{id}_Selected** | — | Сигнал отправляется при выборе строки таблицы. |

## Специальные классы CSS

| Класс | Описание |
|-------|----------|
| **`.lookup-field`** | Базовый класс компонента — добавляется автоматически в конструкторе. |
| **`.lookup-dropdown`** | Класс выпадающего блока — добавляется `$dropdown`. По умолчанию `display: none`. |
| **`.filterfield`** | Класс для поля фильтрации — добавляется `$input` если компонент имеет класс `filterfield`. |

### Примеры использования

```php
// Подключение базы данных
$base = LTS::MySql('tracking', 'localhost', 'root', 'root');
$works = $base->table('works');
$works->string('name', 100);
$works->float('price');

// Создание поле поиска
$workLookup = LTS::LookupField();

// Привязка к таблице базы данных, определение колонок в таблице поиска  
$workLookup->dbtable($works)->head(['name' => 'Работа']);

// Обработка выбора значения через сигнал
$page = LTS::Div();
$page->signal('Selected', 
<<<JS
    function() {
        const selected = LTS(workLookup).selected;
        if (selected) 
            alert('Выбрано: ' + selected.name + ' — ' + selected.price + ' ₽');
    }
JS
, $workLookup);
```

## Примечания
*   Класс расширяет `Div`, поэтому наследует все методы Flexbox (`flex()`, `row()`, `column()`, `content()`, `align()`, `gap()` и др.), управления потомками (`add()`, `del()`, `getchilds()`, `move()`) и стилизации (`css()`, `width()`, `height()`).
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   **Автоподгрузка:** При `$autoupload == true` используется `IntersectionObserver` для отслеживания появления последней строки таблицы. При пересечении — вызывается событие `upload()` для загрузки следующей порции.
*   **Переменные состояния:** Объект `$vars` с ID `"ltsSync_{$this->id}"` хранит `begin` (смещение), `next` (флаг продолжения), `term` (текущий поисковый запрос). Сохраняется в сессию через `$vars->store()`.
*   **Дефолтная функция поиска:** Если не переопределена через `datarequest()` — выполняет запрос к `$dbtable` с условием `LIKE '%term%'` по `$searchfield`. Применяет `$conditions` и `$system`.
*   **Маппинг полей:** Свойство `$fieldmap` используется на клиенте для сопоставления имён полей `DataTable` с полями формы. Устанавливается через `fieldmap()`, передаётся в JS через `compile()`.
*   **Выпадающий блок:** При `$dropdownenable == true` — таблица результатов оборачивается в `$dropdown` с классом `'lookup-dropdown'`. При `false` — таблица добавляется напрямую.
*   **Обработчики событий:** При компиляции добавляются обработчики jQuery для `focus`, `input`, `click` на `$input`, а также глобальный `click` на `document` для скрытия dropdown при клике вне компонента.
*   **Сигнал выбора:** При клике по строке таблицы отправляется сигнал `{id}_Selected` — можно подписаться через `LTS.onSignal()` для реакции на выбор значения.
*   **Сброс состояния:** Метод `reset()` сбрасывает счётчики автоподгрузки — вызывается при `clear()`, при новом поиске, при загрузке страницы.
*   **Кастомный поиск:** Через `datarequest()` можно переопределить логику поиска — например, использовать внешний API, сложные запросы с JOIN, кэширование результатов.
*   **Условия поиска:** Через `condition()` можно добавить постоянные условия — например, фильтрацию по статусу, пользователю, дате. Применяются ко всем запросам.
*   **Поле поиска:** Метод `searchfield()` поддерживает алиасы — `table.field as alias` разделяется на `$searchfield` (для запроса) и `$tablefield` (для отображения).
*   **Label:** Метод `label()` создаёт связанный label-элемент с атрибутом `for` равным `$input->id` — обеспечивает accessibility.
*   **Автоподгрузка:** При `$autoupload == true` после загрузки `$limit` записей — продолжается наблюдение за последней строкой. Если записей меньше `$limit` — наблюдение прекращается.
*   **Wrapper:** Через `wrapper()` можно указать элемент для отслеживания — по умолчанию `$this->dropdown`. Может быть ID, объект с `$id` или DOM-элемент.
*   **Очистка:** Метод `clear()` на клиенте очищает поле ввода, таблицу, переменные состояния, скрывает dropdown — полный сброс компонента.
*   **Выбор строки:** При клике по строке таблицы — значение `$tablefield` записывается в `$input`, отправляется сигнал `{id}_Selected`, скрывается dropdown.
*   **Поиск при вводе:** При вводе текста в `$input` — вызывается `search(term)` если длина > 2 (при `$autoupload == false`) или сразу (при `$autoupload == true`).
*   **Первый фокус:** При первом фокусе на `$input` — вызывается поиск с текущим значением. Флаг `firstloaded` предотвращает повторный поиск.
*   **Клик вне компонента:** При клике вне `$this->id` — скрывается dropdown, восстанавливается значение выбранной строки или очищается поле.
*   **События Events:** Серверные события `search` и `upload` регистрируются через `$events->server()`, клиентские — через `$events->client()`. Обработчики ответов устанавливаются через `LTS.request()`.
*   **IntersectionObserver:** Создаётся при компиляции если `$autoupload == true`. Наблюдает за `$wrapper` или последней строкой таблицы. При пересечении — вызывает `upload()`.
*   **Передача context:** Функция `$datarequest` привязывается к контексту через `bindTo($this)` — внутри функции доступен `$this` для обращения к свойствам компонента.
*   **LIMIT:** Параметр `$limit` по умолчанию `100` — определяет количество записей за одну загрузку. Может быть изменён до компиляции.
*   **Системные условия:** Свойство `$system` используется для временных условий (LIMIT, OFFSET) — сбрасывается после каждого запроса через `$this->system = null`.
*   **Хранение состояния:** `$vars` сохраняет состояние между запросами — `begin`, `next`, `term` доступны после перезагрузки страницы в рамках сессии.
*   **Маппинг на клиенте:** `$fieldmap` передаётся в JS через `compile()` — используется для преобразования имён полей при установке значений в форму.
*   **Таблица результатов:** `$table` — объект `DataTable` с ID `"{$this->id}_results"`. Поддерживает все методы `DataTable`: `create()`, `append()`, `values()`, `clear()`, `sort()`.
*   **Поле ввода:** `$input` — объект `Input` с ID `"{$this->id}_input"`. Имеет атрибут `autocomplete='off'` для отключения браузерного автозаполнения.
*   **Выпадающий блок:** `$dropdown` — объект `Div` с ID `"{$this->id}_dropdown"`. По умолчанию скрыт через `css()->add('display', 'none')`. Показывается при результатах поиска.
*   **События jQuery:** Обработчики регистрируются через `$(document).on()` — работают с динамически созданными элементами.
*   **Сигналы:** Сигнал `{id}_Selected` отправляется при выборе строки — можно использовать для обновления связанных компонентов.
*   **Ошибки:** При ошибке поиска — показывается `alert(result.error)`, скрывается dropdown. Обработка через клиентский обработчик `search()`.
*   **Пустые результаты:** Если поиск не дал результатов — таблица очищается, dropdown скрывается. Не показывается сообщение об ошибке.
*   **Первый запуск:** Флаг `firstloaded` предотвращает повторный поиск при фокусе — устанавливается в `true` после первого поиска.
*   **Восстановление значения:** При клике вне компонента — если есть `selectedrow` — восстанавливается значение в `$input`. Иначе — поле очищается.
*   **Наблюдение:** Метод `observe()` принимает jQuery, строку (ID) или DOM-элемент — универсальный интерфейс для разных сценариев.
*   **Прекращение наблюдения:** Метод `unobserve()` без параметров — прекращает наблюдение за `this.observerow`. С параметром — за указанным элементом.
*   **Сброс переменных:** Метод `reset()` на клиенте вызывает `LTS.vars().set()` — переменные сохраняются в localStorage и сессию.
*   **Загрузка данных:** Метод `upload()` на сервере — инкрементальная загрузка с `begin` и `limit`. Обновляет `$vars->begin` для следующей порции.
*   **Флаг продолжения:** Переменная `$vars->next` — `1` если есть ещё записи, `0` если загружено всё. Управляет продолжением наблюдения.
*   **Поисковый термин:** Переменная `$vars->term` — хранит текущий поисковый запрос. Используется при автоподгрузке для консистентности результатов.
*   **Кастомизация:** Через `datarequest()` можно реализовать любой источник данных — БД, API, файл, кэш. Возвращать массив ассоциативных массивов.
*   **Условия:** Через `condition()` можно добавить фильтрацию — например, только активные записи, только для текущего пользователя.
*   **Сортировка:** Через `condition(['ORDER' => 'name'])` можно задать сортировку результатов. Применяется ко всем запросам.
*   **Лимиты:** Через `condition(['LIMIT' => '0, 50'])` можно переопределить `$limit` для конкретных запросов.
*   **Группировка:** Через `condition(['GROUP' => 'category'])` можно сгруппировать результаты. Используется `QueryBuilder::groupBy()`.
*   **Поля:** Через `condition(['FIELDS' => 'id, name, email'])` можно выбрать конкретные поля. Используется `QueryBuilder::select()`.
*   **WHERE:** Через `condition(['WHERE' => 'status = 1'])` можно добавить сырое условие. Используется `QueryBuilder::where()`.
*   **Маппинг:** `$fieldmap` — ассоциативный массив `['datatable_field' => 'form_field']`. Используется при передаче данных в форму.
*   **Wrapper:** Через `wrapper()` можно указать контейнер для отслеживания — полезно при кастомной вёрстке dropdown.
*   **Автоподгрузка:** При `$autoupload == false` — загрузка всех результатов сразу. При `true` — инкрементальная с `IntersectionObserver`.
*   **События:** События `search` и `upload` регистрируются автоматически — не требуют явного вызова.
*   **Клиентские методы:** Методы `observe()`, `unobserve()`, `reset()`, `select()`, `clear()` генерируются через `compilemethod()` — доступны через `LTS.get('{id}')`.
*   **Input атрибуты:** `$input` имеет атрибут `lookupfield` равный `$this->id` — используется для связи с компонентом.
*   **Dropdown CSS:** `$dropdown` имеет `display: none` по умолчанию — показывается при результатах поиска через jQuery `.show()`.
*   **Table ID:** `$table` имеет ID `"{$this->id}_results"` — используется в селекторах jQuery и клиентских методах.
*   **Events ID:** `$events` имеет автоматический ID — используется для регистрации обработчиков через `LTS.request()`.
*   **Vars ID:** `$vars` имеет ID `"ltsSync_{$this->id}"` — уникальное имя для хранения состояния в сессии.
*   **Поиск по фокусу:** При первом фокусе на `$input` — вызывается поиск с текущим значением. Флаг `firstloaded` предотвращает повтор.
*   **Поиск при вводе:** При каждом вводе символа — вызывается `search(term)`. При `$autoupload == false` — только если `term.length > 2`.
*   **Скрытие dropdown:** При клике вне компонента — dropdown скрывается. Значение `$input` восстанавливается из `selectedrow` или очищается.
*   **Выбор строки:** При клике по строке — `select(row)`, значение в `$input`, сигнал `{id}_Selected`, скрытие dropdown.
*   **Очистка:** Метод `clear()` — сброс всех состояний, очистка `$input`, `$table`, `$vars`, скрытие dropdown.
*   **Наблюдение:** `IntersectionObserver` создаётся один раз при компиляции — переиспользуется для всех порций данных.
*   **Последняя строка:** Наблюдение за последней строкой — при пересечении — загрузка следующей порции через `upload()`.
*   **Флаг next:** Если `result.data.length < $limit` — `$vars->next = 0` — прекращение наблюдения. Иначе — `$vars->next = 1`.
*   **Смещение begin:** При каждой загрузке — `$vars->begin += $limit` — для следующей порции данных.
*   **Термин term:** Сохраняется в `$vars->term` — используется при автоподгрузке для консистентности результатов поиска.
*   **Хранение:** `$vars->store()` вызывается после каждого `upload()` — состояние сохраняется в сессию.
*   **Восстановление:** При загрузке страницы — `$vars->restore()` — восстанавливает `begin`, `next`, `term` из сессии.
*   **Сброс:** Метод `reset()` — сбрасывает `begin = 0`, `next = 1` — для нового поиска.
*   **Клиентский reset:** На клиенте — `LTS.vars().set('begin', 0).set('next', 1)` — синхронизация с сервером.
*   **События jQuery:** `focus`, `input`, `click` на `$input` — управляют поиском и отображением dropdown.
*   **Глобальный click:** На `document` — скрывает dropdown при клике вне компонента.
*   **Сигнал Selected:** Отправляется при выборе строки — для уведомления других компонентов.
*   **Маппинг полей:** `$fieldmap` передаётся в JS — используется при передаче данных в форму.
*   **Wrapper элемент:** Может быть строкой (ID), объектом с `$id`, или DOM-элементом — универсальный интерфейс.
*   **Observer опции:** При указании `$wrapper` — передаётся в `IntersectionObserver` как `root` опция.
