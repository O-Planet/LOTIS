# Класс MySql

## Назначение
Основной класс работы с базами данных MySQL в ORM фреймворка LOTIS. Предназначен для установления соединения с сервером баз данных, управления подключением, выполнения SQL-запросов, подготовки параметризованных запросов и управления транзакциями. Класс предоставляет низкоуровневый API для взаимодействия с MySQL через расширение mysqli, поддерживает автоматическое переподключение при разрыве соединения, экранирование строк и проверку целостности данных при удалении записей. Является владельцем для объектов MySqlTable — создаёт, кэширует и управляет пулом таблиц приложения. Все операции с данными выполняются через экземпляры MySqlTable, полученные от этого класса.

### Основные возможности

* **Подключение** к базе данных
* **Выполнение запросов** различного типа
* **Подготовка данных** для безопасного использования
* **Транзакционная работа** с данными
* **Проверка целостности** данных
* **Управление таблицами** и их структурой

При работе с LOTIS ORM чаще всего будет требоваться два метода этого класса: подключение к базе данных (метод `connect`) и получение ссылки на таблицу (метод `table`). Остальные методы используются в составе `MySqlTable` и `QueryBuilder`.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$link** | `mysqli/null` | `public` | Активное соединение с базой данных. Инициализируется при первом вызове `connect()` или `query()`. До подключения содержит `null`. |
| **$name** | `string` | `public` | Имя базы данных. Используется для формирования SQL-запросов, создания БД и обращения к таблицам через `information_schema`. |
| **$dbtables** | `array` | `public` | Ассоциативный массив объектов MySqlTable по именам таблиц. Кэширует загруженные структуры таблиц для ускорения повторных обращений. Ключи — имена таблиц, значения — объекты MySqlTable. |
| **$server** | `string` | `private` | Адрес сервера MySQL (хост). Устанавливается в конструкторе. Используется при подключении через `new mysqli()`. |
| **$user** | `string` | `private` | Имя пользователя для подключения к БД. Устанавливается в конструкторе. |
| **$password** | `string` | `private` | Пароль для подключения к БД. Устанавливается в конструкторе. |
| **$asarray** | `bool` | `private` | Флаг формата возврата результатов запросов. Если `true` — результаты возвращаются как ассоциативные массивы, если `false` — как объекты stdClass. По умолчанию `false`. Управляется методами `assarray()`/`assobjects()`. |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $name`, `string $server`, `string $user`, `string $password` | — | Конструктор класса. Сохраняет параметры подключения в приватные свойства, инициализирует `$dbtables` пустым массивом, `$link` в `null`. Не устанавливает соединение сразу — подключение происходит лениво при первом запросе через `connect()`, `query()`, `prepare()` или `real_escape_string()`. |

### Методы работы с подключением

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **connect** | — | `bool` | Устанавливает соединение с сервером MySQL через `new mysqli($server, $user, $password)`. Проверяет `$link->connect_errno` — при ошибке возвращает `false`. При успешном подключении сохраняет объект mysqli в `$link` и возвращает `true`. Вызывается автоматически при первом запросе. |
| **query** | `string $str_sql_query` | `mysqli_result/false` | Выполняет произвольный SQL-запрос. Если `$link` не установлен — вызывает `connect()`. При ошибке выполнения (исключение или ошибка mysqli) возвращает `false`. Возвращает объект результата mysqli при успехе. Поддерживает SELECT, INSERT, UPDATE, DELETE, CREATE, DROP и другие запросы. |
| **prepare** | `string $str_sql_query`, `array $values` | `mysqli_stmt/false` | Подготавливает параметризованный SQL-запрос с привязкой значений. Автоматически определяет типы параметров: `i` — int, `d` — float, `s` — string. Вызывает `bind_param()` с распаковкой массива через `...$values` и `execute()`. Возвращает объект mysqli_stmt при успехе, `false` при ошибке. |
| **result** | `mysqli_stmt $stmt`, `bool $asarray = false` | `array/false` | Извлекает результаты из подготовленного запроса. Поддерживает два режима: через `get_result()` (требуется mysqlnd, быстрее) и через `bind_result()` (универсальный fallback). Форматирует результат согласно параметру `$asarray` или свойству `$this->asarray`. Возвращает массив строк (массивов или объектов) или `false` при ошибке. Закрывает stmt после извлечения данных. |

### Методы управления данными

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **create** | — | `mysqli_result/false` | Создаёт базу данных если она не существует. Выполняет запрос `CREATE DATABASE IF NOT EXISTS {$name}`. Если соединение не установлено — вызывает `connect()`. Возвращает результат запроса или `false` при ошибке. Полезно для автоматического развёртывания БД при первом запуске приложения. Не всегда конкретный хостинг позволяет выполнять этот запрос, предполагая создание баз данных через панель управления. |
| **table** | `string $name` | `MySqlTable` | Возвращает или создаёт объект MySqlTable по имени. Если таблица уже есть в `$dbtables` — возвращает существующий объект (кэш). Иначе — создаёт новый MySqlTable, сохраняет в `$dbtables`, устанавливает `$owner` в текущий объект MySql и возвращает. Основной метод получения доступа к таблицам. |
| **existstable** | `string $name` | `bool` | Проверяет наличие таблицы в кэше `$dbtables`. Возвращает `true` если таблица уже загружена через `table()` или `loadtable()`, `false` если нет. Не проверяет реальное наличие таблицы в БД — только наличие в памяти. |
| **loadtable** | `string $name` | `MySqlTable/false` | Загружает структуру таблицы из базы данных. Выполняет запросы к `information_schema.TABLES` (проверка существования), `information_schema.COLUMNS` (поля) и `information_schema.STATISTICS` (индексы). Создаёт объекты MySqlField для каждого поля, определяет типы, nullable, default values, ENUM-значения. Определяет внешние ключи по суффиксу `_id`. Сохраняет таблицу в `$dbtables` и возвращает объект. Возвращает `false` если таблица не существует. |
| **real_escape_string** | `string $str` | `string` | Экранирует специальную строку для безопасного использования в SQL-запросе. Если соединение не установлено — вызывает `connect()`. Возвращает экранированную строку через `$link->real_escape_string()`. Используется для защиты от SQL-инъекций при формировании запросов без параметризации. |
| **integritycheck** | `mixed $table`, `mixed $rec` | `bool` | Проверяет целостность данных перед удалением записи. Принимает таблицу (объект MySqlTable или имя) и запись (ID, массив с ключом `id` или объект с свойством `id`). Ищет ссылки на удаляемую запись в других таблицах через поля типа `TABLE` (внешние ключи). Если найдены ссылки — возвращает `false` (удаление запрещено). Если ссылок нет — возвращает `true`. |

### Транзакционная работа

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **begin** | — | `bool` | Начинает транзакцию через `$link->begin_transaction()`. Если соединение не установлено — вызывает `connect()`. Возвращает `true` при успехе, `false` при ошибке подключения. Работает только с таблицами InnoDB. |
| **commit** | — | `bool` | Фиксирует транзакцию через `$link->commit()`. Возвращает `true` при успехе, `false` если соединение не установлено. Все изменения с момента `begin()` сохраняются в БД. |
| **rollback** | — | `bool` | Откатывает транзакцию через `$link->rollback()`. Возвращает `true` при успехе, `false` если соединение не установлено. Все изменения с момента `begin()` отменяются. |
| **transaction** | `callable $callback` | `mixed/false` | Выполняет код в контексте транзакции с автоматическим rollback при ошибке. Вызывает `begin()`, выполняет `$callback($this)`, при успехе — `commit()`, при ошибке/исключении — `rollback()`. Возвращает результат callback или `false` при неудаче. Рекомендуется для всех операций записи для обеспечения атомарности. |

### Методы получения информации

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **geterror** | — | `string` | Возвращает текст последней ошибки от сервера MySQL через `$link->error`. Используется для отладки после неудачных запросов. |
| **lastInsertId** | — | `int` | Возвращает ID последней вставленной записи через `$link->insert_id`. Используется после INSERT-запросов для получения автоинкрементного ID. Возвращает 0 если последний запрос не был INSERT. |
| **assarray** | — | `$this` | Устанавливает флаг `$asarray` в `true`. Результаты всех последующих запросов будут возвращаться как ассоциативные массивы. Поддерживает fluent interface для цепочечных вызовов. |
| **assobjects** | — | `$this` | Устанавливает флаг `$asarray` в `false`. Результаты всех последующих запросов будут возвращаться как объекты stdClass. Поддерживает fluent interface для цепочечных вызовов. |

### Приватные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **addFieldToTable** | `MySqlTable $table`, `object $col` | — | Приватный метод добавления поля в таблицу при загрузке структуры. Анализирует тип колонки из `information_schema.COLUMNS`, создаёт соответствующий объект MySqlField (VARCHAR, INT, ENUM, DECIMAL, DATETIME и др.). Определяет внешние ключи по суффиксу `_id` — если поле называется `{tablename}_id` и таблица существует — устанавливает тип `TABLE` и ссылку на таблицу. |
| **parseEnumFromColumnType** | `string $columnType` | `array` | Приватный метод извлечения значений из ENUM-типа. Парсит строку вида `enum('value1','value2','value3')` через регулярное выражение. Возвращает массив допустимых значений без кавычек. Используется при загрузке структуры таблицы. |
| **existsTableInDatabase** | `string $tableName` | `bool` | Приватный метод проверки существования таблицы в БД. Выполняет запрос к `information_schema.TABLES` с `LIMIT 1`. Возвращает `true` если таблица найдена, `false` если нет. Используется при определении внешних ключей. |

### Примеры использования

```php
// Подключение к базе данных
$db = LTS::MySql('mydatabase', 'localhost', 'user', 'password');

// Создание базы данных
$db->create();

// Создание таблицы
$table = $db->table('users');
$table->string('name', 100)
     ->email('email')
     ->create();

// Выполнение простого запроса без использования MySqlTable
$result = $db->query("SELECT * FROM users");
```

## Примечания
*   Класс не требует внешнего ORM — все маппинги определяются автоматически на основе структуры БД через `information_schema`.
*   Соединение с БД устанавливается лениво — при первом вызове `query()`, `prepare()`, `connect()`, `real_escape_string()`, `begin()` или `loadtable()`.
*   Параметризованные запросы через `prepare()` автоматически определяют типы параметров — не требуется указывать типы вручную (`i`, `d`, `s` формируются автоматически).
*   Метод `result()` поддерживает два режима работы — через `get_result()` (быстрее, требует mysqlnd) и через `bind_result()` (универсально, работает без mysqlnd).
*   Флаг `$asarray` управляет форматом возврата результатов — может быть установлен глобально через `assarray()`/`assobjects()` или локально через параметр `result()`.
*   Метод `integritycheck()` предотвращает удаление записей на которые есть ссылки в других таблицах — обеспечивает ссылочную целостность на уровне приложения (внешние ключи определяются по суффиксу `_id`).
*   Внешние ключи определяются автоматически по соглашению именования — поля с суффиксом `_id` считаются ссылками на таблицы с соответствующим именем (например, `user_id` → таблица `user`).
*   Метод `transaction()` обеспечивает безопасное выполнение операций с автоматическим откатом при ошибках — рекомендуется для всех операций записи (INSERT, UPDATE, DELETE).
*   Кэш таблиц `$dbtables` ускоряет повторные обращения — структура таблицы загружается из БД только один раз за сессию через `loadtable()` или `table()`.
*   Метод `loadtable()` загружает полную структуру таблицы включая поля, типы, nullable, default values, индексы, комментарии и ENUM-значения.
*   ENUM-поля парсятся автоматически — значения сохраняются в объекте MySqlField для валидации и генерации UI.
*   Класс поддерживает работу с несколькими базами данных — можно создать несколько экземпляров MySql с разными параметрами подключения.
*   Ошибки подключения и выполнения запросов не выбрасывают исключения — возвращают `false` для обработки разработчиком (исключения перехватываются внутри методов).
*   Метод `real_escape_string()` должен вызываться для всех пользовательских данных перед вставкой в запросы без параметризации.
*   Класс совместим с MySQL 5.7+ и MariaDB 10.0+.
*   Метод `create()` полезен для автоматического развёртывания БД при первом запуске приложения — не требует ручного создания через phpMyAdmin.
*   Транзакции работают только с таблицами InnoDB — для MyISAM `begin()`/`commit()`/`rollback()` не имеют эффекта.
*   Метод `lastInsertId()` возвращает 0 если последний запрос не был INSERT — требуется проверка перед использованием.
*   Объекты MySqlTable, созданные через `table()`, сохраняют ссылку на владельца (`$owner`) — это позволяет выполнять запросы в контексте подключения без передачи `$link`.
*   При загрузке таблицы поле `id` игнорируется в `addFieldToTable()` — считается первичным ключом по умолчанию, не добавляется в `$fields`.
*   Индексы загружаются из `information_schema.STATISTICS` — первичный ключ (`PRIMARY`) не включается в список индексов таблицы.
*   Для полей DECIMAL сохраняется точность — параметры `dd` (цифры до запятой) и `dr` (цифры после запятой) извлекаются из `COLUMN_TYPE`.
*   Метод `existstable()` проверяет только кэш — для проверки реального наличия в БД нужно использовать `loadtable()` или прямой запрос к `information_schema`.
*   При ошибке подготовки запроса `prepare()` возвращает `false` — требуется проверка перед вызовом `result()`.
*   Класс не реализует миграции — для изменения структуры БД нужно использовать `MySqlTable::update()` или SQL-запросы напрямую через `query()`.
*   Для пулинга соединений рекомендуется создавать один экземпляр MySql на запрос — повторное использование экономит ресурсы и избегает множественных подключений.
*   Метод `table()` является основным способом доступа к таблицам — не рекомендуется создавать MySqlTable напрямую через `new`.
*   При загрузке структуры таблицы комментарии полей (`COLUMN_COMMENT`) извлекаются но не сохраняются в MySqlField — могут использоваться для генерации документации.
*   Класс не поддерживает именованные параметры в prepared statements — используются позиционные параметры `?`.
*   Для полей DATETIME значения должны быть в формате `'Y-m-d H:i:s'` — автоматического преобразования нет.
*   Булевы поля хранятся как TINYINT(1) — отдельного типа BOOL в MySQL нет.
*   Метод `transaction()` перехватывает исключения — если callback выбросит Exception, будет выполнен rollback.
*   При работе с несколькими БД нужно создавать отдельные экземпляры MySql — один объект = одно подключение.
*   Класс не реализует connection pooling — каждое создание экземпляра потенциально создаёт новое подключение.
*   Для продления времени жизни подключения рекомендуется хранить экземпляр MySql в глобальной переменной или синглтоне.
*   Метод `geterror()` возвращает ошибку последнего запроса — для получения истории ошибок нужно использовать класс `Logger`.
*   При подключении к удалённому серверу требуется настроить доступ в MySQL (пользователь, хост, права).
*   Класс не шифрует пароль в памяти — для повышенной безопасности рекомендуется использовать environment variables.
*   Метод `query()` поддерживает множественные запросы только если разрешено в настройках MySQL (`MULTI_STATEMENTS`).
*   Класс не реализует retry-логику при разрыве соединения — требуется обработка `false` возвратов в пользовательском коде.
