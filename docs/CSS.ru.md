# Класс CSS

## Назначение
Класс управления стилями фреймворка LOTIS, наследующий `Quark`. Предназначен для динамической генерации и подключения CSS-правил к объектам приложения. Поддерживает два режима работы: встраивание инлайн-стилей непосредственно в HTML-документ и подключение внешних CSS-файлов через тег `<link>`. Все стили, определённые через объект CSS, автоматически добавляются в пространство имён (Space) при рендеринге и применяются к элементам с соответствующим ID-классом. Класс работает как контейнер — правила стилей добавляются в его хранилище как потомки.

## Свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$tagname** | `string` | Наследуется от `Quark`. Для CSS-объектов не используется (пустая строка). |
| **$classname** | `string` | Наследуется от `Quark`. CSS-класс объекта, может использоваться как селектор для стилей. |
| **$caption** | `string` | Наследуется от `Quark`. Для CSS-объектов не используется. |
| **$storage** | `array` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (правила стилей) и `'element'` (DOM-элемент). |
| **$jscreate** | `JS/null` | Наследуется от `Quark`. Ссылка на JavaScript-объект для клиентской логики, связанной со стилями. |
| **$type** | `string` | Устанавливается в `'CSS'` в конструкторе. Определяет тип объекта для фильтрации и обработки. |
| **$space** | `Space` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |
| **$id** | `string` | Наследуется от `Construct`. Уникальный идентификатор, используется как CSS-класс для селектора стилей. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Quark`, устанавливает свойство `$type` в значение `'CSS'`. Инициализирует хранилище секциями `'childs'` и `'element'`. |
| **shine** | `Element $parent = null` | — | Генерация CSS-элементов. Перебирает потомков в секции `'childs'`. Если значение содержит `'.css'` — создаёт запись для тега `<link>` с внешним файлом. Иначе — формирует массив инлайн-стилей. Добавляет сгенерированные записи в `$space->storage['elements']` для последующего рендеринга в HTML. |
| **compile** | — | — | Пустая реализация. Наследуется от `Construct`, переопределяется без логики — CSS-объекты не требуют предварительной компиляции метаданных. |
| **childs** | — | — | Пустая реализация. Наследуется от `Construct`, переопределяется без логики — CSS-объекты не создают дочерних DOM-элементов в традиционном понимании. |

## Примечания
*   Класс расширяет `Quark`, поэтому поддерживает все методы управления потомками: `add()`, `get()`, `set()`, `del()`, `getchilds()` и др.
*   Правила стилей добавляются через метод `add()` как пары ключ-значение, где ключ — имя CSS-свойства, значение — его значение.
*   Внешние CSS-файлы определяются по наличию подстроки `'.css'` в значении — такие записи обрабатываются отдельно и генерируют теги `<link rel="stylesheet">`.
*   Если указан `$id` объекта — он используется как CSS-класс в селекторе (запись `'class' => $this->id` в массиве элемента).
*   CSS-объекты не имеют визуального представления в DOM — они генерируют только записи в пространстве имён для вставки в `<head>` или `<style>`.
*   Методы `compile()` и `childs()` переопределены как пустые — жизненный цикл CSS-объекта ограничивается этапом `shine()`.
*   Все стили, добавленные через объект CSS, применяются глобально в рамках текущего пространства (Space).
*   Для применения стилей к конкретному элементу рекомендуется использовать `$id` объекта CSS как класс целевого элемента.
*   Объекты CSS автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   Класс не реализует клиентскую логику работы со стилями — только серверную генерацию HTML/CSS. Для динамического изменения стилей на клиенте используется класс `JS`.
