# Класс DataView

## Назначение
Центральный компонент фреймворка LOTIS для создания интерактивных документов с несколькими табличными частями. Наследует `Grid` и объединяет три элемента интерфейса в единый компонент: `DataTable` (список записей с сортировкой и выбором), `Form` (форма создания/редактирования записей) и `FilterForm` (панель отбора и поиска). Предназначен для быстрого создания CRUD-интерфейсов с поддержкой подчинённых таблиц, событийной моделью сохранения/удаления и адаптивной сеточной вёрсткой.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$table** | `DataTable` | `public` | Объект таблицы для отображения списка записей. Создаётся в конструкторе с ID `"{$this->id}_list"`. |
| **$element** | `Form` | `public` | Объект формы для просмотра и редактирования записей. Создаётся в конструкторе с ID `"{$this->id}_element"`. |
| **$filter** | `FilterForm` | `public` | Объект формы поиска и отбора в списке. Создаётся в конструкторе с ID `"{$this->id}_filter"`. |
| **$newbutton** | `Button` | `public` | Кнопка создания новой записи. Создаётся в конструкторе с ID `"{$this->id}_newbutton"`. |
| **$delbutton** | `Button` | `public` | Кнопка удаления выбранных записей. Создаётся в конструкторе с ID `"{$this->id}_delbutton"`. |
| **$filterbutton** | `Button` | `public` | Кнопка открытия/закрытия панели фильтра. Создаётся в конструкторе с ID `"{$this->id}_filterbutton"`. |
| **$operdialog** | `Dialog` | `public` | Диалоговое окно для запросов подтверждения. Создаётся в конструкторе с ID `"{$this->id}_operdialog"`. |
| **$dbtable** | `MySqlTable` | `public` | Таблица базы данных для основной сущности. Устанавливается через `dbtable()` или `bindtodb()`. |
| **$datarequest** | `callable/null` | `public` | Функция чтения записей из БД. По умолчанию — запрос к `$dbtable->all()`. Переопределяется через `datarequest()`. |
| **$saverequest** | `callable` | `public` | Функция сохранения записей. По умолчанию — вставка/обновление через `$dbtable->insert()/set()`. Переопределяется через `saverequest()`. |
| **$deleterequest** | `callable` | `public` | Функция удаления записей. По умолчанию — удаление через `$dbtable->del()`. Переопределяется через `deleterequest()`. |
| **$events** | `Events` | `public` | Объект событий для сервер-клиент коммуникации. Создаётся в конструкторе. |
| **$loadsubtablesrequest** | `callable/null` | `public` | Функция чтения подчинённых таблиц. Вызывается при открытии записи для загрузки данных подтаблиц. |
| **$savesubtablesrequest** | `callable/null` | `public` | Функция сохранения подчинённых таблиц. Вызывается после сохранения основной записи. |
| **$onsubtablesave** | `callable/null` | `public` | Хук после сохранения табличной части. Вызывается с параметрами `($args, $oldrows, $rows)`. |
| **$noselectalert** | `string` | `public` | Сообщение при попытке операции без выбранных строк. По умолчанию: `'Не выбрано ни одной строки в таблице...'`. |
| **$delquery** | `string` | `public` | Текст запроса подтверждения удаления. По умолчанию: `'Выбранные строки будут удалены. Вы уверены?'`. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Создаёт объекты `$table`, `$element`, `$filter`, `$operdialog`, `$events`, `$newbutton`, `$delbutton`, `$filterbutton`. Регистрирует обработчики событий `save` и `delete` с клиентской и серверной логикой. |
| **_id** | `mixed $name` | `string` | Возвращает ID объекта или сигнала. Если `$name` — объект — возвращает `$name->id`, иначе — `"{$this->id}_{$name}"`. Используется для генерации имён событий и сигналов. |
| **setmodesdefault** | — | `$this` | Устанавливает режимы Grid по умолчанию: `'default'` (список с фильтром), `'element'` (форма редактирования), `'filter'` (панель фильтра). Конфигурирует области `header`, `menu`, `content`, `bar` для desktop и mobile. Поддерживает fluent interface. |
| **setmodessubtable** | — | `$this` | Устанавливает режимы Grid для подчинённой таблицы. Упрощённая конфигурация без `menu` области. Конфигурирует области `bar`, `content`, `filter`, `element`. Поддерживает fluent interface. |
| **dbtable** | `MySqlTable $dbtable` | `$this` | Привязывает таблицу базы данных к компоненту. Сохраняет в `$dbtable`. Используется в `bindtodb()` и запросах. Поддерживает fluent interface. |
| **checksave** | `callable/string $f` | `$this` | Регистрирует хук-валидатор перед сохранением. Если строка — оборачивает в `function (args) { ... }`. Если callable — привязывает контекст через `bindTo()`. Добавляет обработчик `check{eventname}` в события формы. Поддерживает fluent interface. |
| **beforesave** | `callable/string $f` | `$this` | Регистрирует хук предобработки перед сохранением. Может модифицировать аргументы перед отправкой на сервер. Добавляет обработчик `before{eventname}` в события формы. Поддерживает fluent interface. |
| **onsave** | `callable/string $f` | `$this` | Регистрирует хук постобработки после сохранения. Получает результат сохранения. Добавляет обработчик `on{eventname}` в события формы. Поддерживает fluent interface. |
| **onsubtablesave** | `callable $f` | `$this` | Устанавливает хук после сохранения табличной части. Вызывается с параметрами `($args, $oldrows, $rows)`. Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **loadsubtablesrequest** | `callable $f` | `$this` | Устанавливает функцию загрузки подчинённых таблиц. Вызывается при открытии записи. Возвращает массив данных подтаблиц по ID. Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **savesubtablesrequest** | `callable $f` | `$this` | Устанавливает функцию сохранения подчинённых таблиц. Вызывается после сохранения основной записи. Обрабатывает массив подтаблиц из `$args['subtables']`. Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **datarequest** | `callable $f` | `$this` | Переопределяет функцию чтения записей. По умолчанию — `$dbtable->all($conditions)`. Может использовать кастомную логику (кэш, внешние API и др.). Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **saverequest** | `callable $f` | `$this` | Переопределяет функцию сохранения записей. По умолчанию — вставка/обновление через `$dbtable`. Должна возвращать `['result' => bool, 'data' => array]`. Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **deleterequest** | `callable $f` | `$this` | Переопределяет функцию удаления записей. По умолчанию — `$dbtable->del($id)`. Должна возвращать массив ID удалённых записей. Привязывает контекст через `bindTo()`. Поддерживает fluent interface. |
| **question** | `string $name`, `string $quest`, `string $onok`, `string $onno` | `$this` | Создаёт метод для запроса подтверждения. Генерирует клиентский метод с именем `$name`, который устанавливает обработчики `ondialogok`/`ondialogno`, устанавливает текст вопроса и открывает `$operdialog`. Поддерживает fluent interface. |
| **createtable** | `array $head`, `callable $func` | `$this` | Создаёт таблицу с данными из функции. Вызывает `$func([])` для получения записей, устанавливает `$datarequest`, добавляет поле `sel` для выбора, устанавливает заголовки и данные таблицы, отключает `$norows`, устанавливает сортировку. Поддерживает fluent interface. |
| **bindtodb** | `MySqlTable $dbtable`, `array $param = null` | `$this` | Привязывает компонент к таблице базы данных с полной конфигурацией. Параметры: `head` (заголовки), `sort` (сортировка), `fields` (поля выборки), `conditions` (условия), `norows` (отложенная загрузка), `datarequest` (функция чтения), `element` (форма), `inputs` (конфигурация полей), `cells` (раскладка формы), `filter` (конфигурация фильтра), `new`/`delete` (кнопки), `bar`/`content`/`bararea`/`contentarea`/`elementarea`/`filterarea` (области Grid). Автоматически генерирует форму из `$dbtable->generateinputs()` если `inputs` не указан. Возвращает `$this`. |
| **subtable** | `string $name = ''`, `MySqlTable $dbtable = null`, `array $param = null` | `DataView` | Создаёт подчинённую таблицу (табличную часть документа). Создаёт новый `DataView` с упрощёнными режимами через `setmodessubtable`. Устанавливает `fieldmap` для связи с родительской записью (`parent_{table}` → `id`). Регистрирует в `$this->subtables`. Автоматически создаёт `loadsubtablesrequest` и `savesubtablesrequest` если не установлены. Отключает стандартные события `delete` для подтаблицы. Возвращает объект подтаблицы. |
| **bar** | `string $areaName`, `...$extraButtons` | `$this` | Размещает кнопки в области панели. Добавляет `$filterbutton`, `$newbutton`, `$delbutton` в указанную область Grid. Добавляет дополнительные кнопки из `$extraButtons`. Поддерживает fluent interface. |
| **content** | `string $areaName` | `$this` | Размещает таблицу в области контента. Добавляет `$table` в указанную область Grid. Поддерживает fluent interface. |
| **elementgenerate** | `string $areaName`, `array $arr` | `$this` | Генерирует форму редактирования из конфигурации. Вызывает `$this->element->generate($arr)`, добавляет форму в указанную область Grid. Поддерживает fluent interface. |
| **elementcells** | `array $arr` | `Cells` | Организует поля формы в табличную структуру. Вызывает `$this->element->cells($arr)`. Возвращает объект `Cells`. Поддерживает fluent interface. |
| **filterassign** | `string $areaName`, `array $arr` | `$this` | Привязывает форму фильтра к таблице и размещает в области. Вызывает `$this->filter->assign($this->table, $arr)`, добавляет фильтр в указанную область Grid. Поддерживает fluent interface. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Если `$filter` — генерирует методы `filtershow()`/`filterhide()` для переключения режимов Grid. Если `$element` — генерирует методы `elementshow()`/`elementhide()`, регистрирует сигнал `RowClick` для открытия формы при клике на строку, добавляет обработчик закрытия формы. Для подтаблиц — генерирует методы `loadtable()`, `cleartable()`, `values()`, добавляет код сбора данных подтаблиц в `beforesave`. Генерирует методы `clearsubtables()`, `dialogrequest()`, устанавливает сообщения `$delquery` и `$noselectalert`. Добавляет `$events` если есть таблица или подтаблицы. Вызывает `parent::compile()`. |

## События (серверная часть)

| Событие | Параметры | Возвращает | Описание |
|---------|-----------|------------|----------|
| **save** | `array $args` с данными формы | `array ['result' => bool, 'data' => array, 'code' => int]` | Серверный обработчик сохранения. Вызывает `$saverequest`, обрабатывает подтаблицы через `$savesubtablesrequest`, возвращает данные для обновления таблицы. При `code == 1` — редирект на login, при `code == 1000` — на confirm. |
| **delete(rows)** | `array $args` с ключом `'rows'` (массив записей) | `array` (массив ID удалённых записей) | Серверный обработчик удаления. Вызывает `$deleterequest`, удаляет записи из подтаблиц через `$subtable->dbtable->delall()`. Возвращает массив объектов с `ltsDataId` для удаления из DataTable. |
| **loadsubtables(row)** | `array $args` с ключом `'row'` (данные строки) | `array` (данные подтаблиц по ID) | Серверный обработчик загрузки подтаблиц. Вызывает `$loadsubtablesrequest`, возвращает массив данных подтаблиц. Используется при открытии записи для загрузки связанных данных. |

## События (клиентская часть)

| Событие | Параметры | Описание |
|---------|-----------|----------|
| **save** | `object result` | Клиентский обработчик ответа сохранения. При успехе — обновляет строку в DataTable через `values()` или добавляет новую через `append()`, отправляет сигнал `ElementSave`, скрывает форму. При ошибке — показывает `alert(result.error)`. При `result.code == 1/1000` — редирект. |
| **delete(rows)** | `array/mixed result` | Клиентский обработчик ответа удаления. Если массив — удаляет строки через `DataTable.clear(result)`. Если `true` — удаляет `$this->selectedrows`. Отправляет сигнал `RowsDel`. |
| **loadsubtables(row)** | `object result` | Клиентский обработчик загрузки подтаблиц. Для каждой подтаблицы вызывает `LTS.get(id).loadtable(data)` и `mode('default')`. Отправляет сигнал `LoadSubtables`. |
| **{id}_ElementSave** | — | Сигнал отправляется после успешного сохранения записи. Можно подписаться через `LTS.onSignal('{id}_ElementSave', handler)`. |
| **{id}_RowsDel** | — | Сигнал отправляется после удаления записей. Можно подписаться через `LTS.onSignal('{id}_RowsDel', handler)`. |
| **{id}_ElementShow** | — | Сигнал отправляется при открытии формы редактирования. |
| **{id}_ElementHide** | — | Сигнал отправляется при закрытии формы редактирования. |
| **{id}_FilterShow** | — | Сигнал отправляется при открытии панели фильтра. |
| **{id}_FilterHide** | — | Сигнал отправляется при закрытии панели фильтра. |
| **{id}_ElementNew** | — | Сигнал отправляется при создании новой записи (очистка формы). |
| **{id}_ElementLoad** | — | Сигнал отправляется при загрузке данных записи в форму. |
| **{id}_RowClick** | — | Сигнал отправляется при клике на строку таблицы. Загружает подтаблицы и открывает форму. |
| **{id}_CloseAll** | — | Сигнал отправляется перед открытием нового окна/формы для закрытия остальных. |
| **{id}_LoadSubtables** | — | Сигнал отправляется после загрузки подтаблиц. |

## Примечания
*   Класс расширяет `Grid`, поэтому наследует все методы адаптивной сеточной вёрстки (`setMode()`, `device()`, `areas()`, `rows()`, `columns()`, `area()` и др.).
*   Класс расширяет `Div`, `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов и хуки (`$check`, `$before`, `$on`).
*   **Три в одном:** DataView объединяет `DataTable` (список), `Form` (редактирование), `FilterForm` (фильтр) в единый компонент с общей логикой сохранения/удаления.
*   **Режимы Grid:** Методы `setmodesdefault()` и `setmodessubtable()` конфигурируют 3 режима отображения: `'default'` (список), `'element'` (форма), `'filter'` (панель фильтра) — переключение через `LTS({id}).mode()`.
*   **Подтаблицы:** Метод `subtable()` создаёт подчинённые DataView для табличных частей документов — данные загружаются/сохраняются автоматически при открытии/сохранении основной записи.
*   **Связь с БД:** Метод `bindtodb()` автоматически генерирует форму из `$dbtable->generateinputs()`, настраивает фильтр, таблицу, кнопки — минимальный код для CRUD-интерфейса.
*   **События сохранения:** Обработчик `save` на сервере вызывает `$saverequest`, затем `$savesubtablesrequest` — обеспечивает атомарное сохранение основной записи и подтаблиц.
*   **События удаления:** Обработчик `delete` удаляет записи из основной таблицы и всех подтаблиц через `fieldmap` и `conditions` — каскадное удаление.
*   **Загрузка подтаблиц:** При клике на строку — сигнал `RowClick` вызывает `loadsubtables()` — данные подтаблиц загружаются и отображаются в соответствующих областях Grid.
*   **Диалог подтверждения:** `$operdialog` используется для запросов подтверждения удаления — текст задаётся через `$delquery`, обработчики через `ondialogok`/`ondialogno`.
*   **Хуки сохранения:** Методы `checksave()`, `beforesave()`, `onsave()` регистрируют хуки жизненного цикла сохранения — валидация, предобработка, постобработка.
*   **Кнопки управления:** `$newbutton`, `$delbutton`, `$filterbutton` создаются автоматически — могут быть переопределены через параметры `bindtodb()`.
*   **Области Grid:** Методы `bar()`, `content()`, `elementgenerate()`, `filterassign()` размещают компоненты в областях Grid — контролируется через параметры `bararea`, `contentarea`, `elementarea`, `filterarea`.
*   **Сигналы:** DataView отправляет 11+ сигналов для интеграции с другими компонентами — можно подписаться через `LTS.onSignal()`.
*   **Данные формы:** При сохранении данные формы передаются в `$saverequest` через `$args` — включают все поля формы и подтаблицы в `subtables`.
*   **Обновление таблицы:** После сохранения — строка обновляется через `DataTable.values()` если есть `ltsDataId`, или добавляется через `append()` если новая запись.
*   **Выбор строк:** Поле `sel` добавляется автоматически ко всем записям — используется для множественного удаления через `$delbutton`.
*   **Фильтр:** `$filter` привязывается к `$table` через `assign()` — фильтрация применяется к строкам таблицы на клиенте.
*   **Подтаблицы в форме:** Данные подтаблиц собираются в `beforesave` через `LTS({subtableid}).values()` и передаются в `$args['subtables']`.
*   **Обработка ошибок:** При `result.code == 1` — редирект на `login.php`, при `code == 1000` — на `confirm.php`, иначе — `alert(result.error)`.
*   **Метод _id():** Унифицирует получение ID для событий и сигналов — принимает объект или строку.
*   **createtable():** Альтернатива `bindtodb()` для таблиц без БД — данные из `$func`, сортировка по всем полям.
*   **question():** Создаёт методы для диалоговых запросов — устанавливает текст и обработчики, открывает `$operdialog`.
*   **compile():** Генерирует клиентские методы `filtershow/hide`, `elementshow/hide`, `clearsubtables`, `dialogrequest`, собирает данные подтаблиц в `beforesave`.
*   **Регистрация событий:** `$events` добавляется в потомки только если есть таблица или подтаблицы — оптимизация для простых случаев.
*   **Подтаблицы без удаления:** Для подтаблиц отключаются стандартные события `delete` — удаление через `rowsdelete()` метод.
*   **fieldmap подтаблиц:** По умолчанию — `["parent_{table}" => 'id']` — связь через поле родителя в подтаблице.
*   **onsubtablesave:** Хук вызывается после сохранения всех строк подтаблицы — параметры `($args, $oldrows, $rows)` для сравнения изменений.
*   **loadsubtablesrequest:** Возвращает массив `{subtableId => data}` — данные распределяются по подтаблицам через `loadtable()`.
*   **savesubtablesrequest:** Обрабатывает `$args['subtables']` — каждая подтаблица сохраняет свои данные через `$subtable->saverequest`.
*   **elementcells():** Организует поля формы в сетку — передаёт имена полей в `Form::cells()`.
*   **filterassign():** Привязывает фильтр к таблице — поля фильтра соответствуют полям таблицы для автоматической фильтрации.
*   **noselectalert:** Сообщение при удалении без выбора — можно переопределить до компиляции.
*   **delquery:** Текст подтверждения удаления — можно переопределить до компиляции.
*   **Режимы для подтаблиц:** `setmodessubtable()` упрощает layout — только `bar` и `content` без `menu`.
*   **Переключение режимов:** Через `LTS({id}).mode('element')` — форма показывается в области `element`, список скрывается.
*   **Закрытие формы:** Кнопка `close` в форме автоматически регистрирует обработчик `elementhide()` при компиляции.
*   **Клик по строке:** `DataTable::rowclick($this->element)` — при клике данные строки загружаются в форму, открывается режим `element`.
*   **Создание новой:** `$newbutton` очищает форму, сбрасывает `ltsDataId`, очищает подтаблицы, отправляет сигнал `ElementNew`, открывает форму.
*   **Удаление:** `$delbutton` проверяет выбор через `values({sel: 1})`, открывает диалог подтверждения, вызывает `delete(rows)` при подтверждении.
*   **Данные подтаблиц в save:** Собираются через `subtablesjs` в `compile()` — сериализуются с префиксом `<JSON>` для передачи на сервер.
*   **Старые строки подтаблиц:** При сохранении — `$oldrows` загружаются через `$subtable->dbtable->all($conditions)` — для определения удалённых строк.
*   **Удалённые строки подтаблиц:** Вычисляются через `array_diff($oldids, $myids)` — удаляются через `$subtable->dbtable->delall()`.
*   **Контекст функций:** Все callback-функции привязываются через `bindTo($this, $this)` — внутри доступен `$this` DataView.
*   **Параметры bindtodb:** Массив `$param` контролирует все аспекты — `head`, `sort`, `fields`, `conditions`, `inputs`, `cells`, `filter`, `new`, `delete`, `bar`, `content` и области.
*   **Автогенерация inputs:** Если `inputs` не указан в `bindtodb()` — вызывается `$dbtable->generateinputs()` — поля генерируются из структуры БД.
*   **fields в bindtodb:** Если указан — используется в `FIELDS` условии, автоматически генерирует `$head` из списка полей с учётом `AS` алиасов.
*   **norows в bindtodb:** Если `true` — таблица создаётся без данных — загрузка через `datarequest` при открытии.
*   **element в bindtodb:** Если указан — используется кастомная форма, иначе — генерируется из `inputs`.
*   **filter в bindtodb:** Если `false` — `$filterbutton` отключается, иначе — генерируется из массива конфигурации полей.
*   **bar в bindtodb:** Массив кнопок для панели — если не указан — стандартные `$filterbutton`, `$newbutton`, `$delbutton`.
*   **area() для подтаблиц:** Параметр `area` в `subtable()` размещает подтаблицу в указанной области Grid родителя.
*   **rowsave() для подтаблиц:** Метод генерируется для подтаблиц — сохраняет строку без серверного запроса, обновляет таблицу, отправляет сигнал.
*   **Кнопка save в подтаблице:** Регистрируется обработчик click — вызывает `rowsave(values)` вместо стандартного `save` события.
*   **delbutton в подтаблице:** Вызывает `rowsdelete(selectedrows)` — удаляет строки из таблицы без серверного запроса.
*   **rowsdelete() для подтаблиц:** Метод генерируется — очищает строки из DataTable, отправляет сигнал `RowsDel`.
*   **Сбор подтаблиц в save:** Через `subtablesjs` — для каждой подтаблицы вызывается `LTS.get(id).values()`, результат сериализуется в `args.set('subtables')`.
*   **clearsubtables():** Метод генерируется — очищает все подтаблицы через `cleartable()` и `mode('default')`.
*   **dialogrequest(mes):** Метод генерируется — устанавливает текст вопроса в `$operdialog` через `jQuery('#{id}_opermess').text(mes)`.
*   **Ошибки компиляции:** Если `$element` не указан — `$newbutton` устанавливается в `null` — кнопка создания не отображается.
*   **Ошибки компиляции:** Если `$filter` не указан — `$filterbutton` устанавливается в `null` — кнопка фильтра не отображается.
*   **Ошибки компиляции:** Если `$delbutton` пустой — события `delete` отключаются через `client/server(false)`.
