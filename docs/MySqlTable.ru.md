# Класс MySqlTable

## Назначение
Класс представления таблицы базы данных в ORM фреймворка LOTIS. Предназначен для определения структуры таблицы, управления полями и индексами, генерации SQL-запросов для создания и модификации таблицы, а также выполнения CRUD-операций (select, insert, update, delete). Класс хранит метаданные о полях (тип, размер, nullable, default, enum-значения), связях с другими таблицами и индексах. Является основным рабочим объектом ORM — все операции с данными выполняются через экземпляр MySqlTable. Создаётся через метод `MySql::table()` и сохраняет ссылку на владельца-подключение.

### Основные возможности

* **Создание таблиц** в базе данных
* **Управление полями** таблицы
* **Работа с индексами**
* **Генерация форм** на основе структуры таблицы
* **Выполнение CRUD-операций**
* **Проверка целостности** данных

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$name** | `string` | `public` | Имя таблицы в базе данных. Устанавливается в конструкторе. Используется для генерации SQL-запросов. |
| **$fields** | `array` | `public` | Ассоциативный массив объектов MySqlField по именам полей. Хранит структуру таблицы. Ключи — имена полей, значения — объекты MySqlField. |
| **$indexes** | `array` | `public` | Массив имён индексов таблицы. Заполняется при определении структуры через `index()` или при загрузке через `MySql::loadtable()`. |
| **$insert_id** | `int/null` | `public` | ID последней вставленной записи. Заполняется после вызова `insert()`. |
| **$str_sql_query** | `string` | `public` | Последний сгенерированный SQL-запрос. Заполняется при вызове методов `create()`, `insert()`, `all()`, `delall()`, `setall()`. |
| **$get_query** | `bool` | `public` | Флаг возврата SQL-запроса вместо выполнения. Если `true` — методы возвращают строку SQL, если `false` — выполняют запрос. Управляется методом `getQuery()`. |
| **$owner** | `MySql/null` | `public` | Ссылка на объект-владелец MySql. Устанавливается при создании через `MySql::table()`. Используется для выполнения запросов через подключение. |
| **$collation** | `string` | `public` | Кодировка сравнения (collation) для таблицы. По умолчанию пустая строка — используется кодировка подключения. |
| **$charset** | `string` | `public` | Кодировка символов для таблицы. По умолчанию пустая строка — используется кодировка подключения. |
| **$distinct** | `bool` | `public` | Флаг использования DISTINCT в SELECT-запросах. Управляется через QueryBuilder. |
| **$queryBuilder** | `QueryBuilder/null` | `public` | Кэшированный объект QueryBuilder для сложных запросов. Создаётся лениво при первом вызове `with()` или `query()`. |
| **$asarray** | `bool` | `public` | Флаг формата возврата результатов. Если `true` — результаты возвращаются как ассоциативные массивы, если `false` — как объекты stdClass. |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $name` | — | Конструктор класса. Сохраняет имя таблицы, инициализирует `$fields` и `$indexes` пустыми массивами, `$get_query` в `false`, `$charset` и `$collation` в пустые строки, `$owner` в `null`. Не устанавливает соединение — требует вызова через владельца MySql. |

| **string** | `string $name`, `int $len = 100` | `$this` | Добавляет поле типа VARCHAR. Создаёт MySqlField с типом `'VARCHAR'` и указанной длиной. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **text** | `string $name` | `$this` | Добавляет поле типа TEXT. Создаёт MySqlField с типом `'TEXT'`. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **bool** | `string $name` | `$this` | Добавляет поле типа TINYINT для булевых значений. Создаёт MySqlField с типом `'TINYINT'`. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **date** | `string $name` | `$this` | Добавляет поле типа DATETIME. Создаёт MySqlField с типом `'DATETIME'`. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **float** | `string $name`, `int $dd = 15`, `int $dr = 2` | `$this` | Добавляет поле типа DECIMAL. Создаёт MySqlField с типом `'DECIMAL'` и параметрами `dd` (цифры до запятой) и `dr` (цифры после). Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **enum** | `string $name`, `array $values = null` | `$this` | Добавляет поле типа ENUM. Создаёт MySqlField с типом `'ENUM'` и массивом допустимых значений. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **file** | `string $name`, `string $ext`, `bool $dir = false` | `$this` | Добавляет поле для хранения файлов. Создаёт MySqlField с типом `'FILE'` и параметрами расширения и директории. Если `$dir === false` — создаёт дополнительное поле `{$name}_filedata` типа BLOB для хранения данных в БД. Если `$dir !== false` — создаёт директорию если не существует. Возвращает `$this` для поддержки fluent interface. |
| **point** | `string $name` | `$this` | Добавляет поле типа POINT для геопространственных данных. Создаёт MySqlField с типом `'POINT'`. Сохраняет в `$fields[$name]`. Возвращает `$this` для поддержки fluent interface. |
| **table** | `string $name`, `MySqlTable $table` | `$this` | Добавляет поле-ссылку на другую таблицу. Создаёт MySqlField с типом `'TABLE'` и ссылкой на объект таблицы. Используется для определения внешних ключей. Возвращает `$this` для поддержки fluent interface. |
| **parent** | `MySqlTable $table` | `$this` | Добавляет поле родительской связи. Создаёт MySqlField с именем `"parent_{$table->name}"`, типом `'TABLE'` и ссылкой на таблицу. Удобный метод для иерархических структур. Возвращает `$this` для поддержки fluent interface. |
| **index** | `string $name` | `$this` | Добавляет индекс по указанному полю. Проверяет наличие имени в `$indexes` и добавляет если отсутствует. Возвращает `$this` для поддержки fluent interface. |

### Методы структуры таблицы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **create** | — | `bool/string` | Создаёт таблицу в БД на основе определённых полей. Генерирует SQL `CREATE TABLE IF NOT EXISTS` с полями, индексами, PRIMARY KEY (`id`), ENGINE InnoDB. Если `$get_query === true` — возвращает SQL-строку. Иначе — выполняет запрос и возвращает `true` при успехе, `false` при ошибке. |
| **checkupdate** | — | `bool/string` | Проверяет соответствие структуры таблицы в БД и в коде. Если таблица не существует — вызывает `create()`. Если существует — сравнивает поля, типы, nullable, default, ENUM-значения, индексы, collation. Возвращает `true` если структура актуальна, SQL-строку `ALTER TABLE` если есть изменения, `false` при ошибке. |
| **update** | — | `bool` | Обновляет структуру таблицы в БД. Вызывает `checkupdate()`. Если `$get_query === true` — возвращает SQL-строку. Иначе — выполняет ALTER TABLE если есть изменения. Возвращает `true` при успехе, `false` при ошибке. |
| **field** | `string $name`, `mixed $val = null` | `MySqlField/null` | Возвращает объект поля по имени. Если `$val !== null` — устанавливает значение поля через `setvalue()` и возвращает объект. Если поле не найдено — возвращает `null`. |
| **integritycheck** | `mixed $rec` | `bool` | Проверяет целостность данных перед удалением записи. Вызывает `$owner->integritycheck($this, $rec)`. Возвращает `true` если ссылок нет, `false` если есть. |

### Методы записи данных

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **value** | `string $name`, `mixed $val = null` | `mixed/$this` | Универсальный метод получения/установки значения поля. Если `$val === null` — возвращает текущее значение поля. Если передано — устанавливает значение через `setvalue()` и возвращает `$this`. Если поле не найдено — возвращает `null` или `$this`. |
| **values** | `array $arr` | `$this` | Устанавливает значения нескольких полей одновременно. Перебирает массив и вызывает `value()` для каждой пары ключ-значение. Возвращает `$this` для поддержки fluent interface. |
| **freevalues** | — | `$this` | Сбрасывает флаги `$isset` всех полей в `false`. Очищает подготовленные значения для следующей операции вставки/обновления. Возвращает `$this` для поддержки fluent interface. |
| **insert** | — | `int` | Выполняет INSERT запрос с данными из полей. Собирает поля с `$isset === true` и `$value !== null`, генерирует SQL, выполняет через `$owner->prepare()`. Вызывает `freevalues()` после вставки. Возвращает ID вставленной записи через `$owner->lastInsertId()` или `0` при ошибке. |
| **set** | `int $id = null` | `int` | Универсальный метод вставки/обновления. Если `$id === null` или `empty` — вызывает `insert()`. Иначе — вызывает `setall(['id' => $id])`. Если `$get_query === true` — возвращает `true/false`, иначе — возвращает ID или `0`. |
| **setall** | `mixed $conditions = null`, `string $operator = 'AND'` | `bool` | Выполняет UPDATE запрос для записей, удовлетворяющих условиям. Собирает данные из полей с `$isset === true`, создаёт QueryBuilder, добавляет WHERE/ORDER/LIMIT условия, выполняет через `QueryBuilder::setall()`. Вызывает `freevalues()` после обновления. Возвращает `true` при успехе, `false` при ошибке. |


### Методы чтения данных

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **get** | `int $id` | `object/false` | Находит запись по ID. Возвращает один объект stdClass с данными записи или `false` если не найдено или ошибка подключения. Использует prepared statement с `WHERE id = ? LIMIT 1`. |
| **exists** | `int $id` | `bool` | Проверяет существование записи по ID. Выполняет `SELECT COUNT(*)` и возвращает `true` если запись найдена, `false` если нет или ошибка подключения. |
| **with** | `string $relation` | `$this` | Добавляет LEFT JOIN к связанной таблице. Находит поле типа `'TABLE'` по имени, создаёт QueryBuilder если не существует, добавляет JOIN по условию `{$this->name}.{$relation} = {$alias}.id`. Возвращает `$this` для поддержки fluent interface. |
| **all** | `mixed $name = null`, `mixed $val = null` | `array/false` | Выполняет SELECT запрос с опциональной фильтрацией. Если `$name === null` — возвращает все записи. Если строка — добавляет WHERE условие. Если массив — поддерживает ключи: `FIELDS`, `ORDER`, `GROUP`, `LIMIT`, `JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `WHERE`, агрегатные функции. Возвращает массив объектов/массивов или `false` при ошибке. |
| **getQuery** | — | `$this` | Устанавливает флаг `$get_query` в `true`. Последующие вызовы методов будут возвращать SQL-запросы вместо выполнения. Возвращает `$this` для поддержки fluent interface. |
| **query** | — | `QueryBuilder` | Создаёт и возвращает новый объект QueryBuilder для построения сложных SELECT-запросов с JOIN, WHERE, GROUP BY, ORDER BY, LIMIT. QueryBuilder использует текущее подключение `$owner`. |


### Методы удаления данных

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **del** | `int $id` | `bool` | Удаляет запись по ID. Вызывает `delall(['id' => (int) $id])`. Возвращает `true` при успехе, `false` при ошибке. |
| **delall** | `mixed $name = null`, `mixed $val = null` | `bool` | Выполняет DELETE запрос с опциональной фильтрацией. Поддерживает те же условия что и `all()`. Создаёт QueryBuilder, генерирует DELETE SQL, выполняет через prepared statement. Возвращает `true` при успехе, `false` при ошибке. |

## Примеры использования

```php
// Подключение к базе:
$db = LTS::MySql('mydb', 'localhost', 'user', 'pass');

// Определение таблицы:
$users = $db->table('users');
$users->string('name', 100);
$users->string('email', 255);
$users->enum('status', ['active' => 'Активен', 'blocked' => 'Заблокирован']);
$users->table('role_id', $roles);  // внешний ключ
$users->create();  // создать в БД

// Вставка:
$users->value('name', 'Иван')
    ->value('email', 'ivan@test.com')
    ->insert();

// Обновление:
$users->value('status', 'blocked')->set(5);  // по ID=5

// Массовое обновление:
$users->value('status', 'active')->setall(['role_id' => 1]);

// Выборка:
$list = $users->all(['status' => 'active', 'ORDER' => '-created_at', 'LIMIT' => 10]);

// Сложная выборка через QueryBuilder:
$result = $users->query()
    ->where('status', 'active')
    ->where('name', '$иван')  // LIKE '%иван%'
    ->leftJoin($roles, 'users.role_id = roles.id', 'roles')
    ->orderBy('-created_at')
    ->limit(10)
    ->all();
```

## Примечания
*   Класс создаётся через `MySql::table()` — не рекомендуется создавать экземпляры напрямую через `new MySqlTable()`.
*   Все методы добавления полей возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Поля-ссылки (тип `'TABLE'`) автоматически определяют внешние ключи по соглашению `{fieldname}_id` при загрузке через `MySql::loadtable()`.
*   Метод `insert()` автоматически экранирует значения через prepared statements — не требуется ручной вызов `real_escape_string()`.
*   Метод `del()` и `delall()` не проверяют целостность автоматически — требуется явный вызов `integritycheck()` перед удалением.
*   Данные для вставки/обновления накапливаются через `value()`/`values()` — один объект может использоваться для множественных операций.
*   Метод `query()` возвращает QueryBuilder для сложных запросов — простые запросы выполняются через `all()`/`get()`/`exists()`.
*   Формат возврата результатов (массив/объект) определяется флагом `$asarray` или `$owner->asarray`.
*   При загрузке структуры через `MySql::loadtable()` поля и индексы заполняются автоматически из `information_schema`.
*   ENUM-значения сохраняются в объекте MySqlField — могут использоваться для валидации и генерации UI через `generateinputs()`.
*   Метод `create()` полезен для развёртывания структуры БД из кода — не требует ручного создания таблиц через phpMyAdmin.
*   Индексы добавляются через `index()` — создаются неавтоматически, только по явному указанию разработчика.
*   Поле `id` считается первичным ключом по умолчанию — добавляется автоматически в `CREATE TABLE`, не требует явного объявления.
*   Метод `set()` требует передачи ID записи для обновления — обновляет только одну запись по первичному ключу.
*   Для массового обновления нужно использовать `setall()` с условиями WHERE или QueryBuilder.
*   Класс не реализует soft delete — удаление физическое через DELETE.
*   Для транзакционных операций рекомендуется оборачивать вызовы в `$owner->transaction()`.
*   Метод `get()` возвращает `false` если запись не найдена — не выбрасывает исключений.
*   При вставке через `insert()` поле `id` игнорируется — генерируется автоматически автоинкрементом.
*   Класс совместим с MySQL 5.7+ и MariaDB 10.0+.
*   Для полей DATETIME значения должны быть в формате `'Y-m-d H:i:s'` — автоматического преобразования нет.
*   Булевы поля хранятся как TINYINT — `true` = 1, `false` = 0.
*   Метод `checkupdate()` сравнивает текущую структуру в БД с определённой в коде — полезен для миграций.
*   Объекты MySqlTable кэшируются в `$owner->dbtables` — повторные вызовы `table()` возвращают тот же объект.
*   Для работы методов записи (insert/update/delete) требуется установленное соединение — вызывается автоматически.
*   Класс не реализует валидацию данных — ответственность на разработчике или UI-компонентах.
*   Метод `value()` может вызываться многократно — данные накапливаются до вызова `insert()`/`setall()`.
*   При обновлении через `setall()` передаются только поля с `$isset === true` из `$fields`.
*   Для сложных условий WHERE рекомендуется использовать QueryBuilder вместо массива условий в `all()`.
*   Метод `generateinputs()` автоматически определяет тип UI-поля на основе типа поля БД — TEXT → textarea, ENUM → select, TABLE → lookup и т.д.
*   Для полей FILE с `$dir !== false` файлы сохраняются в файловую систему, с `$dir === false` — в BLOB-поле в БД.
*   Метод `with()` кэширует QueryBuilder в `$queryBuilder` — повторные вызовы используют тот же объект.
*   Флаг `$get_query` полезен для отладки — позволяет просмотреть сгенерированный SQL перед выполнением.
*   При создании таблицы через `create()` используется ENGINE InnoDB — требуется для поддержки транзакций и внешних ключей.
*   Метод `update()` не удаляет поля из БД которые отсутствуют в коде — для этого нужно использовать прямой ALTER TABLE.
*   Для полей DECIMAL параметры `dd` и `dr` определяют общую точность и количество знаков после запятой.
*   Метод `exists()` эффективнее `get()` для проверки наличия — выполняет `COUNT(*)` вместо `SELECT *`.
*   При загрузке структуры через `loadtable()` поле `id` игнорируется — считается первичным ключом по умолчанию.
*   Индексы загружаются из `information_schema.STATISTICS` — первичный ключ (`PRIMARY`) не включается в список `$indexes`.
*   Метод `freevalues()` должен вызываться после вставки/обновления если объект используется повторно — предотвращает дублирование данных.
*   Для работы с несколькими БД нужно создавать отдельные экземпляры MySql — один объект = одно подключение.
*   Класс не реализует connection pooling — каждое создание экземпляра потенциально создаёт новое подключение.
*   Метод `all()` возвращает `false` при пустом результате — не пустой массив, требуется проверка `empty()`.
*   Для полей-ссылок тип `'TABLE'` преобразуется в `INT(11)` при генерации SQL — внешний ключ не создаётся автоматически на уровне БД.
*   Метод `checkupdate()` не изменяет структуру автоматически — возвращает SQL для ручного выполнения или вызова `update()`.
*   При генерации UI через `generateinputs()` поле `id` исключается — не включается в форму редактирования.
*   Для ENUM ассоциативные массивы преобразуются в список значений — ключи используются как значения в SQL.
*   Класс поддерживает загрузку структуры из существующей БД через `MySql::loadtable()` — поля создаются автоматически.
*   Метод `setall()` поддерживает специальные ключи условий: `WHERE`, `ORDER`, `LIMIT` — обрабатываются отдельно от полей.
*   Для массовых операций рекомендуется использовать транзакции через `$owner->transaction()` — обеспечивает атомарность.
*   При ошибке подключения методы возвращают `false` или `0` — требуется проверка перед использованием результата.
*   Класс не реализует кэширование результатов запросов — каждый вызов выполняет запрос к БД.
*   Для оптимизации больших выборок рекомендуется использовать `LIMIT` в методе `all()`.
*   Метод `query()` создаёт новый QueryBuilder каждый вызов — не кэшируется для независимости запросов.
