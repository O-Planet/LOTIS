# Класс Input

## Назначение
Класс представления элементов ввода форм фреймворка LOTIS, наследующий `Element`. Предназначен для создания всех типов HTML-форм элементов: текстовые поля, пароли, чекбоксы, радио-кнопки, выпадающие списки, текстовые области, кнопки и HTML5-типы ввода (email, tel, url, date, color, time, number, range). Класс автоматически генерирует связанные label-элементы для подписей полей, поддерживает установку и получение значений, валидацию через атрибуты (required, min, max, pattern) и клиентские методы для работы с данными формы. Все input-элементы интегрируются с системой форм Form через общий механизм доступа к значениям.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$form** | `Form/null` | `public` | Ссылка на родительскую форму. Устанавливается при добавлении Input в Form. Используется для доступа к данным формы через клиентские методы. |
| **$tagname** | `string` | `public` | Наследуется от `Element`. По умолчанию `'input'`. Изменяется методами `select()`, `textarea()`, `button()` на `'select'`, `'textarea'`, `'button'` соответственно. |
| **$classname** | `string` | `public` | Наследуется от `Element`. CSS-классы элемента, управляются методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$caption** | `string` | `public` | Наследуется от `Element`. Текстовое содержимое элемента. Для input используется как значение кнопки, для textarea — как текст области. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки, включая label), `'element'` (DOM-элемент), `'attr'` (HTML-атрибуты), `'options'` (опции). |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id` и для связи с клиентским объектом. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`, устанавливает `tagname` в `'input'`, атрибут `type` в `'text'`. |
| **setinput** | `string $type = null`, `string $name = ''`, `string $caption = ''` | `$this` | Универсальный метод настройки input-элемента. Устанавливает тип input через `attr('type')`, имя поля через `attr('name')`, создаёт label с подписью если `$caption` не пуст. Поддерживает fluent interface. |
| **value** | `mixed $val` | `$this` | Устанавливает значение элемента. Для `textarea` — устанавливает `$caption`. Для `select` — устанавливает атрибут `value` и помечает соответствующий `option` как `selected`. Для остальных типов — устанавливает атрибут `value`. Поддерживает fluent interface. |
| **select** | `string $name`, `string $caption = ''`, `mixed $values = null` | `$this` | Преобразует элемент в выпадающий список. Устанавливает `tagname` в `'select'`, атрибут `name`. Если `$caption` не пуст — создаёт label. Если `$values` передан — создаёт `option`-элементы для каждого значения (ключ = value, значение = caption). Добавляет JS-код для установки значения после загрузки DOM. Поддерживает fluent interface. |
| **textarea** | `string $name`, `string $caption = ''` | `$this` | Преобразует элемент в текстовую область. Устанавливает `tagname` в `'textarea'`, атрибут `name`, `rows` в `3`. Если `$caption` не пуст — создаёт label. Поддерживает fluent interface. |
| **button** | `string $name`, `string $caption = ''` | `$this` | Преобразует элемент в кнопку. Устанавливает `tagname` в `'button'`, вызывает `setinput('button', $name)`, устанавливает `$caption`. Поддерживает fluent interface. |
| **checkbox** | `string $name`, `string $caption = ''` | `$this` | Создаёт чекбокс. Вызывает `setinput('checkbox', $name, $caption)`. Поддерживает fluent interface. |
| **file** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле загрузки файла. Вызывает `setinput('file', $name, $caption)`. Поддерживает fluent interface. |
| **hidden** | `string $name`, `string $caption = ''` | `$this` | Создаёт скрытое поле. Вызывает `setinput('hidden', $name, $caption)`. Поддерживает fluent interface. |
| **image** | `string $name`, `string $caption = ''` | `$this` | Создаёт кнопку-изображение. Вызывает `setinput('image', $name, $caption)`. Поддерживает fluent interface. |
| **password** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле пароля. Вызывает `setinput('password', $name, $caption)`. Поддерживает fluent interface. |
| **radio** | `string $name`, `string $caption = ''` | `$this` | Создаёт радио-кнопку. Вызывает `setinput('radio', $name, $caption)`. Поддерживает fluent interface. |
| **reset** | `string $name`, `string $caption = ''` | `$this` | Создаёт кнопку сброса формы. Вызывает `setinput('reset', $name)`, устанавливает `$caption` как значение. Поддерживает fluent interface. |
| **submit** | `string $name`, `string $caption = ''` | `$this` | Создаёт кнопку отправки формы. Вызывает `setinput('submit', $name)`, устанавливает `$caption` как значение. Поддерживает fluent interface. |
| **text** | `string $name`, `string $caption = ''` | `$this` | Создаёт текстовое поле. Вызывает `setinput('text', $name, $caption)`. Поддерживает fluent interface. |
| **email** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле email (HTML5). Вызывает `setinput('email', $name, $caption)`. Поддерживает fluent interface. |
| **tel** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле телефона (HTML5). Вызывает `setinput('tel', $name, $caption)`. Поддерживает fluent interface. |
| **url** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле URL (HTML5). Вызывает `setinput('url', $name, $caption)`. Поддерживает fluent interface. |
| **date** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле даты (HTML5). Вызывает `setinput('date', $name, $caption)`. Поддерживает fluent interface. |
| **color** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле выбора цвета (HTML5). Вызывает `setinput('color', $name, $caption)`. Поддерживает fluent interface. |
| **time** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле времени (HTML5). Вызывает `setinput('time', $name, $caption)`. Поддерживает fluent interface. |
| **number** | `string $name`, `string $caption = ''` | `$this` | Создаёт поле числа (HTML5). Вызывает `setinput('number', $name, $caption)`. Поддерживает fluent interface. |
| **range** | `string $name`, `string $caption = ''`, `int $min = 1`, `int $max = 100` | `$this` | Создаёт ползунок диапазона (HTML5). Вызывает `setinput('range', $name, $caption)`, устанавливает атрибуты `min` и `max`. Поддерживает fluent interface. |
| **label** | `string $caption = null` | `Html/$this/false` | Управляет связанным label-элементом. Если `$caption === null` — возвращает существующий label или `false`. Если `$caption` передан — создаёт новый label с атрибутом `for` равным `$this->id` и текстом `$caption`, добавляет в потомки. Возвращает объект label. |
| **required** | `bool $r = true` | `$this` | Устанавливает или снимает атрибут `required`. Если `$r === true` — добавляет `required="required"`, иначе — удаляет через `removeattr()`. Поддерживает fluent interface. |
| **min** | `mixed $min` | `$this` | Устанавливает атрибут `min` для числовых полей и дат. Поддерживает fluent interface. |
| **max** | `mixed $max` | `$this` | Устанавливает атрибут `max` для числовых полей и дат. Поддерживает fluent interface. |
| **pattern** | `string $pattern` | `$this` | Устанавливает атрибут `pattern` для валидации по регулярному выражению. Поддерживает fluent interface. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Для типов `submit` и `reset` — компилирует метод `click()` с триггером события. Для остальных типов — компилирует метод `value(val)` который: (1) Находит ближайшую форму через `jQuery('#{$this->id}').closest('form')`. (2) Извлекает имя поля через `attr('name')`. (3) Вызывает `LTS.get(formId).value(name, val)` для установки/получения значения через форму. Вызывает `parent::compile()`. |
| **shine** | `Element $parent = null` | — | Рендеринг элемента. Для каждого потомка типа `html` с `tagname == 'label'` — вызывает `create()` для создания label в DOM, устанавливает `type` в `'none'` чтобы label не рендерился отдельно. Затем вызывает `parent::shine()` для рендеринга основного input-элемента. |

## Клиентские методы (генерируются при compile)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(input).click()** | — | — | Для кнопок `submit` и `reset` — триггерит событие click на элементе. Генерируется автоматически при компиляции. |
| **LTS(input).value(val)** | `mixed val` | `mixed` | Для всех остальных типов — получает или устанавливает значение поля через родительскую форму. Если форма найдена и имя поля определено — вызывает `LTS.get(formId).value(name, val)`. Иначе — возвращает пустую строку. |

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   Все методы создания типов input возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Метод `setinput()` является универсальным — специализированные методы (`text()`, `password()`, `checkbox()` и др.) являются удобными обёртками.
*   Label-элементы создаются автоматически при указании `$caption` — связываются с input через атрибут `for` равный `$this->id`.
*   Метод `value()` работает по-разному для разных типов элементов: для `textarea` устанавливает `$caption`, для `select` помечает `option` как `selected`, для остальных — атрибут `value`.
*   Метод `select()` автоматически создаёт `option`-элементы из массива `$values` — ключ массива становится значением, значение массива становится текстом опции.
*   Для `select` после рендеринга выполняется JS-код `jQuery('#{$this->id}').val(jQuery('#{$this->id}').attr('value'))` — обеспечивает корректную установку значения после загрузки DOM.
*   Метод `label()` позволяет управлять подписью после создания элемента — можно получить существующий label или создать новый.
*   Атрибуты валидации (`required`, `min`, `max`, `pattern`) поддерживаются нативно браузером — не требуют дополнительной клиентской логики.
*   Метод `compile()` генерирует разные клиентские методы для кнопок (`click()`) и полей ввода (`value()`) — в зависимости от типа input.
*   Клиентский метод `value()` интегрируется с формой — использует `LTS.get(formId).value()` для доступа к данным через форму, а не напрямую к полю.
*   Метод `shine()` обрабатывает label-элементы особым образом — создаёт их в DOM но устанавливает `type` в `'none'` чтобы избежать дублирования рендеринга.
*   HTML5-типы ввода (`email`, `tel`, `url`, `date`, `color`, `time`, `number`, `range`) поддерживаются полностью — браузер обеспечивает валидацию и UI.
*   Для `range` по умолчанию устанавливаются `min=1` и `max=100` — могут быть изменены через параметры метода.
*   Для `textarea` устанавливается `rows=3` по умолчанию — может быть изменено через `attr('rows', ...)`.
*   Скрытые поля через `hidden()` полезны для передачи ID записей и других служебных данных в форме.
*   Кнопки `submit` и `reset` имеют специальное поведение — триггерят стандартные события формы браузера.
*   Для кнопок `button` можно назначать обработчики click через Events или клиентский JS.
*   Чекбоксы и радио-кнопки требуют группировки по имени для корректной работы — несколько элементов с одинаковым `attr('name')`.
*   Поля `file` требуют特殊ной обработки при отправке формы — данные передаются через FormData.
*   Метод `required()` добавляет/удаляет атрибут `required` — браузер блокирует отправку формы если поле пустое.
*   Атрибут `pattern` принимает регулярное выражение JavaScript — валидация происходит на клиенте перед отправкой.
*   Для числовых полей `min` и `max` могут быть числами или строками с датами — в зависимости от типа input.
*   Класс не реализует серверную валидацию — ответственность на разработчике или форме Form.
*   При компиляции метод `value()` ищет ближайшую форму через `closest('form')` — работает только если Input находится внутри Form.
*   Для доступа к значению поля вне формы нужно использовать прямые jQuery-методы: `$(field).val()`.
*   Для создания групп радио-кночек нужно создавать несколько Input с одинаковым `attr('name')` но разными `attr('value')`.
*   Метод `select()` с `$values` в формате строки разбивает значения через `\LTS::explodestr()` — формат `"key1:caption1,key2:caption2"`.
*   Для полей `date`, `time`, `datetime-local` значения должны быть в формате ISO (`YYYY-MM-DD`, `HH:MM:SS`) для корректной работы в браузере.
*   Поле `color` возвращает значение в формате HEX (`#RRGGBB`) — может использоваться для настройки цветов интерфейса.
*   Метод `range()` создаёт ползунок — значение может быть получено/установлено через `value()` как число.
*   Для полей `email` и `url` браузер выполняет базовую валидацию формата — может быть дополнена через `pattern`.
*   Кнопки `submit` и `reset` не требуют явных обработчиков — работают через стандартное поведение формы браузера.
*   Метод `label()` возвращает `false` если label не существует и `$caption === null` — позволяет проверить наличие подписи.
*   При создании label через `label($caption)` атрибут `for` автоматически устанавливается в `$this->id` — обеспечивает связь с input.
*   Для `textarea` метод `value()` устанавливает `$caption` — содержимое отображается между тегами `<textarea>` и `</textarea>`.
*   Для `select` метод `value()` ищет `option` с matching `attr('value')` — добавляет атрибут `selected` найденному и удаляет у остальных.
*   Метод `setinput()` с пустым `$caption` не создаёт label — полезно для полей без видимых подписей.
*   Класс не реализует автозаполнение — для этого нужно использовать атрибут `autocomplete` через `attr()`.
*   Для полей `password` значение не отображается в браузере — но доступно через `value()` на клиенте и сервере.
*   Метод `compile()` для `submit`/`reset` генерирует `click()` метод — позволяет назначать обработчики через Events.
*   Класс поддерживает все стандартные HTML-атрибуты через `attr()` — placeholder, disabled, readonly, autofocus и др.
*   Для disabled полей нужно использовать `attr('disabled', 'disabled')` — браузер блокирует взаимодействие.
*   Метод `value()` для Input вне формы возвращает пустую строку — требуется проверка перед использованием.
*   Для массового создания полей формы рекомендуется использовать метод `Form::generate()` с массивом конфигураций.
*   Класс не реализует маски ввода — для этого нужно использовать клиентские библиотеки или `pattern`.
*   Для полей `number` браузер показывает стрелки увеличения/уменьшения — могут быть скрыты через CSS.
*   Метод `range()` с `$min` и `$max` устанавливает диапазон — значение за пределами диапазона корректируется браузером.
*   Для `file` полей значение не может быть установлено программно — только через выбор пользователя.
*   Класс совместим с системой локализации Lang — `$caption` может быть переведён через `$lang->say()`.
