# Класс Logger

## Назначение
Центральный компонент системы логирования фреймворка LOTIS. Отвечает за запись событий, ошибок и отладочной информации в файл и/или вывод их в браузер. Класс реализует перехват стандартных ошибок PHP и необработанных исключений, обеспечивая единый формат журналирования. Работает в статическом режиме (паттерн Singleton), не допускает создания экземпляров. Поведение логгера управляется через глобальные константы, определяемые до инициализации фреймворка.

### Основные возможности

* **Запись логов** в файл или браузер
* **Обработка ошибок** PHP
* **Обработка исключений**
* **Контекстное логирование**
* **Управление режимами** вывода

## Константы

| Константа | Значение | Описание |
|-----------|----------|----------|
| **LOG_DISABLED** | `0` | Режим полного отключения логирования. |
| **LOG_FILE** | `1` | Запись логов только в файл. |
| **LOG_SCREEN** | `2` | Вывод логов только в браузер (HTML). |
| **LOG_BOTH** | `3` | Дублирование логов в файл и в браузер. |

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$enabled** | `bool` | `private static` | Флаг активности логгера. Управляется константой `LOTIS_LOG_DISABLED`. |
| **$mode** | `int` | `private static` | Текущий режим вывода (см. константы). По умолчанию `LOG_SCREEN`. |
| **$logFile** | `string` | `private static` | Полный путь к файлу лога (`lotis.log`). |
| **$baseDir** | `string` | `private static` | Базовая директория для хранения файлов логов. |
| **$errors** | `array` | `private static` | Буфер накопленных записей лога в памяти текущей сессии. |

## Методы

### Инициализация

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **init** | — | `void` | Инициализация системы логирования. Вызывается автоматически при загрузке ядра. Проверяет константы конфигурации, создаёт директорию для логов, регистрирует обработчики ошибок PHP (`set_error_handler`) и исключений (`set_exception_handler`). |

### Основные методы записи

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **log** | `string $level`, `string $message`, `array $context = []` | `void` | Базовый метод записи лога. Формирует строку сообщения с временной меткой, уровнем, именем класса-Caller, файлом и строкой вызова. Записывает в файл и/или выводит в браузер в зависимости от режима. Сохраняет entry в массив `$errors`. |
| **error** | `string $message`, `array $context = []` | `void` | Обёртка для записи сообщений уровня `ERROR`. |
| **warning** | `string $message`, `array $context = []` | `void` | Обёртка для записи сообщений уровня `WARNING`. |
| **info** | `string $message`, `array $context = []` | `void` | Обёртка для записи сообщений уровня `INFO`. |
| **debug** | `string $message`, `array $context = []` | `void` | Обёртка для записи сообщений уровня `DEBUG`. |

### Получение информации

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **getErrors** | — | `array` | Возвращает массив всех записей лога, накопленных в памяти за текущий запрос. |
| **hasErrors** | — | `bool` | Возвращает `true`, если в буфере `$errors` есть записи. |

### Обработка ошибок

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **errorHandler** | `int $severity`, `string $message`, `string $file`, `int $line` | `void` | Статический callback для обработки ошибок PHP. Преобразует ошибку в запись лога уровня `ERROR` с контекстом severity. |
| **exceptionHandler** | `Throwable $exception` | `void` | Статический callback для обработки необработанных исключений. Логирует сообщение, стек trace и выводит информацию на экран в режиме отладки. |
| **getCaller** | — | `array` | Внутренний метод для анализа стека вызовов (`debug_backtrace`). Возвращает массив с ключами `class`, `file`, `line` первого не системного вызова (исключая `Logger` и `Quark`). |

## Настройка

Поведение класса управляется глобальными константами, которые должны быть определены **до** подключения ядра фреймворка:

*   **LOTIS_LOG_DISABLED** (`bool`): `true` полностью отключает логирование.
*   **LOTIS_LOG_MODE** (`int`): Устанавливает режим вывода (1 — файл, 2 — экран, 3 — оба). По умолчанию 2.
*   **LOTIS_LOG_DIR** (`string`): Путь к директории для файла `lotis.log`. По умолчанию `/log/` относительно корня фреймворка.

## Примеры использования

```php
// Запись ошибки
Logger::error('Произошла критическая ошибка', ['user_id' => 123]);

// Запись предупреждения
Logger::warning('Обнаружено подозрительное действие');

// Запись информации
Logger::info('Операция успешно завершена');

// Проверка наличия ошибок
if (Logger::hasErrors()) {
    // Обработка ошибок
    $errors = Logger::getErrors();
}
```

## Примечания
*   Класс запрещает создание экземпляров через `__construct`, `__clone`, `__wakeup` (объявлены private).
*   Вывод в браузер осуществляется через HTML-блоки с инлайн-стилями (красная рамка, моноширинный шрифт), видимые только при режиме `LOG_SCREEN` или `LOG_BOTH`.
*   Запись в файл происходит с эксклюзивной блокировкой (`LOCK_EX`) в режиме дозаписи (`FILE_APPEND`).
*   Метод `getCaller()` игнорирует вызовы из классов `LTS\Logger` и `LTS\Quark` для корректного определения места возникновения ошибки в пользовательском коде.
*   Логгирование можно полностью отключить через константу LOTIS_LOG_DISABLED
*   Режимы логгирования настраиваются через LOTIS_LOG_MODE
*   При работе в CLI-режиме вывод в браузер не производится
*   Все записи содержат временную метку, уровень и контекстную информацию
