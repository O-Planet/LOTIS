# Класс Columns

## Назначение
Класс колоночной вёрстки фреймворка LOTIS, наследующий `Div`. Предназначен для создания гибкой колоночной структуры с автоматическим управлением ячейками (cell). Позволяет добавлять контент в определённые колонки по номеру или имени, автоматически создавая недостающие столбцы при необходимости. Класс предоставляет удобный интерфейс для заполнения колонок как по одной, так и массово через массивы данных. Все колонки автоматически получают CSS-классы для стилизации и могут содержать любые дочерние объекты фреймворка.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$columnscount** | `int` | `public` | Счётчик созданных колонок. Увеличивается при создании каждой новой ячейки через метод `cell()`. Используется для определения следующей доступной позиции колонки. |
| **$tagname** | `string` | `public` | Наследуется от `Div`. По умолчанию `'div'`. Определяет HTML-тег элемента при рендеринге. |
| **$classname** | `string` | `public` | Наследуется от `Element`. Автоматически добавляется класс `'Columns'` в конструкторе. Управляется методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки-колонки), `'element'` (DOM-элемент), `'keys'` (ключи колонок для доступа по номеру). |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id`. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Div`, добавляет CSS-класс `'Columns'`, инициализирует `$columnscount` в `0`. |
| **cell** | `mixed $columnnumb = null` | `Div` | Возвращает или создаёт ячейку (колонку) по номеру или имени. Если `$columnnumb` не указан — создаёт следующую колонку. Если передано целое число — создаёт или возвращает колонку по номеру (1-based). Если передана строка — создаёт или возвращает именованную колонку. Автоматически создаёт промежуточные колонки если запрошенный номер больше текущего счётчика. Каждая колонка получает классы `'Column'` и `'Column{N}'` где N — номер или имя. |
| **add** | `mixed $name`, `mixed $value = null` | `$this` | Переопределённый метод добавления контента в колонки. Поддерживает три режима работы: (1) Если `$name` — массив или объект без `shineobject`, а `$value` не передан — заполняет несколько колонок из массива (ключи определяют номер колонки для ассоциативного массива или последовательные номера для индексного). (2) Если `$value` не передан — добавляет контент в следующую доступную колонку. (3) Если оба параметра переданы — добавляет `$value` в колонку с номером/именем `$name`. Для объектов вызывает `add()` у ячейки, для скаляров — устанавливает `$caption`. Поддерживает fluent interface. |

## Примечания
*   Класс расширяет `Div`, поэтому наследует все методы Flexbox (`flex()`, `row()`, `column()`, `content()`, `align()`, `gap()` и др.), управления потомками (`add()`, `del()`, `getchilds()`, `move()`) и стилизации (`css()`, `width()`, `height()`).
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   Все колонки автоматически получают CSS-класс `'Column'` для общей стилизации и `'Column{N}'` для индивидуальной (где N — номер или имя колонки).
*   Метод `cell()` поддерживает два типа адресации: числовую (1, 2, 3...) и именованную ('header', 'sidebar', 'content'...).
*   При запросе колонки с номером большим чем `$columnscount` — автоматически создаются все промежуточные колонки. Например, если создано 2 колонки, а запрошена 5-я — будут созданы колонки 3, 4 и 5.
*   Ключи колонок хранятся в секции `'keys'` хранилища `$storage` — это позволяет сопоставлять номера колонок с их фактическими ключами в массиве потомков.
*   Метод `add()` с массивом данных удобен для быстрого заполнения колонок — каждый элемент массива помещается в соответствующую колонку.
*   Для ассоциативных массивов ключи используются как номера колонок, для индексных — ключи преобразуются в 1-based номера.
*   Объекты без свойства `shineobject` (например, массивы конфигурации) обрабатываются как данные для заполнения, а не как дочерние элементы.
*   Все методы возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Колонки являются объектами `Div` — в них можно добавлять любые дочерние элементы через стандартный метод `add()`.
*   Для стилизации колонок рекомендуется использовать CSS-классы `'Column'` (общие стили) и `'Column{N}'` (индивидуальные стили для конкретной колонки).
*   Класс не имеет клиентской JavaScript-части — вся логика выполняется на сервере при генерации HTML.
*   Объекты Columns автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   При рендеринге все CSS-свойства, установленные через методы класса, добавляются в объект CSS с селектором `#{$this->id}`.
*   Дочерние элементы (колонки) добавляются через методы `cell()` и `add()` — порядок добавления определяет порядок отображения в DOM.
*   Класс не реализует собственные методы `shine()`, `compile()`, `childs()` — они наследуются от `Div` и `Quark` без изменений.
*   Для создания адаптивной колоночной структуры рекомендуется комбинировать с методами Flexbox: `$columns->flex()->row()->wrap()`.
*   Метод `cell()` возвращает объект `Div` — это позволяет сразу добавлять контент в колонку через цепочку: `$columns->cell(1)->add($element)`.
*   При заполнении из массива пустые значения всё равно создают колонки — это обеспечивает сохранение структуры даже при частичных данных.
*   Класс полезен для создания форм с несколькими колонками, панелей инструментов, сеток контента и других layout-задач требующих колоночной организации.
