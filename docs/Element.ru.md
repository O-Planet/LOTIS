# Класс Element

## Назначение
Базовый класс всех UI-элементов фреймворка LOTIS, наследующий `Quark`. Представляет собой обёртку над HTML-элементами с расширенными возможностями управления стилями, атрибутами, событиями и клиентской логикой. Является родителем для всех визуальных компонентов (Div, Span, Form, Button, Input и др.). Реализует механизм генерации метаданных HTML-разметки через метод `shine()`, поддерживает управление CSS-классами, инлайн-стилями, HTML-атрибутами, а также предоставляет методы для назначения обработчиков событий и сигналов. Класс включает систему хуков для внедрения пользовательской логики в методы объектов.

## Свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$tagname** | `string` | Имя HTML-тега, который будет создан методом `shine()`. По умолчанию `'div'`. Переопределяется в дочерних классах (например, `'span'`, `'button'`, `'input'`). |
| **$classname** | `string` | CSS-классы объекта. Поддерживает множественные классы через пробел. Управляется методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$caption** | `string` | Текстовое содержимое элемента. Отображается как текст внутри тега или как значение атрибута `value` для форм-элементов. |
| **$type** | `string` | Устанавливается в `'html'` в конструкторе. Определяет тип объекта для фильтрации и обработки. |
| **$hucksenable** | `bool` | Флаг активации системы хуков для предопределенных клиентских методов объекта. Если `true` — для всех методов генерируются хуки `check`, `before`, `on`. По умолчанию `false`. Для изменения флага используется метод `openhucks`|

### Наследуемые свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$id** | `string` | Наследуется от `Construct`. Уникальный идентификатор, используется как HTML-атрибут `id` и для связи с клиентским LTS-объектом в `lts.js`. |
| **$storage** | `array` | Наследуется от `Quark`. Хранилище содержит секции: `'childs'` (потомки), `'element'` (DOM-элемент), `'attr'` (HTML-атрибуты), `'options'` (опции), `'openedhucks'` (разрешённые хуки). |
| **$jscreate** | `JS/null` | Наследуется от `Quark`. Ссылка на JavaScript-объект для клиентской логики секции create. |
| **$space** | `Space` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Если ID начинается с `'.'` — извлекает класс из ID, устанавливает пустой ID и добавляет класс через `addclass()`. Иначе — использует ID как идентификатор. Устанавливает `$tagname` в `'div'`, `$type` в `'html'`. Вызывает родительский конструктор `Quark`. |

### Создание и настройка элемента

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **capt** | `string $capt` | `$this` | Устанавливает текстовое содержимое элемента (`$caption`). Поддерживает fluent interface. |
| **tag** | `string $tagname` | `$this` | Изменяет имя HTML-тега элемента. Позволяет динамически менять тип элемента до рендеринга. Поддерживает fluent interface. |
| **css** | `string $name = ''`, `mixed $value = null` | `CSS/$this` | Управление стилями элемента. Если переданы оба параметра — добавляет CSS-свойство через объект CSS. Если только имя — возвращает существующий объект CSS или создаёт новый. Селектор по умолчанию — `#{$this->id}`. |
| **gridarea** | `string $name` | `$this` | Устанавливает CSS-свойство `grid-area` для элемента. Используется при размещении объектов в Grid-контейнерах. Поддерживает fluent interface. |
| **display** | `string $name = 'block'` | `$this` | Устанавливает CSS-свойство `display`. По умолчанию `'block'`. Поддерживает fluent interface. |

### Работа с атрибутами

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **option** | `string $name`, `mixed $val = null` | `mixed/$this` | Управление опциями элемента. Если `$val` не передан — возвращает значение опции. Если передан — устанавливает значение в секции `'options'`. |
| **attr** | `string $name`, `mixed $val = null` | `mixed/$this` | Управление HTML-атрибутами. Если `$val` не передан — возвращает значение атрибута. Если передан — устанавливает атрибут в секции `'attr'`. Поддерживает fluent interface. |
| **removeattr** | `string $name` | `$this` | Удаляет HTML-атрибут по имени. Находит атрибут в секции `'attr'` и удаляет через `unset()`. Поддерживает fluent interface. |

### Размеры и позиционирование

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **left** | `mixed $l` | `$this` | Устанавливает CSS-свойство `left`. Поддерживает fluent interface. |
| **top** | `mixed $t` | `$this` | Устанавливает CSS-свойство `top`. Поддерживает fluent interface. |
| **width** | `mixed $w` | `$this` | Устанавливает CSS-свойство `width`. Поддерживает fluent interface. |
| **height** | `mixed $h` | `$this` | Устанавливает CSS-свойство `height`. Поддерживает fluent interface. |

### События и методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **click** | `string $body` | `$this` | Регистрирует обработчик события `click`. Является обёрткой над методом `on('click', $body)`. Поддерживает fluent interface. |
| **on** | `string $func`, `string $body` | `$this` | Регистрирует обработчик события. Генерирует JavaScript-код для триггера события через jQuery. Добавляет код в JS-объект с областью `'ready'`. Поддерживает fluent interface. |
| **openhucks** | `string $names = null` | `$this` | Активирует систему хуков для предопределенных клиентских методов объекта. Если `$names` не передан — включает хуки для всех предопределенных клиентских методов. Если передан — включает хуки только для указанных имён (список через запятую). Поддерживает fluent interface. |
| **method** | `string $func`, `string $body` | `$this` | Добавляет пользовательский метод на клиенте. Парсит сигнатуру функции (имя и параметры), генерирует JavaScript-код с поддержкой хуков `check`, `before`, `on`. Метод становится доступен через `LTS.get('{id}').{funcname}()` или через `LTS({var}).{funcname}()`, где var - PHP-переменная, ссылающаяся на текущий объект. |
| **compilemethod** | `string $func`, `string $body = null`, `JS $js = null` | `JS` | Внутренний метод компиляции предопределенных клиентских методов объекта с учётом системы хуков. Проверяет флаг `$hucksenable` и список `$openedhucks`. Если хуки включены — генерирует код с проверками и хуками. Иначе — генерирует простой метод без хуков. |
| **signal** | `string $name`, `string $handler`, `Element $sender = null` | `$this` | Подписывает объект на сигнал. Если `$sender` не указан — подписывается на сигнал с именем `$name`. Если указан — подписывается на сигнал `{$sender->id}_{$name}`. Генерирует JavaScript-код через `LTS.get('{id}').signal()`. Поддерживает fluent interface. |

### Работа с классами

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **addclass** | `string $classname`, `bool $toend = true` | `$this` | Добавляет CSS-классы к элементу. Разбивает строку на отдельные классы, проверяет наличие каждого через `hasclass()`, добавляет отсутствующие. Если `$toend == true` — добавляет в конец списка, иначе — в начало. Поддерживает множественные классы через пробел. |
| **hasclass** | `string $classname` | `bool` | Проверяет наличие CSS-класса у элемента. Разбивает `$classname` на массив и проверяет вхождение через `in_array()`. Возвращает `true`, если класс найден. |
| **removeclass** | `string $classname` | `$this` | Удаляет CSS-классы из элемента. Разбивает строку на отдельные классы, находит каждый в массиве `$classes` и удаляет через `unset()`. После удаления собирает массив обратно в строку. Поддерживает множественные классы через пробел. |

### Вспомогательные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **shine** | `Element $parent = null` | — | Генерация метаданных HTML-элемента. Заполняет секцию `'element'` хранилища `storage` данными для рендеринга: тип (`type`), атрибуты (`attr`), тег (`tagname`), ID (`id`), родитель (`parent`), классы (`class`), содержимое (`caption`). Атрибуты обрабатываются с экранированием через `htmlspecialchars()`. Булевы атрибуты (`true`/`false`) обрабатываются особым образом. |
| **_new** | `string $type`, `string $id = ''` | `Construct` | Фабричный метод создания дочернего объекта текущего типа. Вызывает глобальную функцию `\LTS::_new()` с передачей текущего объекта как родителя. |

## Пример использования

```php
// Создание и настройка базового элемента
$myelement = LTS::Element()
    ->tag('ul')
    ->width('300px')
    ->height('200px')
    ->addclass('my-class important');

// Добавление потомков
$myelement->add(LTS::Element()
        ->tag('li')
        ->capt('О нас')
        ->click(
<<<JS
          window.location = 'about.php';
JS
        )
    )
    ->add(LTS::Element()
        ->tag('li')
        ->capt('Помощь')
        ->click(
<<<JS
          window.location = 'help.php';
JS
        )
    );

// Установка стилей
$myelement->css()->add('background-color', '#ffffff')
    ->add('border', 'solid 1px #777777');

// Добавление метода
$myelement->method('show(param)',
<<<JS
    $(myelement).css('display', param ? 'block', 'none');
JS
);

// Подписка на сигнал и использование клиентского метода
$myelement->signal('hideAll',
<<<JS
    LTS(myelement).show(false);
JS
);
```

## Примечания
*   Класс расширяет `Quark`, поэтому поддерживает все методы управления потомками: `add()`, `get()`, `set()`, `del()`, `addin()`, `move()`, `getchilds()` и др.
*   Класс расширяет `Construct`, поэтому поддерживает все хуки жизненного цикла: `$check`, `$before`, `$on`, `$checkchilds`, `$beforechilds`, `$onchilds`.
*   Метод `css()` автоматически создаёт объект `CSS` с селектором `#{$this->id}` если он ещё не существует — это позволяет цепочкой добавлять стили к элементу.
*   Метод `signal()` использует клиентскую систему сигналов LOTIS через `LTS.onSignal()` и `LTS.signal()`.
*   Методы управления классами (`addclass`, `removeclass`, `hasclass`) работают со строкой `$classname` — множественные классы разделяются пробелом.
*   Метод `shine()` не создаёт DOM-элемент напрямую — он заполняет секцию `'element'` метаданными, которые затем обрабатываются классом `Space` при рендеринге HTML.
*   Атрибуты со значением `false` игнорируются при генерации HTML.
*   Атрибуты со значением `true` добавляются как булевы (без значения), например `disabled`, `readonly`, `checked`.
*   Все методы управления стилями, атрибутами и классами возвращают `$this` для поддержки fluent interface.
*   Дочерние классы (Div, Span, Button, Form и др.) переопределяют `$tagname` и добавляют специфичные методы, но наследуют всю базовую функциональность `Element`.
*   Метод `_new()` позволяет создавать дочерние объекты в контексте текущего родителя без явного указания пространства имён.
*   Объекты `Element` автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   Класс не реализует методы `compile()` и `childs()` — они наследуются от `Quark` без изменений.
*   При рендеринге атрибуты экранируются через `htmlspecialchars()` для защиты от XSS-атак.
