# Файл lotis.php

## Назначение
Точка входа и ядро фреймворка LOTIS. Инициализирует сессию, обрабатывает входящие POST-запросы для синхронизации глобальных переменных (`Vars`) и навигации между страницами. Определяет константы путей к ресурсам фреймворка, регистрирует автозагрузчик классов, инициализирует систему логирования. Содержит абстрактный класс `LTS` с фабричными методами для создания всех компонентов фреймворка и вспомогательными функциями для работы с данными.

## Константы

| Константа | Значение | Описание |
|-----------|----------|----------|
| **_LOTIS_INTERNET** | `false` | Флаг интернет-режима. По умолчанию отключён. |
| **_LOTIS_DIR** | `dirname(__DIR__) . '/newlotis/'` | Базовая директория фреймворка. |
| **_LOTIS_JSDIR** | `'/newlotis/JS/'` | Путь к директории JavaScript-файлов. |
| **_LOTIS_CSSDIR** | `'/newlotis/CSS/'` | Путь к директории CSS-файлов. |

## Глобальные переменные

| Переменная | Тип | Описание |
|------------|-----|----------|
| **$__ltsinputcontent** | `string` | Содержимое raw POST-данных из `php://input`. |
| **$__ltsdata** | `array` | Декодированные JSON-данные из POST-запроса. |
| **$__lts_classes** | `array` | Массив загруженных классов фреймворка для подключения CSS/JS. |

## Класс LTS

Абстрактный класс с статическими фабричными методами для создания компонентов и работы с данными.

### Примеры использования

```php
// Подключение фреймворка LOTIS
define('UPPER_DIR', dirname(__DIR__) . '/newlotis/');
include_once UPPER_DIR . 'lotis.php';

// Создание объектов приложения
$div = LTS::Div()->capt('Привет, мир!');

// Построение приложения
LTS::Space()->build($div);
```

## Примечания
*   **Сессия:** Вызывает `session_start()` в начале — требуется для работы `Vars` и хранения состояния.
*   **POST-обработка:** Обрабатывает `php://input` для JSON-запросов и `$_POST` для Events — синхронизирует `__ltsVars` с сессией.
*   **Навигация:** При наличии `__ltspageurl` в POST-данных — возвращает JSON с URL и завершает выполнение для клиентского редиректа.
*   **Автозагрузчик:** Регистрирует `spl_autoload_register` с поиском классов в текущей директории и `_LOTIS_DIR`.
*   **Логирование:** Вызывает `LTS\Logger::init()` после определения констант — инициализирует систему логирования.
*   **Глобальный массив:** `$__lts_classes` заполняется при автозагрузке — используется в `Space` для подключения CSS/JS файлов компонентов.
*   **Фабричные методы:** Все методы `LTS::Component()` используют `_new()` — создают объект и добавляют в родителя если указан.
*   **Родитель:** Параметр `$parent` может быть строкой (ID) или объектом — если объект, вызывается `$parent->add($el)`.
*   **ID:** Если `$id` не передан — генерируется автоматически в конструкторе класса через `Construct::$spaceid`.
*   **MySql:** Метод `MySql()` принимает 4 параметра подключения — возвращает объект `LTS\MySql`.
*   **Stock:** Метод `Stock()` принимает таблицу операций и опциональный маппинг полей — возвращает объект `LTS\Stock`.
*   **MultiTable:** Метод `MultiTable()` принимает таблицу заголовка — возвращает объект `LTS\MultiTable`.
*   **QueryBuilder:** Метод `QueryBuilder()` не принимает параметров — создаёт пустой построитель запросов.
*   **Константы:** Определяются через `define()` — доступны во всём приложении через `_LOTIS_*`.
*   **Пути:** `_LOTIS_DIR` вычисляется через `dirname(__DIR__)` — предполагает размещение в поддиректории `newlotis`.
*   **Фильтрация:** `inputrequest()` использует `FILTER_SANITIZE_SPECIAL_CHARS` по умолчанию — может быть переопределено.
*   **explodestr:** Поддерживает массивы и объекты — преобразует их в массив через `(array)` или `explodestr()`.
*   **Разделители:** Приоритет разделителей в `explodestr()`: `|` → `;` → `,` — для гибкости формата.
*   **log():** Записывает JSON через `file_put_contents()` — для отладки и логирования данных.
*   **scriptversion:** Добавляется к путям CSS/JS в `Space` — для инвалидации кэша браузера.
*   **meta/keywords/description:** Устанавливаются в статические свойства `Space` — используются при генерации `<head>`.
*   **Абстрактный класс:** `LTS` объявлен как `abstract` — не может быть инстанциирован напрямую.
*   **Статические методы:** Все методы `LTS` статические — вызываются через `LTS::method()`.
*   **Пространство имён:** Классы фреймворка в пространстве `LTS\` — автозагрузчик добавляет префикс.
*   **Поиск файлов:** Автозагрузчик ищет файлы в порядке: текущая директория, `_LOTIS_DIR`, с префиксом пути и без.
*   **Инициализация:** Файл должен быть подключён первым в точке входа приложения — до создания объектов.
*   **Сессия Vars:** `__ltsVars` из POST-данных сохраняется в `$_SESSION` — для синхронизации между запросами.
*   **Завершение:** При обработке навигации вызывается `exit()` — прекращает выполнение скрипта.
*   **JSON:** POST-данные декодируются через `json_decode()` с `true` — возвращают ассоциативные массивы.
*   **Logger:** Инициализируется после определения констант — учитывает пользовательские настройки логирования.
*   **Классы:** Массив `$__lts_classes` используется в `Space::__builddoc()` для подключения CSS/JS файлов.
*   **_new():** Проверяет тип родителя — если объект, добавляет через `add()`, если строка — использует как ID.
*   **Типы компонентов:** Все фабричные методы возвращают объекты соответствующих классов фреймворка.
*   **Гибкость:** Родитель может быть не указан — объект создаётся без добавления в иерархию.
*   **Соглашения:** Имена методов фабрики совпадают с именами классов — для единообразия API.
