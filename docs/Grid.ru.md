# Класс Grid

## Назначение
Класс адаптивной CSS Grid-вёрстки фреймворка LOTIS, наследующий `Div`. Предназначен для создания сложных сеточных макетов с поддержкой множественных режимов отображения для разных устройств (mobile, desktop и др.). Позволяет определять именованные области (areas), строки (rows), колонки (columns) и переключаться между конфигурациями макета на клиенте без перезагрузки страницы. Класс автоматически генерирует JavaScript-код для инициализации объекта `ltsGrid` (из файла `Grid.js`), управляет приоритетами устройств и хуками жизненного цикла. Поддерживает готовые шаблоны (cards, layout) для типовых сценариев вёрстки.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$modes** | `array` | `private` | Многомерный массив конфигураций режимов. Структура: `[mode][device] = ['areas' => [...], 'rows' => ..., 'columns' => ...]`. Хранит все определённые макеты для каждого режима и устройства. |
| **$currentDevice** | `string` | `private` | Текущий контекст устройства при настройке режима. По умолчанию `'desktop'`. Используется методами `device()`, `areas()`, `rows()`, `columns()` для записи конфигурации в правильную секцию `$modes`. |
| **$activeMode** | `string/null` | `private` | Имя текущего активного режима. Устанавливается методом `setMode()`. Все последующие вызовы `areas()`, `rows()`, `columns()` записываются в этот режим. |
| **$devices** | `array` | `private` | Ассоциативный массив функций определения устройств. Ключ — имя устройства, значение — JavaScript-функция для `matchMedia`. По умолчанию: `'mobile'` и `'desktop'` с порогом 768px. |
| **$priorityOrder** | `array` | `private` | Приоритетный порядок режимов. Массив вида `['mobile' => 0, 'desktop' => 1]`. Используется клиентским скриптом `ltsGrid` для определения, какой режим применять при совпадении нескольких условий. |
| **$defaultmode** | `string/null` | `private` | Имя режима по умолчанию. Если установлен — автоматически активируется при загрузке страницы через JS-код в области `'ready'`. |
| **$tagname** | `string` | `public` | Наследуется от `Div`. По умолчанию `'div'`. Определяет HTML-тег элемента при рендеринге. |
| **$classname** | `string` | `public` | Наследуется от `Element`. CSS-классы элемента, управляются методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки), `'element'` (DOM-элемент), `'attr'` (атрибуты). |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id` и для связи с клиентским объектом в `lts.js`. |

## Методы (серверная часть, PHP)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Div`, устанавливает `display: grid`, инициализирует массивы `$devices` с функциями определения mobile/desktop, вызывает `priority('mobile, desktop')`. |
| **deviceQuery** | `string $name`, `string $jsFunction` | `$this` | Регистрирует пользовательскую функцию определения устройства. `$jsFunction` — JavaScript-код функции, возвращающей `true`/`false`. Пример: `'() => window.matchMedia("(min-width: 1024px)").matches'`. Поддерживает fluent interface. |
| **priority** | `string $orderString` | `$this` | Устанавливает приоритетный порядок режимов. Принимает строку имён устройств через запятую. Заполняет `$priorityOrder` индексами в порядке следования. Используется клиентским скриптом `ltsGrid` для выбора активного режима. Поддерживает fluent interface. |
| **setMode** | `string $name` | `$this` | Устанавливает активный режим для последующей настройки. Все вызовы `areas()`, `rows()`, `columns()` до следующего `setMode()` будут записаны в этот режим. Поддерживает fluent interface. |
| **defaultMode** | `string $mode` | `$this` | Устанавливает режим по умолчанию. При загрузке страницы клиентский скрипт автоматически переключит сетку в этот режим через `LTS(grid).mode()`. Поддерживает fluent interface. |
| **device** | `string $device` | `$this` | Устанавливает текущий контекст устройства для записи конфигурации. Используется в цепочке: `setMode()->device()->areas()`. Поддерживает fluent interface. |
| **areas** | `array $areas` | `$this` | Определяет именованные области сетки для текущего режима и устройства. Принимает массив строк, где каждая строка — ряд областей (например, `["header header", "menu content", "footer footer"]`). Генерирует CSS `grid-template-areas`. Поддерживает fluent interface. |
| **rows** | `string $rows` | `$this` | Устанавливает размеры строк сетки. Принимает строку в формате CSS Grid (например, `"auto 1fr auto"`). Генерирует CSS `grid-template-rows`. Поддерживает fluent interface. |
| **columns** | `string $columns` | `$this` | Устанавливает размеры колонок сетки. Принимает строку в формате CSS Grid (например, `"200px 1fr"`). Генерирует CSS `grid-template-columns`. Поддерживает fluent interface. |
| **area** | `string $name` | `Div` | Возвращает или создаёт объект-обёртку для именованной области. Создаёт `Div` с ID `"{$this->id}_area_{$name}"`, классом `'GridWrapper'` и атрибутом `data-grid-area`. Позволяет добавлять элементы в конкретную область сетки через `$grid->area('content')->add($element)`. |
| **add** | `string $name`, `mixed $value = null` | `$this` | Переопределённый метод добавления потомка. Если `$value` — объект с методом `gridarea()` — автоматически назначает область через `$value->gridarea($name)`. Иначе — вызывает родительский `Div::add()`. Поддерживает fluent interface. |
| **to** | `string $name`, `...$elements` | `$this` | Добавляет несколько элементов в указанную область. Для каждого элемента вызывает `add($name, $element)`. Удобная обёртка для массового размещения. Поддерживает fluent interface. |
| **justify** | `string $justify` | `$this` | Устанавливает выравнивание элементов по горизонтальной оси (`justify-items`). Допустимые значения: `'start'`, `'end'`, `'center'`, `'stretch'`. Алиасы: `'beg'`, `'begin'` = `'start'`. Поддерживает fluent interface. |
| **flow** | `string $flow` | `$this` | Устанавливает автоматическое размещение элементов (`grid-auto-flow`). Допустимые значения: `'row'`, `'column'`, `'row dense'`, `'column dense'`. Поддерживает fluent interface. |
| **autoRows** | `string $size` | `$this` | Устанавливает размеры автоматических строк (`grid-auto-rows`). Пример: `'100px'`, `'minmax(50px, auto)'`. Поддерживает fluent interface. |
| **autoColumns** | `string $size` | `$this` | Устанавливает размеры автоматических колонок (`grid-auto-columns`). Пример: `'100px'`, `'minmax(50px, auto)'`. Поддерживает fluent interface. |
| **center** | `string $target = 'items'` | `$this` | Центрирование элементов или контейнера. `$target`: `'items'` — центрирует содержимое ячеек, `'container'` — центрирует всю сетку в родителе, `'both'` — оба варианта. Поддерживает fluent interface. |
| **centerInParent** | — | `$this` | Центрирует сетку в родительском контейнере через `margin: 0 auto`. Поддерживает fluent interface. |
| **cards** | `string $minWidth = '250px'` | `$this` | Готовый шаблон адаптивной сетки карточек. Устанавливает `grid-template-columns: repeat(auto-fit, minmax({$minWidth}, 1fr))` и `gap: 10px`. Автоматически перестраивается при изменении ширины экрана. Поддерживает fluent interface. |
| **layout** | — | `$this` | Готовый шаблон макета приложения. Создаёт режим `'default'` с областями: header, menu, content, bar. Строки: `"auto 1fr auto"`, колонки: `"200px 1fr"`, gap: `"10px"`. Поддерживает fluent interface. |
| **isrow** | — | `bool` | Проверяет, заполняется ли сетка по строкам. Возвращает `true`, если `grid-auto-flow` не установлен или содержит `'row'`. |
| **iscolumn** | — | `bool` | Проверяет, заполняется ли сетка по колонкам. Возвращает `true`, если `grid-auto-flow` содержит `'column'`. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Кодирует `$modes`, `$priorityOrder`, `$devices` в JSON. Вызывает `compilemethod()` для создания методов: `ltsGrid.init()`, `setMode()`, `deviceQuery()`, `mode()`, `priority()`, `grid()`, `check()`, `before()`, `on()`. Если установлен `$defaultmode` — добавляет JS-код активации режима в область `'ready'`. Вызывает `parent::compile()`. |

## Методы (клиентская часть, реализованы в Grid.js)

Эти методы становятся доступны на клиенте через `LTS.get('{id}')` после компиляции. Фактическая логика выполняется объектом `ltsGrid` из файла `Grid.js`.

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(grid).setMode(mode)** | `string $mode` | `this` | Переключает сетку в указанный режим на клиенте. Вызывает `ltsGrid.setMode()` с ID сетки и именем режима. Применяет конфигурацию `areas`, `rows`, `columns` для текущего устройства. Поддерживает цепочечные вызовы. |
| **LTS(grid).deviceQuery(device, func)** | `string $device`, `function $func` | `this` | Регистрирует пользовательскую функцию определения устройства на клиенте. `$func` — JavaScript-функция, возвращающая `true`/`false`. Обновляет массив `$devices` в клиентском хранилище конфигурации. Поддерживает цепочечные вызовы. |
| **LTS(grid).mode(name)** | `string $name` | `this` | Алиас для `setMode()`. Переключает режим по имени. Вызывает `ltsGrid.mode()` с ID сетки. Поддерживает цепочечные вызовы. |
| **LTS(grid).priority(order)** | `string $order` | `this` | Изменяет приоритетный порядок режимов на клиенте. Принимает строку имён устройств через запятую. Пересчитывает `$priorityOrder` и применяет новую логику выбора активного режима. Поддерживает цепочечные вызовы. |
| **LTS(grid).grid** | — | `object` | Возвращает внутренний объект сетки из `ltsGrid` для прямого доступа к свойствам и методам. Используется для отладки или расширенной настройки. |
| **LTS(grid).check(f)** | `function $f` | `this` | Регистрирует хук-валидатор для смены режима. `$f` — JavaScript-функция, принимающая данные режима. Если возвращает `false` — смена режима отменяется. Поддерживает цепочечные вызовы. |
| **LTS(grid).before(f)** | `function $f` | `this` | Регистрирует хук предобработки перед сменой режима. `$f` — JavaScript-функция, может модифицировать данные режима. Возвращаемое значение используется для применения. Поддерживает цепочечные вызовы. |
| **LTS(grid).on(f)** | `function $f` | `this` | Регистрирует хук постобработки после смены режима. `$f` — JavaScript-функция, вызывается после применения конфигурации. Может использоваться для дополнительной инициализации. Поддерживает цепочечные вызовы. |

## Примечания
*   Класс расширяет `Div`, поэтому наследует все методы Flexbox (`flex()`, `row()`, `column()`, `content()`, `align()`, `gap()` и др.), управления потомками (`add()`, `del()`, `getchilds()`, `move()`) и стилизации (`css()`, `width()`, `height()`).
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   **Зависимость от Grid.js:** Для корректной работы клиентских методов необходимо подключение файла `Grid.js`, который содержит реализацию объекта `ltsGrid`. PHP-класс `Grid` только генерирует конфигурацию и вызовы методов.
*   Все методы настройки сетки возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Режимы (modes) позволяют определять несколько конфигураций макета и переключаться между ними на клиенте без перезагрузки страницы.
*   Устройства (devices) определяются через JavaScript-функции `matchMedia` — по умолчанию `'mobile'` (≤768px) и `'desktop'` (>768px).
*   Приоритет (priority) определяет порядок проверки устройств — первое совпавшее условие активирует соответствующий режим.
*   Метод `area()` создаёт обёртку `Div` с атрибутом `data-grid-area` — это позволяет размещать элементы в именованных областях через CSS Grid.
*   Переопределённый метод `add()` автоматически назначает область через `gridarea()` если объект поддерживает этот метод — упрощает размещение элементов в сетке.
*   Метод `compile()` генерирует 8 клиентских методов через `compilemethod()` — все они становятся доступны через `LTS.get('{id}')` на клиенте.
*   Хуки `check()`, `before()`, `on()` для клиентских методов работают аналогично хукам событий `Events` — позволяют внедрять пользовательскую логику в процесс смены режима.
*   Готовые шаблоны `cards()` и `layout()` предоставляют типовые конфигурации для распространённых сценариев — адаптивные карточки и макет приложения.
*   Метод `defaultMode()` автоматически генерирует JS-код активации режима при загрузке — не требует явного вызова на клиенте.
*   Конфигурация режимов кодируется в JSON и передаётся клиенту при рендеринге — все данные доступны в `ltsGrid` без дополнительных запросов.
*   Объект `ltsGrid` автоматически отслеживает событие `resize` окна браузера и применяет соответствующий режим при изменении размера экрана (метод `handleResize()`).
*   Методы `isrow()` и `iscolumn()` проверяют значение `grid-auto-flow` через объект CSS — возвращают актуальное состояние после всех изменений.
*   Объекты Grid автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   При рендеринге все CSS-свойства, установленные через методы класса, добавляются в объект CSS с селектором `#{$this->id}`.
*   Дочерние элементы добавляются через методы `add()`, `to()`, `area()->add()` — порядок добавления определяет порядок отображения в DOM.
*   Класс не реализует собственные методы `shine()`, `childs()` — они наследуются от `Div` и `Quark` без изменений.
*   Для создания адаптивных макетов рекомендуется использовать цепочечные вызовы: `$grid->setMode('default')->device('desktop')->areas([...])->rows(...)->columns()->device('mobile')->areas([...])`.
*   Клиентские методы поддерживают хуки жизненного цикла — это позволяет валидировать, модифицировать и логировать смену режимов на стороне браузера.
*   При переключении режима `ltsGrid` автоматически скрывает области (`GridWrapper`), которые не входят в новую конфигурацию, и показывает активные.
