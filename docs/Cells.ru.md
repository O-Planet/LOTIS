# Класс Cells

## Назначение
Класс таблично-ячеечной вёрстки фреймворка LOTIS, наследующий `Div`. Предназначен для создания структурированных сеток с организацией по строкам и столбцам. В отличие от `Grid` (CSS Grid) и `Columns` (Flexbox), класс `Cells` реализует двумерную систему координат, где каждый элемент имеет адрес по строке и колонке. Автоматически создаёт строки (объекты `Columns`) и ячейки (объекты `Div`) при обращении, поддерживая динамическое добавление, вставку и управление контентом через массивы данных. Класс полезен для создания форм с табличной структурой, панелей инструментов, раскрывающихся списков и других интерфейсов требующих строгой сетки.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$rowscount** | `int` | `public` | Счётчик созданных строк. Увеличивается при создании каждой новой строки через методы `line()` или `insert()`. Используется для определения следующей доступной позиции строки. |

### Наследуемые свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id`. |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$tagname** | `string` | `public` | Наследуется от `Div`. По умолчанию `'div'`. Определяет HTML-тег элемента при рендеринге. |
| **$classname** | `string` | `public` | Наследуется от `Element`. Автоматически добавляется класс `'Cells'` в конструкторе. Управляется методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки-строки), `'element'` (DOM-элемент), `'attr'` (атрибуты). |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Div`, добавляет CSS-класс `'Cells'`, инициализирует `$rowscount` в `0`. |

### Управление строками

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **insert** | `int $numb` | `Columns` | Вставляет новую строку на указанную позицию. Если `$numb` больше текущего количества строк — вызывает `line($numb)`. Создаёт объект `Columns` с классами `'Row Row{numb}'`. Пересчитывает классы всех последующих строк (сдвигает номера на +1). Обновляет массив потомков `$storage['childs']` с учётом новой позиции. Увеличивает `$rowscount`. |
| **line** | `int $numb = null` | `Columns` | Возвращает или создаёт строку по номеру. Если `$numb` не указан — создаёт следующую строку. Если запрошенный номер больше текущего счётчика — автоматически создаёт все промежуточные строки. Каждая строка получает классы `'Row'` и `'Row{N}'` где N — номер строки. Возвращает объект `Columns` для добавления ячеек. |
| **cell** | `int $linenumb`, `int $columnnumb` | `Div` | Возвращает или создаёт ячейку по координатам (строка, колонка). Вызывает `line($linenumb)` для получения строки, затем `$row->cell($columnnumb)` для получения ячейки. Удобный метод для прямого доступа к ячейке без цепочки вызовов. |
| **add** | `mixed $name`, `mixed $value = null` | `$this` | Переопределённый метод добавления контента. Поддерживает три режима работы: (1) Если `$name` — массив массивов (записей) — создаёт строку для каждой записи через `line()->add($val)`. (2) Если `$name` — простой массив — добавляет все элементы в одну строку через `line()->add($name)`. (3) Иначе — вызывает родительский `Div::add()`. Автоматически определяет тип массива по структуре данных. Поддерживает fluent interface. |

### Примеры использования

```php
// Создание базовой таблицы
$cells = LTS::Cells();

// Добавление строк
$cells->line(1)->add('Первая строка');
$cells->line(2)->add('Вторая строка');

// Вставка строки на определенное место
$cells->insert(1); // Вставит новую первую строку

// Работа с ячейками
$cells->cell(1, 1)->capt('Ячейка 1-1');
$cells->cell(1, 2)->capt('Ячейка 1-2');

// Массовое добавление строк
$cells->add([
    ['Ячейка 1-1', 'Ячейка 1-2'],
    ['Ячейка 2-1', 'Ячейка 2-2']
]);

// Добавление элементов в ячейки
$cells->cell(1, 1)->add(LTS::Button()->capt('Кнопка')->click('alert("Меня нажали!")'));
$cells->cell(2, 1)->add(LTS::Div()->capt('Текст'));
```

## Примечания
*   Класс расширяет `Div`, поэтому наследует все методы Flexbox (`flex()`, `row()`, `column()`, `content()`, `align()`, `gap()` и др.), управления потомками (`add()`, `del()`, `getchilds()`, `move()`) и стилизации (`css()`, `width()`, `height()`).
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   Все строки автоматически получают CSS-класс `'Row'` для общей стилизации и `'Row{N}'` для индивидуальной (где N — номер строки).
*   Метод `insert()` пересчитывает классы всех последующих строк — это обеспечивает корректную нумерацию после вставки.
*   Метод `line()` автоматически создаёт промежуточные строки если запрошенный номер больше текущего счётчика — аналогично методу `cell()` в классе `Columns`.
*   Метод `cell()` является удобным сокращением для `$cells->line($row)->cell($col)` — позволяет обращаться к ячейке по двумерным координатам.
*   Метод `add()` с массивом данных автоматически определяет структуру: если массив содержит вложенные массивы — каждый вложенный массив становится отдельной строкой.
*   Для ассоциативных массивов вложенные элементы распределяются по колонкам внутри строки.
*   Все ячейки являются объектами `Div` — в них можно добавлять любые дочерние элементы через стандартный метод `add()`.
*   Для стилизации строк рекомендуется использовать CSS-классы `'Row'` (общие стили) и `'Row{N}'` (индивидуальные стили для конкретной строки).
*   Для стилизации ячеек используются классы `'Column'` и `'Column{N}'` — они назначаются автоматически через объект `Columns`.
*   Класс не имеет предопределенных клиентских методов: вся логика выполняется на сервере.
*   Класс не реализует собственные методы `shine()`, `compile()`, `childs()` — они наследуются от `Div` и `Quark` без изменений.
*   Для создания адаптивной структуры рекомендуется комбинировать с методами Flexbox: `$cells->flex()->column()->wrap()`.
*   Метод `line()` возвращает объект `Columns` — это позволяет сразу добавлять ячейки через цепочку: `$cells->line(1)->cell(2)->add($element)`.
*   При заполнении из массива пустые значения всё равно создают строки — это обеспечивает сохранение структуры даже при частичных данных.
*   Класс полезен для создания форм с несколькими строками и колонками, панелей инструментов, табличных раскладок и других layout-задач требующих двумерной организации.
*   В отличие от `Grid`, класс `Cells` не поддерживает именованные области и переключение режимов — это чисто серверная структура с фиксированной сеткой.
*   В отличие от `Columns`, класс `Cells` управляет двумя уровнями вложенности (строки → колонки) вместо одного (колонки).
*   Метод `insert()` полезен для динамического добавления строк в существующую структуру без пересоздания всего объекта.
*   При вставке строки через `insert()` все последующие строки сдвигаются вниз — их классы обновляются автоматически (Row3 → Row4 и т.д.).
*   Все методы возвращают `$this` или соответствующий объект (`Columns`, `Div`) для поддержки fluent interface и цепочечных вызовов.
