# Класс FilterForm

## Назначение
Класс формы фильтрации данных фреймворка LOTIS, наследующий `Form`. Предназначен для создания интерактивных панелей фильтрации, связанных с объектом `DataTable`. Позволяет пользователю фильтровать строки таблицы по значениям полей формы с автоматическим применением фильтра при изменении значений. Класс автоматически создаёт кнопки очистки полей и кнопки выбора значений из списка уникальных значений таблицы. Поддерживает отложенное применение фильтра (debounce 300ms) для оптимизации производительности при частых изменениях.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$datatable** | `DataTable/null` | `public` | Ссылка на объект DataTable для фильтрации. Устанавливается через метод `assign()`. Используется для применения фильтра к строкам таблицы. |
| **$tagname** | `string` | `public` | Наследуется от `Form`. Устанавливается в `'form'`. Определяет HTML-тег элемента при рендеринге. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор формы, используется для связи с клиентским объектом и DataTable. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки-поля), `'element'` (DOM-элемент), `'attr'` (HTML-атрибуты). |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |
| **$events** | `Events` | `public` | Наследуется от `Form`. Объект событий для сервер-клиент коммуникации. Создаётся в конструкторе формы. |

## Методы (серверная часть, PHP)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Form`, добавляет CSS-класс `'filterform'` через `addclass()`. |
| **assign** | `DataTable $datatable`, `mixed $objects`, `bool $auto = true` | `Cells` | Привязывает форму к DataTable и генерирует поля фильтрации. Принимает объект DataTable, массив конфигураций полей и флаг автоматической настройки. Если `$auto == true` — добавляет класс `'filterfield'` ко всем полям и атрибут `data-table` с ID таблицы. Возвращает объект `Cells` с организованной сеткой полей. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Создаёт метод `filter(values)` который: (1) Проверяет пустоту всех значений фильтра. (2) Если все пустые — вызывает `ltsDataTable.clearFilter()` для показа всех строк. (3) Если есть значения — вызывает `ltsDataTable.filter()` и `ltsDataTable.applyFilter()` для применения фильтра. Вызывает `parent::compile()` в конце. |

## Методы (клиентская часть, FilterForm.js)

Клиентская логика реализована в отдельном файле `FilterForm.js` и регистрируется через `$(document).ready()`.

| Событие/Метод | Параметры | Возвращает | Описание |
|---------------|-----------|------------|----------|
| **change на .filterfield** | `Event e` | — | Обработчик изменения значений полей фильтра. Использует debounce 300ms через `setTimeout`/`clearTimeout`. При изменении собирает все значения формы через `ltsForm.values()`, вызывает `LTS.get(formId).filter(values)` для применения фильтра к DataTable. |
| **click на .filterdelbutton** | `Event e` | — | Обработчик клика по кнопке очистки поля. Извлекает `data-form`, `data-field` из атрибута кнопки. Очищает значение поля через `ltsForm.value(formId, fieldName, '')`. Собирает все значения формы, вызывает `LTS.get(formId).filter(values)` для обновления фильтра. |
| **click на .filterlistbutton** | `Event e` | — | Обработчик клика по кнопке списка значений. Извлекает `data-table`, `data-form`, `data-field` из атрибута кнопки. Получает уникальные значения поля через `ltsDataTable.fieldvalues()`. Создаёт динамический dropdown-menu с позиционированием относительно кнопки. Добавляет обработчики для закрытия при клике вне и нажатии ESC. |
| **click на .dropdown-menu li a** | `Event e` | — | Обработчик выбора значения из dropdown. Извлекает `data-form`, `data-field`, `data-value` из атрибута ссылки. Находит input по имени в форме, устанавливает значение через `.val(value).trigger('change')`. Закрывает dropdown через `closeDropdown()`. |
| **bindOutsideClick()** | — | — | Вспомогательная функция. Регистрирует обработчик клика на `document` для закрытия dropdown при клике вне его границ. Проверяет через `is()` и `has()` что клик не по dropdown. Вызывает `closeDropdown()`. |
| **bindEscapeKey()** | — | — | Вспомогательная функция. Регистрирует обработчик нажатия клавиш на `document`. При нажатии `Escape` вызывает `closeDropdown()`. |
| **closeDropdown()** | — | — | Вспомогательная функция. Удаляет dropdown-menu из DOM через `.remove()`. Снимает временные обработчики через `$(document).off('click.dropdown-close')` и `$(document).off('keydown.dropdown-close')`. |

## Специальные классы CSS

| Класс | Описание |
|-------|----------|
| **`.filterform`** | Базовый класс формы фильтрации — добавляется автоматически в конструкторе. |
| **`.filterfield`** | Класс полей фильтрации — добавляется автоматически через `assign()` если `$auto == true`. |
| **`.filterdelbutton`** | Класс кнопки очистки поля — создаётся для каждого поля кроме кнопок. |
| **`.filterlistbutton`** | Класс кнопки списка значений — создаётся для полей с `addlist` в конфигурации. |
| **`.dropdown-menu`** | Класс выпадающего списка значений — создаётся динамически при клике на `.filterlistbutton`. |

## Специальные атрибуты данных

| Атрибут | Описание |
|---------|----------|
| **`data-form`** | ID формы фильтрации — добавляется к кнопкам `.filterdelbutton` и `.filterlistbutton`. |
| **`data-field`** | Имя поля формы — добавляется к кнопкам для идентификации поля для очистки/заполнения. |
| **`data-table`** | ID DataTable для фильтрации — добавляется к кнопкам для связи с таблицей. |
| **`data-value`** | Значение для установки — добавляется к элементам dropdown-menu. |

## Примечания
*   Класс расширяет `Form`, поэтому наследует все методы создания полей (`text()`, `select()`, `date()` и др.), организации (`cells()`, `grid()`) и работы с событиями (`event()`, `error()`).
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки (`$check`, `$before`, `$on` и др.).
*   **Debounce 300ms:** Изменения полей фильтра не применяются мгновенно — используется задержка 300ms через `setTimeout` для предотвращения частых перерисовок таблицы при быстром вводе.
*   **Кнопка очистки:** Для каждого поля (кроме кнопок) автоматически создаётся кнопка `.filterdelbutton` с иконкой ❌ — при клике очищает значение поля и обновляет фильтр.
*   **Кнопка списка:** Для полей с ключом `'addlist'` в конфигурации создаётся кнопка `.filterlistbutton` с иконкой 🔻 — при клике показывает dropdown с уникальными значениями из DataTable.
*   **Уникальные значения:** Метод `ltsDataTable.fieldvalues()` извлекает уникальные значения указанного поля из всех строк таблицы — используется для заполнения dropdown.
*   **Динамический dropdown:** Выпадающий список создаётся динамически при клике — позиционируется относительно кнопки через `offset().top` и `offset().left`.
*   **Закрытие dropdown:** Dropdown закрывается при клике вне его границ или нажатии клавиши Escape — обработчики регистрируются только после открытия.
*   **Пустой фильтр:** Если все значения фильтра пустые — вызывается `ltsDataTable.clearFilter()` для показа всех строк таблицы.
*   **Применение фильтра:** При наличии значений — вызывается `ltsDataTable.filter()` для фильтрации и `ltsDataTable.applyFilter()` для обновления отображения.
*   **Связь с DataTable:** Через метод `assign()` форма связывается с DataTable — фильтр применяется к строкам этой таблицы.
*   **Автоматическая настройка:** При `$auto == true` — все поля получают класс `'filterfield'` и атрибут `data-table` с ID таблицы.
*   **Организация полей:** Метод `assign()` возвращает объект `Cells` — поля организуются в табличную структуру с подписями и полями ввода в разных колонках.
*   **Конфигурация полей:** Принимает массив или JSON-строку с описаниями полей — аналогично `Form::generate()`.
*   **Ключ addlist:** Если в конфигурации поля есть ключ `'addlist'` — создаётся кнопка `.filterlistbutton` для выбора из списка.
*   **Обработчик change:** Регистрируется через `$(document).on()` — работает с динамически созданными полями.
*   **Сбор значений:** При изменении поля собирается вся форма через `ltsForm.values()` — фильтр применяется ко всем полям одновременно.
*   **Очистка поля:** При клике на `.filterdelbutton` — значение устанавливается в пустую строку, триггерится `change` для применения фильтра.
*   **Выбор из списка:** При клике на элемент dropdown — значение устанавливается в input, триггерится `change` для применения фильтра.
*   **Временные обработчики:** Обработчики для закрытия dropdown регистрируются с namespace `.dropdown-close` — снимаются при закрытии.
*   **Позиционирование dropdown:** Вычисляется динамически через `offset().top + outerHeight()` — появляется под кнопкой.
*   **Удаление dropdown:** При закрытии dropdown удаляется из DOM через `.remove()` — создаётся заново при следующем открытии.
*   **Проверка вне:** При клике вне dropdown проверяется через `is()` и `has()` — закрывается только если клик не по dropdown.
*   **Клавиша Escape:** Обработчик на `keydown` проверяет `e.key === 'Escape'` — закрывает dropdown.
*   **Метод filter():** Генерируется при компиляции через `compilemethod()` — доступен через `LTS.get('{id}').filter()`.
*   **Проверка пустоты:** Использует `Object.values().every()` для проверки что все значения пустые (null, undefined, пустая строка).
*   **clearFilter():** Показывает все строки таблицы — не удаляет данные, только меняет отображение через `.show()`.
*   **applyFilter():** Скрывает строки не прошедшие фильтр через `.toggle()` — сохраняет данные в памяти.
*   **fieldvalues():** Возвращает уникальные значения через `Set` — удаляет дубликаты автоматически.
*   **data-атрибуты:** Используются для связи кнопок с полями и таблицей — извлекаются через `.data()`.
*   **trigger('change'):** После установки значения из dropdown — триггерится событие change для применения фильтра.
*   **preventDefault():** Для ссылок в dropdown — предотвращает переход по href='#'.
*   **stopPropagation():** Для клика по кнопке списка — предотвращает немедленное закрытие dropdown.
*   **CSS классы:** `.filterform`, `.filterfield`, `.filterdelbutton`, `.filterlistbutton`, `.dropdown-menu` — для стилизации.
*   **Иконки:** ❌ для очистки, 🔻 для списка — используются UTF-символы в `capt()`.
*   **Шрина кнопки:** 60px для кнопок очистки и списка — устанавливается через `width()`.
*   **Владелец кнопки:** Устанавливается в ID ячейки `Cells` через `$listbutton->owner` — для позиционирования в сетке.
*   **Массив конфигурации:** Поддерживает те же ключи что и `Form::generate()` — `name`, `caption`, `type`, `addlist` и др.
*   **JSON строка:** Если `$objects` — строка — парсится через `json_decode()` в массив.
*   **Возврат Cells:** Метод `assign()` возвращает объект `Cells` — позволяет дополнительную настройку сетки полей.
*   **Колонки:** По умолчанию подписи в колонке 1, поля ввода в колонке 2 — контролируется через `cells()`.
*   **Класс filterfield:** Добавляется к полям для идентификации в обработчике change — через jQuery селектор.
*   **Атрибут data-table:** Добавляется к кнопкам для связи с DataTable — используется в `fieldvalues()`.
*   **Атрибут data-form:** Добавляется к кнопкам для идентификации формы — используется в `ltsForm.value()`.
*   **Атрибут data-field:** Добавляется к кнопкам для идентификации поля — используется для очистки/заполнения.
*   **Атрибут data-value:** Добавляется к элементам dropdown — содержит значение для установки в поле.
*   **Обработчик document:** Все обработчики регистрируются на `document` через `.on()` — работают с динамическими элементами.
*   **Namespace событий:** Обработчики закрытия используют namespace `.dropdown-close` — для селективного снятия.
*   **Проверка длины:** Для dropdown проверяется `$('#dropdown-menu').length` —是否存在.
*   **Сохранение display:** При скрытии wrapper сохраняется оригинальный `display` в `grid.wrapperDisplay` — для восстановления.
*   **Восстановление display:** При показе wrapper используется сохранённый `display` или `'block'` по умолчанию.
*   **Метод valuesCompare():** Используется в `ltsDataTable.filter()` — поддерживает сравнение дат, строк, чисел, RegExp.
*   **Пустая строка:** В `valuesCompare()` пустая строка всегда совпадает — показывает все строки для этого поля.
*   **RegExp:** Если значение фильтра — RegExp — используется `test()` для проверки.
*   **Даты:** Парсятся через `tryParseDate()` — поддерживаются форматы `DD.MM.YYYY` и `YYYY-MM-DD`.
*   **Подстроки:** Для строк используется `includes()` — проверка на вхождение подстроки.
*   **Числа:** Для чисел используется строгое сравнение `===`.
*   **По умолчанию:** Строгое сравнение `===` для остальных типов.
*   **tryParseDate():** Поддерживает множественные форматы дат — возвращает `Date` объект или `null`.
*   **filterAndApply():** Комбинирует фильтрацию и применение — проверяет пустоту фильтра.
*   **clearFilter():** Показывает все строки через `.show()` — не требует перезагрузки данных.
*   **Метод values():** Для DataTable — поддерживает фильтрацию и обновление строк.
*   **Метод findRows():** Для DataTable — возвращает jQuery-объект строк по фильтру.
*   **Метод Row():** Для DataTable — возвращает jQuery-объект одной строки по ID.
*   **Метод fieldvalues():** Для DataTable — возвращает уникальные значения поля.
*   **Связь компонентов:** FilterForm связывается с DataTable через `assign()` — фильтр применяется к этой таблице.
*   **Автоматическое применение:** При изменении любого поля — фильтр применяется ко всем полям одновременно.
*   **Оптимизация:** Debounce 300ms предотвращает частые перерисовки при быстром вводе.
*   **UX:** Кнопки очистки и списка улучшают пользовательский опыт — не требуется ручной ввод.
*   **Accessibility:** Кнопки имеют текстовые иконки — могут быть заменены на SVG для лучшей доступности.
*   **Мобильные:** Dropdown позиционируется относительно кнопки — работает на мобильных устройствах.
*   **Браузеры:** Требуется jQuery — для DOM-манипуляций и событий.
*   **Зависимости:** Требуется `lts.js`, `Form.js`, `DataTable.js` — для базовой функциональности.
