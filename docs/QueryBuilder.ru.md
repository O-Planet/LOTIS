# Класс QueryBuilder

## Назначение
Класс построения сложных SQL-запросов в ORM фреймворка LOTIS. Предназначен для программной генерации SELECT, UPDATE, DELETE-запросов с поддержкой JOIN, WHERE, GROUP BY, ORDER BY, LIMIT, агрегатных функций, UNION и подзапросов. Позволяет строить запросы через fluent interface без написания сырого SQL, обеспечивает автоматическое экранирование значений, защиту от SQL-инъекций через prepared statements и автоматическое добавление JOIN для связанных таблиц. Класс используется через метод `MySqlTable::query()` или `MySqlTable::all()` и выполняет запрос через владельца MySql.

### Основные возможности

* **Построение запросов** различных типов
* **Управление условиями** поиска
* **Работа с JOIN** и объединениями
* **Агрегатные функции**
* **Пагинация** и сортировка
* **Параметризация** запросов

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$table** | `MySqlTable` | `public` | Ссылка на таблицу-владелец. Устанавливается в конструкторе через `MySqlTable::query()`. Используется для определения имени таблицы и полей при генерации SQL. |
| **$owner** | `MySql` | `public` | Ссылка на объект подключения к базе данных. Устанавливается в конструкторе. Используется для выполнения подготовленных запросов через `prepare()` и `result()`. |
| **$fields** | `array` | `public` | Массив полей для выборки. По умолчанию `['*']`. Заполняется через метод `select()` или `fields()`. При генерации SQL поля обрабатываются через `processField()`. |
| **$where** | `array` | `public` | Массив WHERE-условий. Заполняется через методы `where()`, `andWhere()`, `orWhere()`, `addCondition()`. Условия комбинируются с операторами из `$operators`. |
| **$params** | `array` | `public` | Массив параметров для prepared statements. Заполняется автоматически при добавлении условий через `parseOperator()`, `esc()`. Используется при выполнении запроса через `$owner->prepare()`. |
| **$operators** | `array` | `public` | Массив логических операторов для WHERE-условий (`AND`, `OR`). По умолчанию `['AND']`. Заполняется при добавлении условий через `where()`, `addCondition()`. |
| **$order** | `array` | `public` | Массив ORDER BY cláusulas. Заполняется через метод `orderBy()`. Каждое условие хранится как строка `"field ASC"` или `"field DESC"`. |
| **$group** | `array` | `public` | Массив GROUP BY полей. Заполняется через метод `groupBy()`. Используется при генерации SQL для агрегатных запросов. |
| **$limit** | `string` | `public` | LIMIT-клаузула. Устанавливается через метод `limit()`. Может содержать одно число или два числа через запятую (`"10, 20"`). |
| **$aggregates** | `array` | `public` | Массив агрегатных функций. Заполняется через метод `aggregate()`. Поддерживаемые функции: SUM, MIN, MAX, AVG, STD, VARIANCE, GROUP_CONCAT, COUNT, DISTINCT, BIT_AND, BIT_OR, BIT_XOR, JSON_ARRAYAGG. |
| **$distinct** | `bool` | `public` | Флаг использования DISTINCT в SELECT. Управляется через метод `distinct()`. Игнорируется если есть агрегатные функции. |
| **$joins** | `array` | `public` | Массив JOIN-условий. Заполняется через методы `join()`, `leftJoin()`, `rightJoin()`, `with()`, `autoJoinForField()`. Каждый элемент содержит: `type`, `dbName`, `tableName`, `alias`, `on`. |
| **$union** | `array` | `public` | Массив UNION-запросов. Заполняется через методы `union()`, `unionAll()`. Каждый элемент содержит: `builder` (объект QueryBuilder), `type` (`UNION` или `UNION ALL`). |
| **$str_sql_query** | `string` | `public` | Последний сгенерированный SQL-запрос. Заполняется методом `getSQL()`. Используется для отладки и логирования. |
| **$get_query** | `bool` | `public` | Флаг возврата SQL-запроса вместо выполнения. Управляется через метод `getQuery()`. Если `true` — методы `all()`, `setall()` возвращают строку SQL. |
| **$asarray** | `bool` | `public` | Флаг формата возврата результатов. Если `true` — результаты возвращаются как ассоциативные массивы, если `false` — как объекты stdClass. |
| **$processedFields** | `array` | `private` | Кэш обработанных имён полей. Используется методом `processField()` для предотвращения повторной обработки одинаковых полей. |
| **$joinedTables** | `array` | `private` | Кэш автоматически добавленных JOIN-таблиц. Используется методом `autoJoinForField()` для предотвращения дублирования JOIN. |
| **$aggritems** | `static array` | `public static` | Статический массив поддерживаемых агрегатных функций. Используется для валидации типа агрегации в методе `aggregate()`. |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `MySqlTable $table`, `MySql $owner` | — | Конструктор класса. Сохраняет ссылки на таблицу и подключение, инициализирует `$get_query` в `false`. Не выполняет запросов — только подготовка к построению. |

### Определение полей

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **select** | `mixed $fields` | `$this` | Устанавливает список полей для выборки. Принимает строку (разделяется по запятой через `\LTS::explodestr()`) или массив. Поля обрабатываются через `processField()`. Поддерживает fluent interface. |
| **fields** | `mixed $fields` | `$this` | Алиас для `select()`. Устанавливает список полей для выборки. Поддерживает fluent interface. |

### Определение условий

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **where** | `mixed $conditions`, `string $operator = 'AND'` | `$this` | Добавляет WHERE-условия. Поддерживает: (1) Строку — добавляет как есть в скобках. (2) Массив — перебирает ключи-значения, вызывает `addCondition()`. Специальные ключи: `'WHERE'` — сырое условие, агрегатные функции — игнорируются. Поддерживает fluent interface. |
| **andWhere** | `mixed $conditions` | `$this` | Добавляет условия с оператором AND. Обёртка над `where($conditions, 'AND')`. Поддерживает fluent interface. |
| **orWhere** | `mixed $conditions` | `$this` | Добавляет условия с оператором OR. Обёртка над `where($conditions, 'OR')`. Поддерживает fluent interface. |
| **whereGroup** | `mixed $conditions`, `string $operator = 'AND'` | `$this` | Добавляет группу условий в скобках. Создаёт временный QueryBuilder, заполняет его условиями, оборачивает результат в скобки. Поддерживает callable (функция-колбэк) или массив условий. Поддерживает fluent interface. |
| **addCondition** | `string $name`, `mixed $value`, `string $operator` | — | Внутренний метод добавления условия. Проверяет суффиксы `:left` (>=) и `:right` (<=) в имени поля. Для `:right` добавляет время `23:59:59` если не указано. Вызывает `addWhereCondition()`. |

### Соединение таблиц

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **with** | `string $relation` | `$this` | Автоматически добавляет LEFT JOIN для связанной таблицы. Находит поле типа `'TABLE'` по имени `$relation`, определяет целевую таблицу и добавляет JOIN по условию `{$this->table->name}.{$relation} = {$alias}.id`. Предотвращает дублирование через `$joinedTables`. Поддерживает fluent interface. |
| **join** | `mixed $table`, `string $on`, `string $type = 'INNER'`, `string $alias = null` | `$this` | Добавляет JOIN-условие. `$table` может быть объектом MySqlTable или именем таблицы. `$on` — условие соединения. `$type` — тип JOIN (INNER, LEFT, RIGHT). `$alias` — псевдоним таблицы. Поддерживает fluent interface. |
| **leftJoin** | `mixed $table`, `string $on`, `string $alias = null` | `$this` | Добавляет LEFT JOIN. Обёртка над `join()` с типом `'LEFT'`. Поддерживает fluent interface. |
| **rightJoin** | `mixed $table`, `string $on`, `string $alias = null` | `$this` | Добавляет RIGHT JOIN. Обёртка над `join()` с типом `'RIGHT'`. Поддерживает fluent interface. |

### Порядок, группировка

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **orderBy** | `string $fields` | `$this` | Устанавливает ORDER BY. Принимает строку с полями через запятую. Префикс `-` перед полем означает DESC (например, `'-created_at'`). Поля обрабатываются через `processExpression()`. Поддерживает fluent interface. |
| **groupBy** | `string $fields` | `$this` | Устанавливает GROUP BY. Принимает строку с полями через запятую. Поля обрабатываются через `processExpression()`. Поддерживает fluent interface. |
| **aggregate** | `string $type`, `string $fields` | `$this` | Добавляет агрегатную функцию. `$type` — имя функции (SUM, MIN, MAX, COUNT, DISTINCT, AVG, STD, VARIANCE, GROUP_CONCAT, BIT_AND, BIT_OR, BIT_XOR, JSON_ARRAYAGG). `$fields` — поля через запятую. Поддерживает алиасы через `AS` (`SUM(total) AS total_sum`). Для `DISTINCT` генерирует `COUNT(DISTINCT field)`. Поддерживает fluent interface. |
| **distinct** | `bool $enable = true` | `$this` | Включает/выключает DISTINCT. Устанавливает `$distinct`. Игнорируется если есть агрегатные функции. Поддерживает fluent interface. |


### Вложенные запросы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **subquery** | `callable $callback` | `QueryBuilder` | Создаёт новый QueryBuilder для подзапроса. Вызывает `$callback($sub)` для настройки. Возвращает объект подзапроса для использования в `whereInSubquery()`, `whereEqualSubquery()`. |
| **whereInSubquery** | `string $field`, `callable $callback` | `$this` | Добавляет условие `field IN (subquery)`. Создаёт подзапрос через `subquery()`, оборачивает в `IN (...)`. Мерджит параметры подзапроса в `$params`. Поддерживает fluent interface. |
| **whereEqualSubquery** | `string $field`, `callable $callback` | `$this` | Добавляет условие `field = (subquery)`. Создаёт подзапрос через `subquery()`, оборачивает в `= (...)`. Мерджит параметры подзапроса в `$params`. Поддерживает fluent interface. |

### Ограничения

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **limit** | `mixed $limit` | `$this` | Устанавливает LIMIT. Принимает число или строку (`"10"`, `"10, 20"`). Сохраняет в `$limit`. Поддерживает fluent interface. |
| **first** | — | `mixed/false` | Возвращает первую запись. Устанавливает `LIMIT 1`, вызывает `all()`, возвращает первый элемент массива или `false`. Удобно для получения одной записи без `get()`. |

### Контроль

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **getQuery** | — | `$this` | Устанавливает флаг `$get_query` в `true`. Последующие вызовы `all()`, `setall()` будут возвращать SQL-строку вместо выполнения запроса. Поддерживает fluent interface. |

### Построение запросов

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **all** | — | `array/false` | Выполняет SELECT-запрос. Генерирует SQL через `getSQL('SELECT')`, получает параметры через `getParams()`, выполняет через `$owner->prepare()`, извлекает результаты через `$owner->result()`. Если `$get_query === true` — возвращает SQL-строку. Возвращает `false` при пустом результате или ошибке. |
| **setall** | `array $data` | `bool` | Выполняет UPDATE-запрос. Генерирует SQL через `getUpdateSQL()`, мерджит параметры SET и WHERE, выполняет через `$owner->prepare()`. Если `$get_query === true` — выводит параметры и возвращает SQL. Возвращает `true` при успехе, `false` при ошибке. |
| **getSQL** | `string $type = 'SELECT'` | `string` | Генерирует SQL-запрос. Поддерживает типы: `'SELECT'`, `'DELETE'`. Формирует: SELECT/DISTINCT/агрегаты/поля, FROM, JOIN, WHERE, GROUP BY, ORDER BY, LIMIT, UNION. Сохраняет в `$str_sql_query`. Возвращает строку SQL. |
| **getUpdateSQL** | `array $data` | `string` | Генерирует UPDATE-запрос. Формирует: UPDATE, JOIN, SET (с параметрами `?`), WHERE, ORDER BY, LIMIT. Не выполняет запрос — только генерация SQL. Используется методом `setall()`. |
| **getParams** | — | `array` | Возвращает массив параметров для prepared statement. Используется внешними методами для выполнения запроса. |

### Приватные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **addRangeCondition** | `string $field`, `string $from`, `string $to` | — | Приватный метод добавления диапазона. Генерирует условие `field >= ? AND field <= ?`, добавляет два параметра в `$params`. Используется для обработки `day:`, `period:` префиксов. |
| **processField** | `string $f` | `string` | Приватный метод обработки поля. Проверяет кэш `$processedFields`. Обрабатывает алиасы `AS`. Вызывает `processExpression()` для квалификации имён. Сохраняет результат в кэш. |
| **processExpression** | `string $expr` | `string` | Приватный метод обработки выражения. Автоматически добавляет JOIN для полей с точкой через `autoJoinForField()`. Добавляет обратные кавычки вокруг имён таблиц и полей. Квалифицирует имена полей через `{$this->table->name}.{$field}`. |
| **autoJoinForField** | `string $field` | — | Приватный метод автоматического JOIN. Если поле содержит точку (`table.field`) — находит поле типа `'TABLE'`, добавляет LEFT JOIN если ещё не добавлен. Сохраняет в `$joinedTables` для предотвращения дублирования. |
| **quoteQualifiedField** | `string $field` | `string` | Приватный метод квалификации поля. Если есть точка — возвращает `` `table`.`field` ``, иначе `` `field` ``. Используется при генерации WHERE-условий. |
| **addWhereCondition** | `string $field`, `string $op`, `mixed $value`, `string $operator` | — | Приватный метод добавления условия. Квалифицирует поле, вызывает `autoJoinForField()`. Проверяет префиксы: `@` (raw SQL), `day:` (диапазон дня), `period:` (диапазон дат). Иначе — вызывает `parseOperator()`. |
| **parseOperator** | `string $field`, `string $op`, `mixed $value` | `string/false` | Приватный метод парсинга оператора. Поддерживает: массивы (IN), двухсимвольные (`<=`, `>=`), односимвольные (`!`, `<`, `>`, `%`, `$`, `(`), пустую строку. Вызывает `esc()` для экранирования значений. Возвращает строку условия или `false`. |
| **esc** | `mixed $v`, `int $stripPrefix = 0` | `string` | Приватный метод экранирования. Если `$stripPrefix > 0` — удаляет префикс из строки. Добавляет значение в `$params`, возвращает `?` для prepared statement. |

## Специальные префиксы значений в WHERE-условиях

| Префикс | Описание | Пример |
|---------|----------|--------|
| **`!`** | Оператор `!=` (не равно) | `'status' => '!active'` → `status != ?` |
| **`<`** | Оператор `<` (меньше) | `'age' => '<18'` → `age < ?` |
| **`>`** | Оператор `>` (больше) | `'price' => '>100'` → `price > ?` |
| **`<=`** | Оператор `<=` (меньше или равно) | `'count' => '<=10'` → `count <= ?` |
| **`>=`** | Оператор `>=` (больше или равно) | `'count' => '>=10'` → `count >= ?` |
| **`%`** | Оператор `LIKE` (начинается с) | `'name' => '%John'` → `name LIKE ?` |
| **`$`** | Оператор `LIKE` (содержит) | `'name' => '$John'` → `name LIKE '%John%'` |
| **`(`** | Оператор `IN` (список) | `'id' => '(1,2,3)'` → `id IN (1,2,3)` |
| **`@`** | Raw SQL (без экранирования) | `'date' => '@NOW()'` → `date = NOW()` |
| **`day:`** | Диапазон дня | `'created' => 'day:2024-01-15'` → `created >= '2024-01-15 00:00:00' AND created <= '2024-01-15 23:59:59'` |
| **`period:`** | Диапазон дат | `'created' => 'period:2024-01-01,2024-01-31'` → диапазон между датами |
| **`:left`** | Суффикс для `>=` | `'date:left' => '2024-01-01'` → `date >= ?` |
| **`:right`** | Суффикс для `<=` | `'date:right' => '2024-01-31'` → `date <= ?` (автоматически добавляет `23:59:59`) |

## Примеры использования

```php
// Создание построителя
$query = new QueryBuilder($table, $db);

// Простой запрос
$result = $query->select(['id', 'name'])
    ->where(['active' => true])
    ->orderBy('name')
    ->limit(10)
    ->all();

// Запрос с JOIN
$result = $query->select(['users.name', 'roles.title'])
    ->join('roles', 'users.role_id = roles.id')
    ->where(['users.active' => true])
    ->all();
```

## Примечания
*   Класс создаётся через `MySqlTable::query()` — не рекомендуется создавать экземпляры напрямую через `new QueryBuilder()`.
*   Все методы построения запроса возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Метод `with()` автоматически определяет связанные таблицы по полям типа `'TABLE'` — не требуется вручную указывать JOIN для внешних ключей.
*   Автоматический JOIN через `autoJoinForField()` предотвращает дублирование — каждое соединение добавляется только один раз через кэш `$joinedTables`.
*   Подготовленные запросы через `prepare()` защищают от SQL-инъекций — все значения экранируются через `esc()` и передаются как параметры.
*   Флаг `$get_query` полезен для отладки — позволяет просмотреть сгенерированный SQL перед выполнением через `all()`, `setall()`.
*   Метод `getSQL()` поддерживает два типа запросов: `'SELECT'` (по умолчанию) и `'DELETE'` — для UPDATE используется `getUpdateSQL()`.
*   Агрегатные функции определяются через статический массив `$aggritems` — при добавлении неизвестной функции она игнорируется в `where()`.
*   Для DISTINCT в агрегатах используется специальный синтаксис: `aggregate('DISTINCT', 'field')` генерирует `COUNT(DISTINCT field)`.
*   Подзапросы через `subquery()` создают независимый QueryBuilder — параметры автоматически мерджатся в родительский запрос.
*   Метод `whereGroup()` позволяет создавать вложенные условия в скобках — полезно для сложной логики `(A AND B) OR (C AND D)`.
*   Префиксы значений (`!`, `<`, `>`, `%`, `$`, `@`, `day:`, `period:`) упрощают запись условий — не требуется вручную указывать операторы.
*   Raw SQL через префикс `@` отключает экранирование — использовать только для проверенных выражений (функции MySQL).
*   Диапазоны дат через `day:` и `period:` автоматически добавляют время `00:00:00` и `23:59:59` — удобно для фильтрации по датам.
*   Суффиксы `:left` и `:right` в именах полей преобразуются в `>=` и `<=` — удобно для диапазонов без явного указания операторов.
*   Метод `first()` устанавливает `LIMIT 1` автоматически — удобно для получения одной записи без цепочки `limit(1)->all()`.
*   Параметры запроса хранятся в `$params` — используются при выполнении через `$owner->prepare($sql, $params)`.
*   Кэш `$processedFields` ускоряет повторную обработку полей — каждое поле обрабатывается только один раз за запрос.
*   Метод `processExpression()` добавляет обратные кавычки вокруг имён — защита от конфликтов с зарезервированными словами MySQL.
*   Для полей с алиасами (`field AS alias`) метод `processField()` корректно обрабатывает обе части — алиас не квалифицируется именем таблицы.
*   UNION-запросы через `union()` и `unionAll()` поддерживают цепочку — можно добавить несколько UNION в один запрос.
*   Метод `setall()` мерджит параметры SET и WHERE в один массив — порядок важен для prepared statement.
*   При ошибке выполнения методы возвращают `false` — требуется проверка перед использованием результата.
*   Класс не реализует кэширование результатов — каждый вызов `all()` выполняет запрос к БД.
*   Для оптимизации больших выборок рекомендуется использовать `limit()` и `orderBy()` — предотвращает загрузку всех записей в память.
*   Метод `getUpdateSQL()` не выполняет запрос — только генерация SQL для отладки или ручного выполнения.
*   Флаг `$asarray` управляет форматом возврата — может быть установлен глобально через `$owner->asarray` или локально в QueryBuilder.
*   При генерации DELETE с JOIN используется специальный синтаксис MySQL: `DELETE table FROM table AS table JOIN ...`.
*   Метод `join()` поддерживает объекты MySqlTable — автоматически извлекает имя таблицы и имя базы данных из объекта.
*   Для сложных условий рекомендуется использовать `whereGroup()` с callable — позволяет инкапсулировать логику группировки.
*   Класс совместим с MySQL 5.7+ и MariaDB 10.0+ — использует стандартный синтаксис SQL.
*   При работе с несколькими БД нужно передавать правильный `$owner` — каждый QueryBuilder привязан к одному подключению.
*   Метод `aggregate()` поддерживает алиасы через `AS` — удобно для именования результирующих колонок.
*   Для полей `*` в агрегатах автоматически подставляется `id` — `COUNT(*)` преобразуется в `COUNT(id)`.
*   Метод `orderBy()` поддерживает префикс `-` для DESC — не требуется явно указывать направление сортировки.
*   При генерации SQL метод `getSQL()` сохраняет результат в `$str_sql_query` — доступно для отладки после выполнения.
*   Класс не реализует транзакции — для атомарности операций нужно использовать `$owner->transaction()`.
*   Метод `all()` возвращает `false` при пустом результате — не пустой массив, требуется проверка `empty()` или `=== false`.
*   Для массовых обновлений рекомендуется использовать `setall()` с условиями WHERE — эффективнее множественных `update()`.
*   При использовании подзапросов параметры автоматически мерджатся — не требуется ручное управление `$params`.
*   Метод `quoteQualifiedField()` обеспечивает корректное экранирование имён — защита от SQL-инъекций через имена полей.
*   Класс не поддерживает именованные параметры — используются позиционные параметры `?` для prepared statements.
*   Для работы с JSON-полями MySQL можно использовать агрегат `JSON_ARRAYAGG` — поддерживается в `$aggritems`.
*   Метод `parseOperator()` для массивов генерирует `IN (?, ?, ?)` — количество плейсхолдеров соответствует размеру массива.
*   При генерации UPDATE через `setall()` порядок параметров: сначала SET, потом WHERE — важно для корректной привязки.
*   Класс не реализует soft delete — для удаления используется стандартный DELETE запрос.
*   Для отладки сложных запросов рекомендуется использовать `$query->getQuery()->all()` — возвращает SQL вместо выполнения.
