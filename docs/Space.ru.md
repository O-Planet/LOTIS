# Класс Space

## Назначение
Центральный класс рендеринга фреймворка LOTIS, наследующий `Element`. Отвечает за преобразование метаданных объектов в готовый HTML-документ или JSON-метаданные для клиентской сборки. Является точкой входа для финальной сборки приложения — координирует выполнение жизненных циклов всех объектов, распределяет элементы по типам (CSS, JS, HTML, VARS), формирует структуру DOM и генерирует итоговую страницу. Класс поддерживает два режима работы: серверный рендеринг полного HTML и клиентский рендеринг через передачу метаданных в браузер. Все объекты приложения должны быть созданы в контексте пространства Space для корректной регистрации и рендеринга.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$title** | `string` | `public static` | Заголовок страницы. Используется для генерации тега `<title>`. |
| **$keywords** | `string` | `public static` | Ключевые слова страницы. Генерирует мета-тег `<meta name="keywords">`. |
| **$description** | `string` | `public static` | Описание страницы. Генерирует мета-тег `<meta name="description">`. |
| **$meta** | `string` | `public static` | Дополнительные мета-теги. Вставляется непосредственно в `<head>`. |
| **$scriptversion** | `string` | `public static` | Версия скриптов. Добавляется как query-параметр `?version=...` к путям CSS/JS файлов для контроля кэширования. |
| **$jqueryui** | `bool` | `public` | Флаг подключения jQuery UI. Если `true` — в `<head>` добавляются ссылки на `jquery-ui.css` и `jquery-ui.js`. По умолчанию `true`. |
| **$ltsjs** | `bool` | `public` | Флаг подключения клиентского ядра `lts.js`. Если `true` — скрипт добавляется в `<head>`. По умолчанию `true`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Содержит секции для организации элементов перед рендерингом: `'elements'` (исходные метаданные), `'DOM_css'`, `'DOM_link'`, `'DOM_scripts'`, `'DOM_ready'`, `'DOM_eventsready'`, `'DOM_body'`, `'DOM_childs'`. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор пространства. Обычно не используется явно. |
| **$type** | `string` | `public` | Наследуется от `Construct`. Устанавливается в `'html'` через цепочку наследования `Element`. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`, инициализирует секцию `'elements'` пустым массивом в хранилище. |
| **html** | — | — | Точка входа для рендеринга HTML. Вызывает `build($this)` для запуска процесса сборки. |
| **build** | `Element $element` | `$this` | Основной метод сборки страницы. Последовательно вызывает: `compile()` для компиляции метаданных элемента, `create($this)` для создания объекта в пространстве, `Events::build()` для обработки входящих AJAX-запросов событий. Затем вызывает `DOMBuilder()` для генерации HTML и вывода в браузер. |
| **metadata** | `Element $element`, `bool $ret = true` | `array/null` | Извлечение метаданных без рендеринга HTML. Выполняет `compile()`, `create()`, `Events::build()`. Если `$ret == true` — возвращает массив элементов из секции `'elements'`. Если `false` — возвращает `null`. Используется для клиентского рендеринга через `clientbuild()`. |
| **DOMBuilder** | `array $elements = null` | — | Распределение элементов по типам для последующего рендеринга. Инициализирует секции хранилища (`'DOM_css'`, `'DOM_link'`, `'DOM_scripts'`, `'DOM_ready'`, `'DOM_eventsready'`, `'DOM_body'`, `'DOM_childs'`). Перебирает массив элементов и вызывает соответствующие методы `process*()` в зависимости от поля `'type'`. После обработки всех элементов вызывает `__builddoc()` для генерации финального HTML. |
| **processCSS** | `array $el` | — | Обработка CSS-элементов. Перебирает массив стилей `$el['style']`, формирует строку CSS-правил. Если ключ числовой — значение считается полным правилом, иначе — парой `свойство: значение`. Сохраняет результат в секцию `'DOM_css'` с селектором из `$el['class']`. |
| **processLink** | `array $el` | — | Обработка внешних ссылок (CSS/JS файлы). Генерирует тег `<link>` с атрибутами `href` и `rel`. Сохраняет в секцию `'DOM_link'`. |
| **processScript** | `array $el` | — | Обработка внешних скриптов. Генерирует тег `<script src="...">`. Поддерживает атрибуты `async` и `defer` — если присутствуют в `$el`, добавляются в тег. Сохраняет в секцию `'DOM_link'`. |
| **processFile** | `array $el` | — | Обработка файловых ресурсов. Проверяет существование файла по `$el['filename']`, читает содержимое через `file_get_contents()`. Формирует запись для секции `'DOM_body'` с поддержкой иерархии (parent-child). Если указан `$id` — регистрирует элемент по ID, иначе — добавляет как корневой. |
| **processVARS** | `array $el` | — | Обработка глобальных переменных. Генерирует JavaScript-код инициализации `LTS.vars("{id}", {body})` где `{body}` — JSON-представление хранилища переменных. Сохраняет в секцию `'DOM_scripts'`. |
| **processJS** | `array $el` | — | Обработка JavaScript-элементов. В зависимости от `$el['area']` (`'script'` или `'ready'`) и флага `$el['on']` формирует различный код: функции в `<head>`, обработчики событий jQuery для `ready`, анонимные скрипты. Если `$el['eventsready'] == true` — сохраняет в `'DOM_eventsready'`, иначе — в `'DOM_scripts'` или `'DOM_ready'`. |
| **processHTML** | `array $el` | — | Обработка HTML-элементов. Генерирует открывающий тег с атрибутами `id`, `class`, `attr`. Формирует запись для секции `'DOM_body'` с полями: `'tag'`, `'tagname'`, `'parent'`, `'childs'`, `'caption'`. Поддерживает иерархию — если указан `$parent` — добавляет `$id` в массив `'childs'` родительского элемента. |
| **processOther** | `array $el` | — | Обработка элементов неизвестного типа. Создаёт пустую запись в `'DOM_body'` для регистрации объекта в пространстве без визуального представления. Если не указан `$parent` — добавляет ID в `'DOM_childs'` как корневой элемент. |
| **__builddoc** | — | — | Генерация финального HTML-документа. Формирует структуру `<!DOCTYPE html><html><head>...<body>...</body></html>`. В `<head>` добавляет: мета-теги (charset, description, keywords, title), подключения jQuery/jQuery UI/lts.js, внешние CSS/JS файлы классов, инлайн-стили из `'DOM_css'`, скрипты из `'DOM_scripts'`, код инициализации `jQuery(document).ready()` с содержимым `'DOM_ready'` и `'DOM_eventsready'`. В `<body>` рекурсивно строит DOM-дерево через `__buildchild()` для элементов из `'DOM_childs'`. Выводит результат через `echo`. |
| **__buildchild** | `string $id` | `string` | Рекурсивная сборка дочерних элементов. Получает запись элемента из `'DOM_body'` по ID. Если есть `'body'` — возвращает как есть (для файлов). Иначе — формирует открывающий тег, содержимое (`'caption'`), рекурсивно обрабатывает всех детей из `'childs'`, закрывающий тег. Возвращает собранную строку HTML. |
| **clientbuild** | `Element $element` | — | Альтернативный режим сборки для клиентского рендеринга. Выполняет `compile()`, `create()`, `Events::build()`, извлекает метаданные через `get('elements')`. Генерирует минимальный HTML-каркас с подключениями jQuery, jQuery UI, lts.js, `lotisbuilder.js`. В `<head>` добавляет скрипт инициализации с глобальными переменными: `window.__LOTIS_METADATA__` (метаданные элементов), `window.__LOTIS_CONFIG__` (конфигурация страницы), `window.__lts_classes` (список классов). После `DOMContentLoaded` вызывает `LotisBuilder.build()` для клиентской сборки интерфейса. |

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`.
*   Статические свойства (`$title`, `$keywords`, `$description`, `$meta`, `$scriptversion`) управляются глобально — устанавливаются до вызова `build()`.
*   Метод `build()` является основной точкой входа для серверного рендеринга — после его вызова HTML выводится в браузер и выполнение скрипта завершается.
*   Метод `clientbuild()` используется для сценариев, где интерфейс строится на клиенте — сервер передаёт только метаданные в JSON-формате.
*   Метод `DOMBuilder()` автоматически вызывает `Events::build()` — это обеспечивает обработку входящих AJAX-запросов событий до рендеринга страницы.
*   Скрипты распределяются по трём секциям выполнения: `'DOM_scripts'` (в `<head>` до загрузки DOM), `'DOM_ready'` (после `DOMContentLoaded`), `'DOM_eventsready'` (обработчики событий с флагом `eventsready`).
*   Иерархия DOM-элементов строится через поля `'parent'` и `'childs'` — каждый элемент знает своего родителя и список детей по ID.
*   Метод `__buildchild()` рекурсивно обходит дерево элементов — порядок обхода соответствует порядку добавления в `'childs'`.
*   Подключение внешних ресурсов (jQuery, jQuery UI, lts.js) управляется флагами `$jqueryui` и `$ltsjs` — могут быть отключены при необходимости.
*   Версионирование скриптов через `$scriptversion` добавляет query-параметр ко всем CSS/JS файлам — полезно для инвалидации кэша браузера при обновлении.
*   Класс `Space` является синглтоном в контексте приложения — обычно создаётся один экземпляр через `LTS::Space()`.
*   Все объекты, созданные в контексте пространства, автоматически регистрируются в секции `'elements'` через метод `addinspace()`.
*   Метод `metadata()` может использоваться для предпросмотра или отладки — возвращает массив метаданных без вывода HTML.
*   При клиентском рендеринге через `clientbuild()` требуется наличие файла `lotisbuilder.js` — он отвечает за преобразование метаданных в DOM на стороне браузера.
*   Класс не реализует методы `shine()` и `compile()` самостоятельно — они наследуются от `Element` и `Quark` соответственно.
*   Обработка ошибок и исключений осуществляется через класс `Logger` — все критические ошибки логируются перед завершением выполнения.
*   Для корректной работы событий `Events::build()` должен вызываться до генерации HTML — это обеспечивается последовательностью в методе `build()`.
*   Глобальные переменные `$_SESSION` используются классом `Vars` для синхронизации состояния — Space не управляет сессией напрямую.
*   Все HTML-атрибуты и содержимое экранируются через `htmlspecialchars()` для защиты от XSS-атак.
*   Класс поддерживает работу в CLI-режиме — проверка `php_sapi_name()` используется в `Logger` для определения режима вывода.
