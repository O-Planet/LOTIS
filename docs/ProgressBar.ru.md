# Класс ProgressBar

## Назначение
Класс визуального индикатора прогресса фреймворка LOTIS, наследующий `Div`. Предназначен для отображения прогресса выполнения операций через виджет jQuery UI Progressbar. Поддерживает настройку минимального/максимального значения, текущего значения прогресса и текстовой подписи. Класс автоматически генерирует клиентский JavaScript-код для инициализации виджета и предоставляет методы для программного обновления значения прогресса, включения/выключения индикатора. Интегрируется с системой событий LOTIS для реакции на изменения состояния.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$tagname** | `string` | `public` | Наследуется от `Div`. По умолчанию `'div'`. Определяет HTML-тег элемента при рендеринге. |
| **$classname** | `string` | `public` | Наследуется от `Element`. Автоматически добавляется класс `'ProgressBar'` в конструкторе. Управляется методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки), `'element'` (DOM-элемент), `'options'` (параметры прогрессбара), `'text'` (текстовая подпись). |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'html'`. Определяет тип объекта для фильтрации и обработки. |
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id` и для связи с клиентским объектом. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Div`, добавляет CSS-класс `'ProgressBar'` через `addclass()`. |
| **value** | `mixed $val` | `$this` | Устанавливает текущее значение прогресса. Вызывает `option('value', $val)`. Сохраняет значение в секции `'options'`. Поддерживает fluent interface. |
| **min** | `mixed $val` | `$this` | Устанавливает минимальное значение прогресса. Вызывает `option('min', $val)`. Сохраняет в секции `'options'`. Поддерживает fluent interface. |
| **max** | `mixed $val` | `$this` | Устанавливает максимальное значение прогресса. Вызывает `option('max', $val)`. Сохраняет в секции `'options'`. Поддерживает fluent interface. |
| **text** | `string $val = ''` | `$this` | Устанавливает текстовую подпись прогрессбара. Инициализирует `options` через `option('value', 0)`, сохраняет текст в секции `'text'`. Если `$val` не передан — используется пустая строка. Поддерживает fluent interface. |
| **compile** | — | — | Генерация клиентских JavaScript-методов. Извлекает параметры из секции `'options'`, кодирует через `json_encode()`. Извлекает текст из секции `'text'`, генерирует код обработчика `progresschange` для обновления подписи. Создаёт JS-объект с методами: `value(val)`, `disable`, `enable`. Добавляет JS-код инициализации в область `'ready'`: создаёт jQuery UI progressbar, добавляет элемент `.progress-label` для отображения текста, регистрирует обработчик изменения прогресса. Вызывает `parent::compile()` в конце. |

## Клиентские методы (генерируются при compile)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **LTS(progressbar).value(val)** | `mixed val` | — | Обновляет значение прогресса на клиенте. Вызывает `jQuery('#{id}').progressbar('option', 'value', val)`. |
| **LTS(progressbar).disable()** | — | — | Отключает прогрессбар. Вызывает `jQuery('#{id}').progressbar('disable')`. |
| **LTS(progressbar).enable()** | — | — | Включает прогрессбар. Вызывает `jQuery('#{id}').progressbar('enable')`. |

## Примечания
*   Класс расширяет `Div`, поэтому наследует все методы Flexbox, управления потомками и стилизации.
*   Класс расширяет `Element`, `Quark`, `Construct`, поэтому поддерживает полный жизненный цикл объектов и хуки.
*   **Зависимость от jQuery UI:** Прогрессбар использует jQuery UI Progressbar — требуется подключение `jquery-ui.css` и `jquery-ui.js` в пространстве `Space`.
*   Все методы настройки возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Метод `text()` инициализирует `options` через `option('value', 0)` — обеспечивает корректную работу при установке текста до значения.
*   Текстовая подпись отображается через элемент `.progress-label` — позиционируется абсолютно по центру прогрессбара.
*   Обработчик `progresschange` обновляет текст подписи при изменении значения — показывает `{text}: {value}%` или `{value}%` если текст не задан.
*   Метод `compile()` генерирует 3 клиентских метода: `value()`, `disable()`, `enable()` — доступны через `LTS.get('{id}')`.
*   При компиляции параметры кодируются через `json_encode()` с `JSON_UNESCAPED_UNICODE` — кириллица в тексте сохраняется без экранирования.
*   Для кастомизации внешнего вида — использовать CSS-классы `.ProgressBar`, `.progress-label`, `.ui-progressbar`.
*   Значение прогресса должно быть в диапазоне `min`–`max` — по умолчанию `0`–`100`.
*   Для неопределённого прогресса — установить `value` в `false` (режим indeterminate jQuery UI).
*   Метод `disable()` блокирует взаимодействие — полезно при завершении операции.
*   Метод `enable()` разблокирует прогрессбар — для повторного использования.
*   Событие `progresschange` триггерится при каждом изменении значения — можно использовать для дополнительной логики.
*   Элемент `.progress-label` создаётся динамически при инициализации — добавляется через `appendTo()`.
*   Стили `.progress-label` задаются инлайн через JS — `position: absolute`, `top: 50%`, `left: 50%`, `transform: translate(-50%, -50%)`.
*   Шрифт подписи — `bold 12px Arial`, цвет — `#555` — может быть переопределён через CSS.
*   Для обновления значения на клиенте — использовать `LTS(progressbar).value(newValue)`.
*   При изменении значения через JS — обработчик `progresschange` обновляет текст автоматически.
*   Для сброса прогресса — установить `value(0)` — возвращает индикатор в начальное состояние.
