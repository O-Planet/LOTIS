# Класс JS

## Назначение
Класс управления клиентским JavaScript-кодом фреймворка LOTIS, наследующий `Quark`. Предназначен для серверной генерации JavaScript-кода, который выполняется в браузере. Позволяет встраивать скрипты непосредственно в HTML-документ или подключать внешние JS-файлы. Класс автоматически преобразует PHP-объекты в клиентские ссылки по их ID, обрабатывает специальные синтаксические конструкции (`LTS(var)`, `$(var)`) для доступа к объектам фреймворка, и управлению выполнения скриптов через области (areas). Все JS-объекты добавляются во вселенную (Space) и рендерятся в соответствующих местах HTML-документа.

## Свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$area** | `string` | Область выполнения скрипта. Допустимые значения: `'script'` (в `<head>`), `'ready'` (после загрузки DOM), `'create'` (при создании объекта). По умолчанию `'script'`. |
| **$compile** | `bool` | Флаг компиляции кода. Если `true` — обрабатываются специальные конструкции `LTS(var)` и `$(var)` для подстановки значений переменных. По умолчанию `true`. |

### Наследуемые свойства

| Свойство | Тип | Описание |
|----------|-----|----------|
| **$id** | `string` | Наследуется от `Construct`. Уникальный идентификатор, используется для связи серверного и клиентского объектов. |
| **$type** | `string` | Устанавливается в `'JS'` в конструкторе. Определяет тип объекта для фильтрации и обработки. |
| **$tagname** | `string` | Наследуется от `Quark`. Для JS-объектов не используется (пустая строка). |
| **$classname** | `string` | Наследуется от `Quark`. CSS-класс объекта, может использоваться для селекторов. |
| **$caption** | `string` | Наследуется от `Quark`. Для JS-объектов не используется. |
| **$storage** | `array` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (скрипты), `'element'` (DOM-элемент), `'localvars'` (локальные переменные для подстановки). |
| **$jscreate** | `JS/null` | Наследуется от `Quark`. Ссылка на JavaScript-объект для клиентской логики. |

## Методы

### Жизненный цикл

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Если `$id` равен `'script'`, `'ready'` или `'create'` — устанавливает `$area` в это значение и очищает ID. Иначе — использует `$id` как идентификатор, `$area` устанавливается в `'script'`. Вызывает родительский конструктор `Quark`, устанавливает `$type` в `'JS'`, `$compile` в `true`. |
| **compile** | — | — | Пустая реализация. Наследуется от `Construct`, переопределяется без логики — JS-объекты не требуют предварительной компиляции метаданных. |
| **shine** | `Element $parent = null` | — | Генерация JS-элементов. Перебирает потомков в секции `'childs'`. Если значение содержит `'.js'` — создаёт запись для тега `<script src="...">` (поддержка префикса `LTS:` для загрузки локальных файлов). Иначе — формирует массив инлайн-скриптов с обработкой переменных через `javascript()`. Добавляет сгенерированные записи в `$space->storage['elements']` для последующего рендеринга в HTML. |
| **childs** | — | — | Пустая реализация. Наследуется от `Construct`, переопределяется без логики — JS-объекты не создают дочерних DOM-элементов в традиционном понимании. |

### Преобразование строк с поиском и заменой LTS(var) и $(var)

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **javascript** | `string $body`, `mixed $local = false` | `string` | Статический метод обработки JavaScript-кода. Заменяет синтаксические конструкции: `->` на `.`, `LTS(var)` на `LTS.get('id')`, `$(var)` на `jQuery('#id')`. Вызывает `resolveVar()` для подстановки значений переменных из локального хранилища или глобальной области видимости. Возвращает обработанный JS-код. |
| **resolveVar** | `string $name`, `mixed $local`, `string $left`, `string $right` | `string` | Приватный статический метод разрешения переменных. Проверяет наличие переменной в локальном хранилище (`$local`), затем в `$GLOBALS`. Если найдена — вызывает `getVarReplacement()`. Если не найдена — возвращает исходное имя с обёрткой (`$left + $name + $right`). |
| **getVarReplacement** | `mixed $value`, `string $left`, `string $right` | `string` | Приватный статический метод получения замены для переменной. Для объектов с `$id` — возвращает `LTS.get('id')`. Для массивов/объектов — кодирует в JSON. Для строк — экранирует через `json_encode()`. Для скаляров — возвращает значение как есть. |
| **paramobject** | `array $arr` | `string` | Статический метод конвертации PHP-массива в JavaScript-объект. Обрабатывает ключи (экранирует спецсимволы), значения (вызывает `encodeJSValue()`). Возвращает отформатированную строку JavaScript-объекта с отступами. |
| **encodeJSValue** | `mixed $value` | `string` | Приватный статический метод кодирования PHP-значений в JavaScript-формат. Для объектов с методом `id()` — возвращает `LTS.get('id')`. Для массивов — рекурсивно вызывает `paramobject()`. Для строк-функций — возвращает как есть. Для булевых, null, скаляров — соответствующее JS-представление. |

## Примеры использования

```php
// Создание контейнера для JavaScript
$js = LTS::JS('script');

// Добавление внешнего файла
$js->add('path/to/script.js');

// Добавление функции в область скриптов
$js->add('hello()',
<<<JS
    alert("Привет, мир!");
JS
);

// Добавление обработчика события для объекта
$myelement->js('ready')->add('change',
<<<JS
    alert("Объект изменен!");
JS
);

// Добавление встроенного кода в область ready
$myelement->js('ready')->add(
<<<JS
    $(myelement).css('display', 'block');
    LTS(myelement).mymethod();
JS
);
```

## Примечания
*   Класс расширяет `Quark`, поэтому поддерживает все методы управления потомками: `add()`, `get()`, `set()`, `del()`, `getchilds()` и др.
*   Скрипты добавляются через метод `add()` как пары ключ-значение, где ключ — имя/идентификатор скрипта, значение — JS-код или путь к файлу.
*   Внешние JS-файлы определяются по наличию подстроки `'.js'` в значении — такие записи обрабатываются отдельно и генерируют теги `<script src="...">`.
*   Префикс `LTS:` в пути к файлу указывает на загрузку локального файла из файловой системы (через `file_get_contents()`).
*   Три области выполнения: `'script'` — в `<head>` до загрузки DOM, `'ready'` — после `DOMContentLoaded`, `'create'` — при создании объекта (ленивая инициализация).
*   Специальный синтаксис `LTS(var)` заменяется на `LTS.get('id')` для доступа к объектам фреймворка на клиенте.
*   Специальный синтаксис `$(var)` заменяется на `jQuery('#id')` для доступа к DOM-элементам через jQuery.
*   Метод `javascript()` автоматически преобразует PHP-объекты в клиентские ссылки по их свойству `$id`.
*   Локальные переменные для подстановки хранятся в секции `'localvars'` хранилища `$storage`.
*   Методы `compile()` и `childs()` переопределены как пустые — жизненный цикл JS-объекта ограничивается этапом `shine()`.
*   Все скрипты, добавленные через объект JS, автоматически регистрируются в пространстве имён через механизм наследования `addinspace()` из класса `Quark`.
*   Класс `JS` используется методом `Quark::js()` для ленивого создания JavaScript-объектов при первом обращении.
*   При компиляции кода (`$compile == true`) все PHP-переменные, переданные в JS-контекст, автоматически сериализуются в JavaScript-формат.
*   Для функций в значениях массивов проверяется паттерн `function(` — такие строки не экранируются, а вставляются как исполняемый код.
*   Объекты JS не имеют визуального представления в DOM — они генерируют только записи в пространстве имён для вставки в `<script>` теги.
