# Класс Stock

## Назначение
Класс системы управления операциями и синхронизации табличных данных фреймворка LOTIS. Предназначен для атомарного обновления записей с автоматическим пересчётом связанных коллекторов (остатков, сумм, статистик). Решает задачу согласованного обновления двух связанных сущностей: таблицы операций (детальные строки документа) и таблицы остатков (агрегированные значения по ключевым полям). При изменении операций автоматически корректируются остатки без риска рассогласования данных. Класс сравнивает старые и новые значения, определяет изменения (добавление/обновление/удаление), фиксирует их в базе данных и пересчитывает коллекторы с учётом дельты значений.

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$dbtable** | `MySqlTable` | `public` | Таблица базы данных для операций. Устанавливается в конструкторе. Используется для чтения старых значений и записи новых. |
| **$fieldmap** | `array` | `public` | Сопоставление имён полей базы данных с именами полей в передаваемом массиве значений. Формат: `['nameintable' => 'nameinvalues']`. Устанавливается в конструкторе. |
| **$collectors** | `array` | `public` | Массив конфигураций коллекторов для пересчёта остатков. Каждый элемент содержит: `dbtable` (таблица остатков), `selectname` (поле связи), `fieldmap` (сопоставление полей), `plus` (режим: true=увеличение, false=уменьшение). Заполняется через `collector()`. |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `MySqlTable $dbtable`, `array $fieldmap = []` | — | Конструктор класса. Сохраняет таблицу операций в `$dbtable`. Если `$fieldmap` — строка — разбивает через `\LTS::explodestr()` и создаёт маппинг 1:1. Иначе — использует переданный массив. Инициализирует `$collectors` пустым массивом. |
| **collector** | `MySqlTable $dbtable`, `string $selectname`, `array $fieldmap`, `bool $plus = true` | `$this` | Добавляет коллектор для пересчёта остатков. `$dbtable` — таблица остатков. `$selectname` — поле в таблице коллектора, по которому происходит связь (например, ID товара). `$fieldmap` — сопоставление полей коллектора с полями операций. `$plus` — режим: `true` = значения увеличиваются, `false` = уменьшаются. Сохраняет конфигурацию в `$collectors`. Поддерживает fluent interface. |
| **update** | `array $keys`, `array $newvalues` | `$this` | Основной метод фиксации изменений. `$keys` — массив ключей для выборки старых записей (например, `['parent_doc' => 3, 'type' => 'incoming']`). `$newvalues` — массив новых строк документа. Выполняет: (1) Чтение старых значений через `$dbtable->all($keys)`. (2) Сопоставление старых и новых через `matchArrays()`. (3) Удаление удалённых строк через `$dbtable->delall()`. (4) Обновление изменённых через `$dbtable->set()`. (5) Вставка новых через `$dbtable->insert()`. (6) Пересчёт всех коллекторов через `updateCollector()`. Возвращает `$this`. |

## Приватные методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **updateCollector** | `array $collector`, `array $mathresult` | — | Пересчитывает значения в таблице коллектора. Получает конфигурацию коллектора и результат сопоставления из `matchArrays()`. Для каждой пары старых/новых значений вычисляет дельту: старые значения вычитаются, новые прибавляются (или наоборот в зависимости от `$plus`). Накопленные дельты по ID записей применяются через `$db->set()`. Пропускает поля где `abs($val) < 1e-9`. |
| **remapNewValuesKeys** | `array $newItem` | `array` | Преобразует ключи нового значения из формата входного массива в формат базы данных через `$fieldmap`. Для каждого ключа ищет обратное соответствие в `$fieldmap`. Если не найдено — использует ключ как есть. Возвращает массив с ключами базы данных. |
| **matchArrays** | `array $oldvalues`, `array $newvalues` | `array` | Сопоставляет старые и новые значения по ключевым полям из `$fieldmap`. Возвращает массив с ключами: `'matched'` (массив пар `['old' => ..., 'new' => ...]`), `'unmatched'` (массив новых записей без пары). Для каждой старой записи ищет совпадение в новых по всем полям `$fieldmap`. Если найдено — добавляет в `matched` и удаляет из `$remainingNew`. Если не найдено — добавляет в `matched` с `'new' => null` (удаление). |
| **updateDB** | `array $keys`, `array $newItem`, `int $id = null` | — | Обновляет или вставляет запись в базу данных. Для каждого поля из `$newItem` проверяет наличие через `$dbtable->field()` и устанавливает через `$dbtable->value()`. Добавляет ключи из `$keys` (например, ID документа). Вызывает `$dbtable->set($id)` если `$id` указан, иначе `$dbtable->insert()`. |

## Примечания
*   Класс не наследуется от `Construct` — это функциональный класс без жизненного цикла UI-объектов.
*   **Атомарность:** Метод `update()` выполняет все операции в одном вызове — удаление, обновление, вставка и пересчёт коллекторов.
*   **Сопоставление записей:** Через `matchArrays()` определяется какие строки удалены (старая есть, новой нет), обновлены (есть обе) или добавлены (только новая).
*   **Дельта коллекторов:** Для каждого коллектора вычисляется разница между старыми и новыми значениями — применяется к существующим остаткам через сложение/вычитание.
*   **Режим коллектора:** Параметр `$plus` в `collector()` определяет направление: `true` = приход (увеличение), `false` = расход (уменьшение).
*   **Поле связи:** `$selectname` в `collector()` — поле в таблице коллектора для связи с операцией (например, `goods` для связи с товаром).
*   **Маппинг полей:** `$fieldmap` преобразует имена полей между базой данных и входным массивом — позволяет использовать разные имена в коде и БД.
*   **Ключи выборки:** `$keys` в `update()` определяют какие старые записи читать — обычно ID документа или другие идентификаторы группы операций.
*   **Удаление записей:** Записи с `'new' => null` в `matched` удаляются через `$dbtable->delall('id', $idsToDelete)`.
*   **Обновление записей:** Записи с обеими частями в `matched` обновляются через `$dbtable->set($pair['old']['id'])`.
*   **Вставка записей:** Записи из `unmatched` вставляются через `$dbtable->set()` без ID (автоинкремент).
*   **Точность вычислений:** В `updateCollector()` проверяется `abs($val) >= 1e-9` — игнорируются минимальные дельты для избежания ошибок округления.
*   **Несколько коллекторов:** Через `collector()` можно добавить неограниченное количество коллекторов — все пересчитываются в одном вызове `update()`.
*   **Идентификация записей:** В `$newvalues` должно быть поле соответствующее `$selectname` для связи с коллектором (например, ID товара).
*   **Старые значения:** Читаются через `$dbtable->all($keys)` с `$dbtable->asarray = true` — возвращаются как ассоциативные массивы.
*   **Свобода значений:** После `update()` вызывается `$dbtable->freevalues()` — сбрасывает флаги `$isset` для полей.
*   **Ошибки маппинга:** Если поле не найдено в `$dbtable->field()` — пропускается в `updateDB()` через проверку `!== null`.
*   **Группировка дельт:** В `updateCollector()` дельты накапливаются по ID записи в `$objs[$id]` — затем применяются одним `set()` на запись.
*   **NULL значения:** В `updateCollector()` если `$oldval === null` — используется `0` для вычисления дельты.
*   **Обратный маппинг:** В `remapNewValuesKeys()` используется `array_search($key, $fieldmap, true)` — строгое сравнение типов.
*   **Строковый fieldmap:** Если `$fieldmap` передан как строка в конструкторе — разбивается через `\LTS::explodestr()` и создаётся маппинг 1:1.
*   **Коллектор строка:** Если `$fieldmap` в `collector()` передан как строка — разбивается и создаётся маппинг 1:1.
*   **Возврат $this:** Методы `collector()` и `update()` возвращают `$this` для поддержки fluent interface.
*   **Проверка входных данных:** В `update()` проверяется `is_array($keys)` и `is_array($newvalues)` — возвращает `$this` если не массивы.
*   **Пустые старые значения:** Если `$oldvalues === false` — используется пустой массив `[]` — все новые записи считаются добавленными.
*   **ID записей:** В `matchArrays()` используется `$oldvalues[$oldselectname]` для получения ID записи в коллекторе.
*   **Уникальность ID:** В `updateCollector()` через `array_unique()` для `$deletedids` и `$changedids` — предотвращает дублирование.
*   **Исключение удалённых:** В `updateCollector()` через `array_diff($changedids, $deletedids)` — удалённые не обновляются.
*   **Поля коллектора:** В `updateCollector()` проверяется `$db->field($dbkey) !== null` — только существующие поля обновляются.
*   **Строка fieldmap:** В `collector()` и конструкторе — поддерживается строковый формат для простого маппинга 1:1.
*   **Массив collectors:** В `$collectors` хранится конфигурация — каждый элемент обрабатывается в цикле `update()`.
*   **Типы операций:** В `matchArrays()` определяется: добавление (только новое), обновление (оба), удаление (только старое).
*   **Транзакции:** Для атомарности рекомендуется оборачивать `update()` в `$db->transaction()` на уровне MySql.
*   **Производительность:** Все операции выполняются в памяти — минимизирует запросы к базе данных.
*   **Масштабируемость:** Поддерживает большие документы с множеством строк операций и коллекторов.
