# Класс Html

## Назначение
Класс представления произвольных HTML-элементов фреймворка LOTIS, наследующий `Element`. Предназначен для создания HTML-тегов с динамически определяемым именем на основе параметра конструктора. Поддерживает два режима работы: создание стандартного HTML-элемента (div, span, label, p и др.) или подключение внешнего файла содержимого. Класс используется для генерации семантических HTML-тегов, обёрток, контейнеров и вставки внешнего HTML-контента в структуру страницы.

### Основные возможности

* **Создание HTML-элементов** через конструктор
* **Работа с внешними файлами** HTML
* **Гибкая настройка** типа элемента
* **Интеграция** с LTS-объектами

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$filename** | `string/null` | `public` | Путь к внешнему файлу для подключения содержимого. Устанавливается в конструкторе если `$id` содержит точку. Используется методом `shine()` для загрузки файла. |

### Наследуемые свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта, используется как HTML-атрибут `id`. |
| **$type** | `string` | `public` | Наследуется от `Element`. Устанавливается в `'file'` если `$id` содержит точку, иначе `'html'`. Определяет тип обработки элемента. |
| **$tagname** | `string` | `public` | Наследуется от `Element`. Устанавливается в конструкторе на основе `$id`. По умолчанию `'div'` если `$id` пустой. Определяет HTML-тег элемента при рендеринге. |
| **$caption** | `string` | `public` | Наследуется от `Element`. Текстовое содержимое элемента. Отображается как текст внутри тега или игнорируется для файлов. |
| **$classname** | `string` | `public` | Наследуется от `Element`. CSS-классы элемента, управляются методами `addclass()`, `removeclass()`, `hasclass()`. |
| **$storage** | `array` | `public` | Наследуется от `Quark`. Хранилище содержит секции `'childs'` (потомки), `'element'` (DOM-элемент), `'attr'` (HTML-атрибуты). |

## Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Element`. Если `$id` содержит точку (`.`) — устанавливает `$type` в `'file'` и сохраняет `$id` как `$filename` для загрузки внешнего файла. Иначе — устанавливает `$tagname` в значение `$id` или `'div'` если `$id` пустой. |
| **shine** | `Element $parent = null` | — | Рендеринг элемента. Вызывает родительский `shine()` для стандартной обработки. Если `$type == 'file'` — устанавливает имя файла в секцию `'element'` через `set('element', 'filename', $this->filename)` для последующей загрузки содержимого файла пространством `Space`. |

## Примеры использования

```php
// Создание ul-элемента
$ul = LTS::Html('ul');
$ul->add(LTS::Html('li')->capt('Item 1'))
    ->add(LTS::Html('li')->capt('Item 2'))
    ->add(LTS::Html('li')->capt('Item 4'));
```

Работа с файлами

```php
// Загрузка HTML из файла
$html = new Html('path/to/file.html');
```

## Примечания
*   Класс расширяет `Element`, поэтому наследует все методы управления потомками, стилями, атрибутами и событиями.
*   Класс расширяет `Quark` и `Construct`, поэтому поддерживает полный жизненный цикл объектов: `compile()`, `shine()`, `childs()`, `addinspace()`, а также хуки.
*   **Режим тега:** Если `$id` не содержит точку — создаётся HTML-тег с именем `$id` (например, `'p'`, `'span'`, `'label'`, `'section'`).
*   **Режим файла:** Если `$id` содержит точку (например, `'header.html'`) — содержимое загружается из файла через механизм `Space::processFile()`.
*   **Тег по умолчанию:** Если `$id` пустой и не является файлом — создаётся тег `'div'`.
*   **Атрибуты:** Дополнительные атрибуты добавляются через `attr()` — `class`, `style`, `data-*` и др.
*   **Потомки:** Элемент может содержать потомков через `add()` — рендерятся внутри тега.
*   **Рендеринг:** При `shine()` создаётся HTML-тег с атрибутами и содержимым, или загружается файл.
*   **Пространство имён:** Регистрируется через `addinspace()` — доступно в пространстве `Space` для сборки страницы.
*   **События:** Поддерживает назначение обработчиков событий через `on()` или `click()` — генерируют клиентский JavaScript-код при компиляции.
*   **Хуки:** Поддерживает хуки жизненного цикла — `check`, `before`, `on` для внедрения логики.
*   **Стили:** CSS-свойства добавляются через `css()` — для кастомизации внешнего вида.
*   **Классы:** CSS-классы добавляются через `addclass()` — для стилизации через внешние файлы.
*   **ID:** Уникальный идентификатор используется для связи с клиентским объектом через `LTS.get('{id}')`.
*   **Тип:** Устанавливается в `'html'` или `'file'` — для рендеринга в `Space`.
*   **Компиляция:** Поддерживает `compile()` для генерации клиентского JavaScript-кода при необходимости.
*   **Содержимое:** Текст устанавливается через `$caption` или метод `capt()` — отображается внутри тега.
*   **Файлы:** Для режима файла содержимое загружается через `file_get_contents()` в `Space::processFile()`.
*   **Путь к файлу:** Относительный путь от корня проекта — файл должен существовать до рендеринга.
*   **Кэширование:** Содержимое файла не кэшируется — загружается при каждом рендеринге.
*   **Безопасность:** Путь к файлу не валидируется — ответственность на разработчике.
*   **Вложенность:** Может содержать другие элементы через `add()` — для сложной структуры.
*   **Наследование:** Все методы `Element` доступны — `css()`, `attr()`, `add()`, `on()` и др.
*   **Использование:** Подходит для семантических тегов, обёрток, контейнеров и вставки внешнего HTML.
