# Класс Vars

## Назначение
Класс управления глобальными переменными фреймворка LOTIS. Предназначен для хранения и синхронизации данных между сервером и клиентом в рамках пользовательской сессии. Серверная часть обеспечивает сохранение значений в `$_SESSION`, восстановление состояния при загрузке страницы и автоматическую синхронизацию при AJAX-запросах событий. Клиентская часть предоставляет JavaScript API для работы с переменными в браузере, включая поддержку localStorage для клиентского кэширования. Все экземпляры класса регистрируются в статическом реестре для автоматической синхронизации при обработке событий.

## Свойства (серверная часть, PHP)

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$storage** | `array` | `public` | Ассоциативный массив для хранения переменных. Ключи — имена переменных, значения — произвольные PHP-данные (скаляры, массивы, объекты). |
| **$elements** | `array` | `private` | Массив метаданных для генерации клиентского JavaScript-кода. Заполняется методом `shine()` и используется при рендеринге пространства. |
| **$all** | `static array` | `public static` | Статический реестр всех созданных объектов Vars по ключам вида `"__globals{id}"`. Используется классом `Events::build()` для автоматической синхронизации переменных при обработке событий. |

### Наследуемые свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$id** | `string` | `public` | Наследуется от `Construct`. Уникальный идентификатор объекта Vars. Если не передан — генерируется автоматически. Используется как часть ключа сессии. Позволяет создавать отдельные именованные хранилища для глобальных переменных |
| **$type** | `string` | `public` | Наследуется от `Construct`. Устанавливается в `'Vars'` в конструкторе. Определяет тип объекта для фильтрации и обработки. |
| **$space** | `Space` | `public` | Наследуется от `Construct`. Ссылка на пространство имён, куда добавляются сгенерированные элементы. |

## Методы (серверная часть, PHP)

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $id = ''` | — | Конструктор класса. Вызывает родительский конструктор `Construct`, устанавливает `$type` в `'Vars'`, инициализирует `$storage` пустым массивом. Вызывает `restore()` для загрузки данных из сессии. Регистрирует объект в статическом реестре `Vars::$all` по ключу `"__globals{$id}"`. |

### Управление данными

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **restore** | — | `$this` | Восстанавливает значения переменных из сессии PHP. Проверяет наличие ключа `"__globals{$id}"` в `$_SESSION`. Если данные существуют и являются массивом или объектом — преобразует в массив и загружает в `$storage`. Возвращает `$this` для поддержки fluent interface. |
| **store** | — | `$this` | Сохраняет текущие значения `$storage` в сессию PHP. Записывает массив в `$_SESSION["__globals{$id}"]`. Вызывается автоматически при обработке событий через `Events::build()`. Возвращает `$this` для поддержки fluent interface. |
| **clearsession** | — | `bool` | Удаляет данные объекта из сессии PHP. Находит ключ `"__globals{$id}"` в `$_SESSION` и удаляет через `unset()`. Не затрагивает `$storage` в памяти. Возвращает `true` при успехе. |
| **clearvalues** | `bool $session = true` | `bool` | Очищает значения всех переменных, сохраняя ключи. Проходит по `$storage` и устанавливает каждое значение в пустую строку. Если `$session == true` — также вызывает `clearsession()`. Возвращает `true`. |
| **clear** | `bool $session = true` | `bool` | Полностью очищает хранилище переменных. Устанавливает `$storage` в пустой массив. Если `$session == true` — также вызывает `clearsession()`. Возвращает `true`. |
| **get** | `string $name` | `mixed/null` | Получает значение переменной по имени. Если ключ существует в `$storage` — возвращает значение. Иначе возвращает `null`. Не вызывает исключений при отсутствии ключа. |
| **set** | `string $name`, `mixed $value = null` | `$this` | Устанавливает значение переменной. Если `$value === null` — удаляет переменную из `$storage` через `unset()`. Иначе записывает значение по ключу. Возвращает `$this` для поддержки fluent interface. |
| **value** | `string $name`, `mixed $value = null` | `mixed/$this` | Универсальный метод получения/установки значения. Если `$value === null` — работает как `get()` (возвращает значение). Если передано — работает как `set()` (устанавливает значение и возвращает `$this`). |
| **del** | `string $name` | `$this` | Удаляет переменную из хранилища. Находит ключ в `$storage` и удаляет через `unset()`. Не затрагивает сессию до вызова `store()`. Возвращает `$this` для поддержки fluent interface. |

### Жизненный цикл

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **shine** | `Element $parent = null` | — | Генерация метаданных для клиентского JavaScript. Кодирует `$storage` в JSON через `json_encode()`. Заполняет `$elements` массивом с типом `'VARS'`, областью `'script'`, ID объекта и телом скрипта. Не создаёт DOM-элемент напрямую. |
| **addinspace** | — | — | Регистрация переменных во вселенной. Формирует ключ `"__globals{$id}"` и добавляет `$elements` в `$space->storage['elements']` через метод `set()`. Обеспечивает передачу данных клиенту при рендеринге страницы. |

## Клиентские Методы

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **constructor** | `string id` | — | Конструктор класса. Инициализирует свойство `id` и пустой объект `values`. Вызывается автоматически при инициализации через `LTS.vars()`. |
| **get** | `string name` | `mixed/null` | Получает значение переменной по имени из клиентского хранилища `values`. Если ключ не существует — возвращает `null`. |
| **set** | `string name`, `mixed value` | `this` | Устанавливает значение переменной в клиентском хранилище. Если `value === undefined` или `null` — удаляет переменную из `values`. Возвращает `this` для поддержки цепочечных вызовов. |
| **varsname** | — | `string` | Возвращает имя ключа для сессии/localStorage. Формирует строку вида `"__globals{id}"` или `"__globals"` если ID пуст. |
| **store** | — | — | Сохраняет текущие значения `values` на сервер через AJAX POST-запрос. Отправляет данные через `LTS.post('__ltssavevars', window.location.href, args)`. Данные сохраняются в `$_SESSION` на сервере. |
| **localsave** | `string name = null` | — | Сохраняет значения в localStorage браузера. Если `name` указан — сохраняет только одну переменную. Если не указан — сохраняет все `values`. Данные сохраняются под ключом из `varsname()`. |
| **localload** | `string name = null` | `mixed/bool` | Загружает значения из localStorage браузера. Если `name` указан — загружает и возвращает одну переменную. Если не указан — загружает все значения в `values` и возвращает `true`. Возвращает `false` если данные не найдены. |

## Глобальные функции объекта LTS для работы с Vars

| Функция | Параметры | Возвращает | Описание |
|---------|-----------|------------|----------|
| **LTS.vars(name, values)** | `string name`, `object values` | `Vars/null` | Если `values` передан — создаёт или обновляет объект Vars с указанными значениями. Если не передан — возвращает существующий объект Vars по имени или `null`. Для объекта без ID (name не указан) используется `"__globals"`. |
| **LTS.vars(varsname).values** | — | `object` | Прямой доступ к хранилищу значений объекта Vars. varsname - не обязателен. Если он указан, то работа осуществляется с вектором глобальных переменных `"__globals{id}"`, если не указан, то с `"__globals"` |
| **LTS.vars(varsname).set(name, value)** | `string name`, `mixed value` | `Vars` | Установка значения переменной на клиенте. |
| **LTS.vars(varsname).get(name)** | `string name` | `mixed` | Получение значения переменной на клиенте. |
| **LTS.vars(varsname).store()** | — | — | Синхронизация клиентских значений с сервером. |

## Примеры использования

### На сервере

```php
// Создание хранилища
$vars = LTS::Vars('global');

// Установка значений
$vars->set('counter', 10);
$vars->value('name', 'John');

// Получение значений
$value = $vars->get('counter');

// Сохранение в сессию
$vars->store();

// Очистка
$vars->clear();
```

### На клиенте

```JS
// Получение значений
const name = LTS.vars('global').get('name');
let counter = LTS.vars('global').get('counter');

// Установка значений 
LTS.vars('global').set('counter', ++counter);

// Сохранение значений в сессии
LTS.vars('global').store();

```

## Примечания
*   Класс расширяет `Construct`, поэтому поддерживает все хуки жизненного цикла: `$check`, `$before`, `$on`, `$checkchilds`, `$beforechilds`, `$onchilds`.
*   Все объекты Vars автоматически регистрируются в статическом реестре `Vars::$all` — это необходимо для синхронизации с клиентом при AJAX-запросах.
*   Ключ сессии формируется как `"__globals{$id}"` — при пустом ID будет `"__globals"`.
*   Метод `store()` вызывается автоматически в `Events::build()` после обработки каждого события — данные синхронизируются без явного вызова разработчиком.
*   Метод `value()` поддерживает два режима работы: геттер (без второго параметра) и сеттер (с вторым параметром) — аналогично jQuery `.val()`.
*   При установке значения в `null` через `set()` переменная удаляется из хранилища — это отличается от установки пустой строки.
*   Метод `clearvalues()` обнуляет значения, но сохраняет ключи — полезно для сброса формы без потери структуры данных.
*   Метод `clear()` полностью очищает хранилище — удаляет все ключи и значения.
*   Данные в `$storage` могут быть любого типа — массивы, объекты, скаляры. При сохранении в сессию происходит сериализация PHP.
*   При рендеринге `$storage` кодируется в JSON и передаётся клиенту — сложные типы данных преобразуются в JavaScript-объекты.
*   На клиенте данные доступны через `LTS.vars('{id}').values` или `LTS.vars().values` для объекта без ID.
*   Методы `shine()` и `addinspace()` не создают визуальных элементов — Vars является функциональным объектом без DOM-представления.
*   Класс не реализует методы `compile()` и `childs()` — они наследуются от `Construct` без изменений.
*   При обработке событий через `Events::build()` все объекты Vars автоматически вызывают `store()` — изменения сохраняются в сессию перед отправкой ответа клиенту.
*   Для очистки сессии при выходе пользователя рекомендуется явно вызывать `clearsession()` или `clear(true)`.
*   Объекты Vars с одинаковым ID делят одно сессионное хранилище — создание второго объекта с тем же ID загрузит те же данные из сессии.
*   Метод `restore()` вызывается только в конструкторе — для принудительной перезагрузки из сессии нужно явно вызвать `restore()`.
*   Все методы модификации данных (`set`, `value`, `del`, `clear`, `clearvalues`) возвращают `$this` для поддержки fluent interface и цепочных вызовов.
*   Клиентский класс Vars.js поддерживает localStorage для сохранения данных между перезагрузками страницы без серверной синхронизации.
*   Метод `localsave()` полезен для сохранения состояния формы или настроек интерфейса без нагрузки на сервер.
*   Метод `localload()` позволяет восстановить данные из localStorage при загрузке страницы до получения ответа от сервера.
*   Серверная синхронизация через `store()` происходит автоматически при событиях — явный вызов требуется только для сохранения без событий.
*   Для работы клиентской части требуется наличие файла `Vars.js`, который подключается автоматически через пространство имён Space.
*   При рендеринге страницы данные Vars передаются через скрипт инициализации `LTS.vars("{id}", {body})` где `{body}` — JSON-представление хранилища.
*   Изменения, сделанные на клиенте через `LTS.vars().set()`, не сохраняются на сервере до вызова `store()` или обработки события.
*   Клиентские и серверные данные синхронизируются двусторонне — изменения на сервере передаются клиенту в ответе события через поле `__ltsVars`.
