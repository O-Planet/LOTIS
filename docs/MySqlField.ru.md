# Класс MySqlField

## Назначение
Класс представления поля таблицы базы данных в ORM фреймворка LOTIS. Предназначен для хранения метаданных о поле: имя, тип, размер, nullable, значение по умолчанию, ENUM-значения, ссылки на другие таблицы. Используется классом MySqlTable для определения структуры таблицы и генерации SQL-запросов CREATE TABLE, ALTER TABLE. Класс предоставляет методы для настройки свойств поля через fluent interface и генерации SQL-определения типа поля. Объекты MySqlField создаются автоматически при определении таблицы через методы MySqlTable (string(), int(), enum() и др.) или при загрузке структуры из БД через MySql::loadtable().

## Свойства

| Свойство | Тип | Видимость | Описание |
|----------|-----|-----------|----------|
| **$name** | `string` | `public` | Имя поля в таблице. Устанавливается в конструкторе. Используется как ключ в массиве `$table->fields`. |
| **$type** | `string` | `public` | Тип поля (VARCHAR, INT, TEXT, ENUM, DECIMAL, DATETIME, TABLE, FILE, BOOL и др.). Устанавливается в конструкторе. Определяет SQL-тип и поведение поля. |
| **$param** | `mixed/null` | `public` | Параметр типа поля. Для VARCHAR/CHAR — длина (int). Для DECIMAL — массив `['dd' => x, 'dr' => y]`. Для FILE — массив `['ext' => ..., 'dir' => ...]`. Для остальных типов — `null`. |
| **$table** | `string/MySqlTable/null` | `public` | Ссылка на таблицу для полей типа `'TABLE'`. Может быть именем таблицы (string) или объектом MySqlTable. Используется для определения внешних ключей. |
| **$value** | `mixed/null` | `public` | Текущее значение поля для вставки/обновления. Устанавливается через `setvalue()`. Используется при генерации INSERT/UPDATE запросов. |
| **$values** | `array/null` | `public` | Допустимые значения для типа ENUM. Массив строк или ассоциативный массив (ключ => значение). Используется при генерации SQL и валидации. |
| **$isset** | `bool` | `public` | Флаг установки значения. Устанавливается в `true` при вызове `setvalue()`. Используется для определения, какие поля включать в INSERT/UPDATE. |
| **$default** | `mixed/null` | `public` | Значение по умолчанию для поля. Устанавливается напрямую или через MySqlTable. Используется при генерации CREATE TABLE. |
| **$nullable** | `bool/null` | `public` | Флаг возможности NULL-значения. `true` — NULL разрешён, `false` — NOT NULL, `null` — не определено. Управляется методом `nullable()`. |
| **$out** | `mixed/null` | `public` | Дополнительное поле для пользовательских данных. Не используется в базовой функциональности ORM. Может использоваться для расширения. Предполагается использование для интерпретации полученных данных. |

## Методы

### Конструктор

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **__construct** | `string $name`, `string $type`, `mixed $param = null` | — | Конструктор класса. Сохраняет имя и тип поля. Для VARCHAR/CHAR `$param` — длина (int). Для DECIMAL `$param` — массив `['dd' => x, 'dr' => y]`. Для FILE `$param` — массив с настройками хранения. Инициализирует все свойства в значения по умолчанию (`null`, `false`). |

### Установка значений

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **setvalue** | `mixed $val` | `$this` | Устанавливает значение поля для последующей вставки/обновления. Сохраняет в `$value` и устанавливает флаг `$isset` в `true`. Возвращает `$this` для поддержки fluent interface. Вызывается через `MySqlTable::value()` или напрямую. |
| **nullable** | `bool $val = true` | `$this` | Устанавливает флаг `$nullable`. Если `$val === true` — поле может принимать NULL. Если `false` — NOT NULL. Возвращает `$this` для поддержки fluent interface. Вызывается при определении структуры таблицы. |

### Получение информации

| Метод | Параметры | Возвращает | Описание |
|-------|-----------|------------|----------|
| **getnull** | — | `string` | Возвращает SQL-представление nullable-флага. Если `$nullable === null` — возвращает пустую строку. Если `true` — `'NULL'`. Если `false` — `'NOT NULL'`. Используется при генерации CREATE TABLE. |
| **gettype** | — | `string` | Генерирует SQL-определение типа поля для запроса CREATE TABLE. Возвращает строку вида `VARCHAR(100)`, `INT(11)`, `DECIMAL(15,2)`, `ENUM('val1','val2')`, `TINYINT(1)`. Для типа `'TABLE'` возвращает `'INT(11)'` (внешний ключ). Для типа `'FILE'` возвращает `'VARCHAR(255)'`. |

## Примеры использования

```php
$table->field('name')->nullable(false);
```

## Примечания
*   Класс создаётся автоматически через методы MySqlTable (`string()`, `int()`, `enum()`, `table()` и др.) — прямое создание через `new MySqlField()` не рекомендуется.
*   Все методы настройки возвращают `$this` для поддержки fluent interface и цепочечных вызовов.
*   Тип `'TABLE'` обозначает внешнюю ссылку — при генерации SQL создаётся поле INT(11) с суффиксом `_id`.
*   Метод `gettype()` используется при создании таблицы — генерирует корректное определение поля для MySQL.
*   Для ENUM-полей свойство `$values` содержит массив допустимых значений — используется для валидации и генерации UI через `MySqlTable::generateinputs()`.
*   Флаг `$nullable` влияет на генерацию SQL — добавляет `NULL` или `NOT NULL` в определение поля через `getnull()`.
*   Значение по умолчанию `$default` не экранируется в этом классе — экранирование происходит в MySqlTable::create().
*   Флаг `$isset` определяет, включается ли поле в INSERT/UPDATE запросы — только поля с `$isset === true` обрабатываются.
*   Для полей DECIMAL свойство `$param` хранит два параметра — `dd` (цифры до запятой) и `dr` (цифры после).
*   При загрузке структуры через `MySql::loadtable()` объекты MySqlField создаются автоматически на основе `information_schema.COLUMNS`.
*   Свойство `$table` для полей-ссылок может содержать имя таблицы или объект — при генерации SQL используется имя.
*   Метод `setvalue()` вызывается через `MySqlTable::value()` — прямое использование не рекомендуется.
*   Типы полей соответствуют MySQL — VARCHAR, INT, TEXT, DECIMAL, DATETIME, ENUM, TINYINT, FILE, TABLE, POINT.
*   Для булевых полей используется тип `'BOOL'` — при генерации SQL преобразуется в `TINYINT(1)`.
*   Метод `gettype()` не включает имя поля — только определение типа и модификаторов.
*   При валидации ENUM проверяется вхождение в `$values` — регистр имеет значение.
*   Для полей TABLE валидация требует подключения к БД — проверяет существование записи по ID.
*   Объекты MySqlField кэшируются в `$table->fields` — повторные обращения возвращают тот же объект.
*   При изменении структуры таблицы нужно пересоздавать поля — кэш не обновляется автоматически.
*   Метод `nullable()` может вызываться многократно — последнее значение переопределяет предыдущее.
*   Для полей TEXT и BLOB длина не указывается — свойство `$param` игнорируется в `gettype()`.
*   Значение по умолчанию `CURRENT_TIMESTAMP` для DATETIME поддерживается — передаётся как строка в `$default`.
*   При генерации SQL для DECIMAL используется формат `DECIMAL(dd, dr)`.
*   Для ENUM при генерации SQL значения экранируются — одинарные кавычки внутри значений удваиваются.
*   Класс не реализует кастомные типы — только стандартные типы MySQL.
*   Свойство `$name` используется как ключ в `$table->fields` — имена полей уникальны в пределах таблицы.
*   При загрузке из БД комментарии полей (`COLUMN_COMMENT`) игнорируются — не сохраняются в объекте.
*   Для автоинкрементных полей отдельного типа нет — определяется через EXTRA поле в `information_schema`.
*   Класс не реализует валидацию данных — ответственность на разработчике или UI-компонентах.
*   При генерации INSERT/UPDATE только поля с `$isset === true` включаются в запрос.
*   Для полей FILE параметр `$param` может содержать директорию для хранения файлов — проверяется в MySqlTable::file().
*   Класс совместим с MySQL 5.7+ и MariaDB 10.0+.
*   Объекты MySqlField не имеют методов для прямой работы с БД — все операции выполняются через MySqlTable.
*   При загрузке структуры таблицы поле `id` игнорируется — считается первичным ключом по умолчанию.
*   Для полей-ссылок тип `'TABLE'` преобразуется в `INT(11)` при генерации SQL — внешний ключ не создаётся автоматически.
*   Для ENUM ассоциативные массивы преобразуются в список значений — ключи используются как значения в SQL.
*   Класс не поддерживает изменения типа поля после создания — для модификации нужно использовать MySqlTable::update().
*   Свойство `$value` сбрасывается после INSERT/UPDATE через `MySqlTable::freevalues()` — флаг `$isset` устанавливается в `false`.
